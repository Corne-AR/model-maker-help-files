MENU : Irr - Def & Chk - Defaults

 

This dialog box is activated by clicking the Defaults button on the Default and Check page on the irrigation main menu.

 

 

Certain defaults applicable to the plotting and calculations are user definable.

 

Edit

 

This setting has an effect on the manner that Irri Maker treats the adding of new nodes to positions outside of the active block area.

 

Allow

 

New nodes outside the active block area will be added to the data base. No complaint or warning will be given.

 

Allow / warn

 

You will be warned that the node is outside the block area. The node will only be added to the project after confirmation by the user.

 

Not allowed

 

No new nodes outside the active block area will be accepted.

Use any of the three settings according to your designing style.

 

Plot

 

Plotting pipe Class & Length

These settings allow you to specify the attributes and text style when writing relative information along the pipes.

 

Length                : the actual length of the pipe can be written along side each pipe.

Name                : the description of the pipe, as specified in the pipe data base, can be written with the pipe.

Class                : the pipe class can be displayed.

 

Use the Above and Below option to specify which labels should be on which side of the pipe.

 

Check the box on the pipe page to activate the labeling of the pipes.

 

Text direction

 

                               

 

The text will be plotted along the pipe and in the direction that the water flows.

This could result in some text being written upside down.

 

       

 

The text will be plotted along the pipe in such a way that it is always in a readable upright position.

This would prevent text being written upside down but it would not give any indication of the direction of the water flow.

 

Do not label short pipes

You may enter the minimum length of pipe that must not be labeled. Enter a value of zero if all the pipes must be labeled.

Change this value if you find that the text becomes cluttered in areas of short length pipes.

If necessary, copy the pipe information as regular text to CAD where it may manipulated in full.

 

Maximum deflection

This parameter controls the maximum deflections that a sequence of pipes may have whilst treating them as a straight line. If multiple nodes are in a straight line and are connected by means of pipes with the same diameter, Irri Maker will not write the length and diameter of each segments individually but only label the total length of all the segments in the centre of the pipe line.

If the maximum deflection angle is increased, pipe segments running around a slight curve will also be treated as a straight line.

 

Minimum distance between labels

This parameter can be used to control the density of the labels on the pipes. If a certain value is specified, Irri Maker will not label 2 of the same pipes when the one is closer than the specified number of millimeters from the other. Notice that the entered distance is in plotting units and is related to the plotting scale.

If you enter a value of 20millimeters and the plot is done at a scale of 1/500, the pipes will then not be labelled if they are physically closer than 10meters.

 

Block boundary

If this box is not checked the boundary lines of all the blocks will, although displayed on the screen, not be drawn on the plotter. This is often useful if you do not want to confuse the farmer with lines around the design blocks. Also, the block boundaries could overlay the actual pipe lines and therefore obliterate important information.

 

Pipe line widths

By default all the pipes will be drawn with a single pixel wide line. If you want to easily distinguish between the different pipe types, use the scroll boxes on the side and assign different values to each. A good idea is to draw the main lines with 3 pixels, the manifolds with 2 pixels and the lateral pipes with 1 pixel wide lines.

 

Shift Colours

This is the colour specified in the node and pipe data bases. By assigning different colours to each shift and selecting the shift colour option on the node and pipe pages, Irri Maker will draw the irrigation elements in the colour of their specific shift number.

 

To change the colour of a shift, click on the shift panel. A colour dialog box will be displayed. From this colour dialog box pick the colour you want to associate with the selected shift.

 

Block fill % colour

Use this value to control the intensity of the background shift colour. The actual filling of the blocks with the solid shift colour is done on the shifts page.

When doing the hydraulic calculations, Irri Maker uses the Hazen Williams formula to determine the friction losses in the pipes. You may specify that the Darcy Weisbach formula should rather be used for pipes smaller than the specified diameter.

Hazen Williams

Model Maker uses this equation to calculate the friction losses unless the Darcy Weisbach option is selected for pipes with an inside diameter smaller than the user defined value.

 

         Hf = L  [  3.59 Q  /  Ch d^2.63  ] ^ 1.852

         

         Hf                = friction

         L                = pipe length

         Q                = flow rate

         Ch                = Hazen Williams co-efficient - (This value is set in the pipe data base)

         d                = internal diameter of pipe.

 

This equation is generally freely used in the turbulent flow region (Re>4000) for pipes with larger diameters.

 

Darcy Weisbach

This formula is more complicated than Hazen Williams and depends on the Reynolds value.

 

Note:

The Darcy Weisbach formula is only used when the option is selected (in the above dialog) and then only for pipes with a diameter smaller than the indicated size.

 

         Re = d v p / u

 

         Re                = Reynolds number

         d                = internal diameter of pipe

         v                = average velocity of flow

         p                = density of liquid (997.28kg / m3 at 24 deg for water)

         u                = dynamic viscosity (0.00092 kg / m.s for water at 24 deg Celsius)

 

         Darcy Weisbach formula

         

         hf = f L v^2 / 2 g d

 

         Hf                = friction

         f                = friction factor

         L                = pipe length

         v                = average velocity of flow

         g                = gravitational acceleration (10 m/s^2)

         d                = internal diameter of pipe

 

         The friction factor depends on the Reynolds number and are calculated as follows.

 

         Re < 2000

 

         f = 64 / Re

 

         Re > 4000

 

         In this case the Colebrook White transition law should be used but unfortunately, due to its implicit nature this equation is   seldom used.

 

         Model Maker uses an explicit substitute called Jain's equation to calculate the friction factor where k=roughness in mm

 

         1 / sqrt(f) = 1.14 - 2.log [ k / d + 21.25 / Re^0.9 ]

 

         f                = friction factor

         k                = equivalent pipe roughness factor  (This value is set in the pipe data base)

         d                = internal diameter of pipe

         Re                = Reynolds number

         

 

Note:

When using either one of the 2 formulas, ensure that the correct roughness co-efficient is specified for the pipe in the pipe data base.

 

If this box is checked, Irri Maker will warn you when it cannot interpolate an elevation for the node from the DTM surface you each time a node is added or moved  This could either be because no survey data is loaded or the node falls outside the surveyed area.

 

If this box is checked, all the calculations will be done by using the pipe length as calculated between the 2 end nodes taking the elevation difference into account. This would result in a longer pipe length in the case of a steep terrain.

If the box is not checked, the pipe lengths will be calculated as a 2D plan length.

Irrelevant of the setting of this check box, all node elevations will be taken into account when calculating the pressure differences in the system.
