MENU : Utility - Electronic - Trig data edit

 

 

Edit the code,Y,X or elevation of the trig data. Data can also be manually added to the list. Use the save data function to save this data to disk. The data in this trig data section is only used to reduce fieldbook observations and is not automatically added to the normal DTM database or plotted on the screen.  

 

Init trigs

The trig file is not cleared automatically between reductions of different fieldbook data. This may cause confusion in some instances when the same trig names are used on different projects. However, some users only on one site and do not want to have the trig file cleared every time MM is started. This option deletes all the trigs in memory. Save the current trigs in a YXZ file before this operation if the data is going to be used again.  

 

Load crd

Load the trigdata file as produced by BOOKER. Use Comms Link to download the file from the Psion to the PC. Use .crd as the file extension. Although this is a ASCII file, the columns are spaced by a TAB character and can not be load by Model Maker in the normal format. This trig data are used to reduce the fieldbook observations.

 

Load yxz

Load normal MM yxz data into the trig section and use the co-ordinates to reduce the fieldbook observations.  

 

Save trig data                          

Saves the trig data in the normal Model Maker ASCII format. It does not matter if the data was loaded as CRD or ASCII format. This file can later again be used as trig data or, in the files menu, be loaded as normal DTM data.  

 

 

Specify default codes.  Any reduced point with a code starting with any of the default characters will be added to the trig list.  2 Default codes may be specified. Note that each one works in a different format.

 

Code 1 : The code of any reduced point starting with this characters will be kept unchanged.

Eg.:  CODE 1=ST  and File Code=STBB1  > trigfile=STBB1

 

Code 2 : The code of any reduced point starting with this characters will be changed. The default characters will be taken off.

Eg.:  CODE 2=ST  and File Code=STBB1  > trigfile=BB1 or   CODE 2=;   and File Code=;A     > trigfile=A  

 

Checking the Active box instructs Model Maker to apply the testing to the data.
