Description - Model Maker Code and Tag

Use the options on this  panel to configure the reading of the Code and Tag fields.

Most of the electronic data formats allow the surveyor to identify each surveyed point by means of a description and a number. In Model Maker, each DTM point has a code and tag field.

 

EXAMPLE of the Booker format:

L6                                2                8.0394                                9.5959                0.000                1.563

L5                                4                76.3749                                88.2003                0.000                0.000

L7                                4                352.4348                88.4313                0.000                0.000

321                HEK                5                353.5244                88.363                81.380                1.300

322                HEK                5                353.5607                88.3545                77.363                1.300

323                DRD                5                354.0425                88.4126                66.861                1.300

 

Notice that the last 3 lines have both a line number and description in the first 2 columns.

 

Use the four check boxes to set the relation between the file description and numbers and the tag and code fields in Model Maker. In this example the point number in the electronic data file will be placed in the Code field in the Model Maker data base. The description of each survey point will be placed in the Tag field in the Model Maker data base.

 

Yom data format

Select which of the 2 Yoms formats must be used to reduce the Yoms survey data.

 

Defaults

Use the defaults page to specify the radius of the earth and what the temperature was when the survey was done. These values are used when the correction factors are calculated when reducing the fieldbook data.

 

Press the Defaults button to use the standard values.
