SURVEY : Line area

 Top  Previous  Next

 

MENU : sUr - Line_area

 

This is a quick way of calculating the area as defined by 3 or more CAD lines. Simply point to the inside of the area to be calculated.

 

Note:

The final result is not 100% accurate since Model Maker uses a manual method to approximate the answer.

From the position where the cursor is placed, the program will draw horizontal lines in steps of 100mm.. The lengths of these lines will be determined by where it intersects the surrounding CAD lines. The area is thus accumulated by calculating the area that each line contributes to the final picture. (Area = line length * 100mm).

Note also that this function does not require the boundary CAD lines to be a continuous polyline.

Because of the graphical nature of this method, Model Maker is also not able to calculate the answer correctly if the defined area consists of 'dead spaces'.

The answer will not be complete if no intersection with a CAD line is found.

 

EXAMPLES:

 

This figure shows the temporary brown lines used to calculate the area between lines a,b,c,d.

Note the white cursor position in the centre.

The area as indicated by the red cursor position is impossible to calculate since the program is unable to find an intersection with any of the 4 CAD lines to the right of the cursor.

 

 

 

A complicated area is not calculated correctly because of the 'dead space' indicated by 'A'.

 

The function provides a easy way to quickly calculate the area of certain shapes that do not require a precise answer. (eg., cross-section areas).

If you are concerned about the accuracy of this method, first calculate a sample area using the accurate area method. Compare the 2 results and then decide if this method can be used.
