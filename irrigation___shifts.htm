MENU : Irr - Shifts

 

Irri Maker makes extensive use of designing groups of nodes and pipes together in blocks. These blocks normally consists of sprinkler nodes, lateral pipes joining them up and are connected to the source valve by means of manifold pipes and fitting nodes.

Most of the time, the main line source pump cannot supply enough water to the whole irrigation system to simultaneously maintain the flow to all the block. The blocks are then designated to operate in certain shifts.

 

Apart from grouping sprinkler in blocks, regular valves and sprinklers can also be defined to operate directly in the main line. The purpose of this in not to eliminate or substitute the blocks but to manage certain pipe layouts not associated with blocks like wind rows between the blocks.

 

VALVES

 

A valve node in Irri Maker can have two types of functions.

1 : It acts as a demand node which simulates a position where water is taken out of the system

2 : It acts as a shutoff valve to turn the flow to all the downstream pipes connected to it, on and off.

 

Valves defined by the blocks act as demand nodes within the main line. Other regular valves defined by the user within the main line can either be demand or shutoff valves.

 

BLOCKS

 

The difference between valves linked to blocks and regular valves is the following.

 

Valves on blocks

 

After creating each block boundary, a valve must be added to the block. This valve controls the main entry into the block and the flow and pressure values are determined when the hydraulics of each block is calculated. Although the valve belongs to the block, it is available to the main line to make the necessary pipes connections. The main line can only connect pipes to this node but cannot change the hydraulic properties of the valve. This can be done by first activating the block in question and then recalculating the hydraulics or by editing the valve properties manually.

 

These block valves act as demand nodes because on the main line, it simulates the water draw off that all the sprinkler would consume. These valves are always in an open state and cannot be used to shutoff the water flow through it to the down lines pipes.

 

Regular valves

 

The main line may hold any number of valves which can either be the block valves or other, user defined regular valves.

Regular valves can be used to simulate a draw off node and/or be used to shut the water flow off to the downstream pipes. Controlling and setting the shifts in which these valves perform their duties are set by the valve grid editor.

 

Shift block editor

 

This dialog box allows you to easily assign the blocks to their respective shifts.

 

 

Input grid

 

This grid input box is used to assign blocks to shifts.

Each defined block occupies a single row. Each shift occupies a column.

 

The first column holds all the block numbers. The first row holds all the shift numbers.

The second column indicates the block demand.

 

If a block is to be activated or deactivated in a certain shift, double click on the grid at the intersection of the block row and the shift column. The '#' character is used to indicate that the block is active in a specific shift. Any one block may be active in up to 4 shifts.

 

Note:

Blocks may also be assigned to shifts in the Block page.

 

Shift #

 

When selecting a new shift number in the list box, the information box below will indicate the number of blocks operating in the shift and also the total demand of the shift.

 

Editor

 

Click on the Editor button to view a report of all the operational shifts, the blocks associated with each shift and the shift demand.

 

Clear

 

Removes all the shift settings.

 

The % option

 

 

If the % check box is checked the blocks will be filled with a solid colour reflecting the shift in which they operate.

The shift colours and solid fill intensity is set on the defaults menu.

 

The # option

 

If this check box is checked the shift numbers will be plotted inside each block on every replot.

 

REGULAR VALVES

 

The operation of a regular valve can be linked to any of the defined shifts. The valve can simultaneously act as a demand node and a shutoff node in either the same shift or in different shifts. It can also be set to act in only one of the two modes or can be set to be totally inactive.

 

Regular valves are normally used as shutoff nodes to control the water flow to sprinklers operating in positions other than regular blocks. This can be to a pipe line irrigating the trees between blocks or to the tap in the farmer's garden.

Valve grid editor

 

 

In the example, four regular valves are displayed. Valves 'a' and 'b' will act as demands with a value of 10m3/h and will be active in shift 3 and shift 4. The pressure at which the demand is drawn off is also entered. This value will be used in the calculations of the automatic pipe sizing so that the pipe diameters and pump pressure can be set to supply the demand amount at the desired pressure.

Valve 'a' is also used as a shutoff node and will only let water through during the operation of shift 4.

 

Valves 'c' and 'd' are used only as shutoff nodes and will be open during shift 5. For all the other shifts the flow through the valves will be closed.

 

Note:

The open shift for valve 'b' is set to zero. By setting the shift number to zero the valve will be open in all the shifts. Because the demand shift for valves 'c' and 'd' is also set to zero, they will draw water out of the system during all the shifts if a demand value is specified.

 

 

Single edit

: In this mode any of the entries in the grid editor may be changed.

 

Select range

: Click this option to globally apply changes to the selected range of valve. Select a range of cells by clicking on the first cell and while keeping the mouse button down, drag the cursor over the other cells. Click on the edit range button to apply the changes. Select what property must be changed and enter the new value.  These values will be added to all the selected valves. Remember to recalculate the hydraulics of the main line after changes have been made.
