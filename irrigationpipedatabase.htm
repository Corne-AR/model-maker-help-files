MENU : Irr - Pipes - Database

 

This is a database and editing system that allows you to enter and maintain all the pipe types used in the irrigation project.

 

Navigating the dialog

It is important to understand the layout of the dialog box to make full use of the functions.

The database dialog box consists of a single page defining the different pipe types.

 

 

Pipe Groups

 

               

On the left of the page you find the pipe index list. This table lists all the defined pipes. The active pipe type will be the selected one in the index list. Do not enter data directly into the index list. Data is entered and edited using the Data and Classes boxes to the right of the page.

 

Each pipe type can be drawn in its own defined colour and line type. On the pipe group, double click on the selected pipe. The pen colour definition box will be displayed. Set the correct attributes and close the box. The new pen colours will automatically be associated with the pipe type.

 

Pipe Data

 

         

 

Each pipe type is defined by the pipe data in the pipe classes.

Use the add, edit and delete buttons to manage the pipe groups.

Between the pipe data and classes boxes, the following buttons are found.

 

When the dialog box is entered and no changes have been made, the Delete button is visible. Pressing this button will remove the selected pipe from the data base.

Press the Add button to clear all the input boxes in preparation of a new pipe group. Start a new pipe group by entering the data in the PIPE DATA input boxes and then the diameters in the PIPE GROUP grid.

In the past, MM only had one Ch factor for the whole family of pipes. Since version 4.40 it has been changed so each diameter has it's own value. If your design was developed using a pre-version 4.40, then all the pipes in the design will have the same Ch value.

If you have now changed your database and gave each pipe diameter a diffferent Ch value, you will have to re-apply the database changes to your actual design.

This can be done by using the edit AREA function on the pipes page and set all the pipes to the new database values.

 

The KD factor

The KD value refer to the secondary pressure losses created by emitters inside the pipe and is only relevant when the pipe is used as drip lines.

In drip lines, the kd factor of the pipe is used when secondary losses are calculated.

If the kd value of the pipe is zero and the drip emitters have a kd value, then that value is used.

 

 

Follow this sequence to edit the pipes

1) To enter a new pipe, click the ADD button and start entering the relevant pipe data and pipe classes in the appropriate boxes. Press      the OK button to record the pipe to the index list.

2) To edit an existing pipe, select the pipe from the index list. Enter pipe data and pipe classes in the appropriate boxes. Press the OK     button to update the pipe information on the index list.

 

Note:

Before pipes can be placed between defined nodes, at least one pipe type must be defined in the data base.

 

Important:

When a pipe is selected from the pipe data base and used in a project, all the attributes of the pipe is copied to the project. Changes made to the pipe data base thereafter will not automatically update the information in the design project.

To update the attributes of the pipes in a design project, use the edit functions on the pipes page. The pipe data values (diameters, prices, roughness coefficients etc) will then be updated.
