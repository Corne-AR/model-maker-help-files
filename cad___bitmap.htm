MENU : Cad - Images                                

 

Model Maker reads and displays any regular image files. The images can either be loaded as a background image or as part of the CAD environment.

 

 

Explanation to ensure you understand how images work in Model Maker

 

When you load an image and save a kcd or mal file then the image file itself is not saved within the KCD.   The kcd will only record/save the file name (including full path from where this file was loaded) and the position where it was placed on the drawing.  Now if you exit MM and next time load the kcd again it will load the file from the location saved in the kcd file and place it at the saved position.

This means that if you give your KCD file to another person or just load it yourself from another computer you must also give the original image file to the other person or transfer it also to the other computer.  MM will try to go neatly out of the function if you load a KCD and the corresponding image file is not available but it seems in some cases if the file was originally on an external drive (like A:) then it can not exit from and ignore the proper loading of the image.

 

If you load a KCD that contained an image and the image is not available you can go to the CAD - IMAGES option and :

1. CLEAR all the images from the data and save the kcd file then again without the image - or

2. Use the option (on the dialog )  : FILE - RENAME CURRENT  and select an available image which may be the old image but at a new location on your hard drive.

 

To include the source of the image (not just it's location) into a saved file, use the MAL file format and ensure that the is checked on the MAL file settings.

 

 

 

Background / CAD

 

This selection sets the environment in which the bitmap will operate. Up to 16 images may be loaded simultaneously into the background mode. This mode will retain the bitmap pictures in their defined positions even if different CAD (kcd) files are loaded. To save the positions of the background bitmaps use the File option.

 

The CAD environment can also manage 16 images. The difference between this mode and the background mode is that the bitmap settings - the file names and top left and bottom right co-ordinates - are saved along with all the other CAD elements when the kcd file is saved. Clearing CAD or loading another kcd file will have an influence on the bitmaps allocated to the CAD mode.

Open button

 

Click on this button to load a bitmap from file. The loaded bitmap will be allocated to the

 

selected mode and to the active bitmap number

 

After the bitmap is loaded a rectangle will be displayed. Snap onto the grip points and resize the area if needed. The position of the bitmap can also be updated by using the Indicate 9 option.

 

Paste

 

 

Use the PASTE option to copy the bitmap to be displayed from the clipboard. This can either be a bitmap placed on the clipboard by another program or by Model Maker itself.

Use the UTILITY option to capture a bitmap.

 

Active bitmap

 

 

Select the plotting number of the bitmap. Up to 9 bitmap per mode (background or CAD) can be displayed. In each mode the bitmaps are displayed starting sequentially from number 1. A higher numbered bitmap will overwrite the picture of a lowered number bitmap if they overlap in any way.

 

Bitmap order

To change the plotting order of the bitmaps use the and to move the images around. For example, if the image at position 3 must be plotted first, it must be moved up to position one. To do that click on number 3 then click the button twice.

 

Clear

 

The image at the selected plotting position will be cleared. The image name will be removed from the plotting list and the following bitmaps moved up. This will not have any effect on the bmp or jpg file that holds the bitmap data.

 

Indicate 9

This function will place 9 grip points on the bitmap. By selecting a grip point you may stretch and move the bounding rectangle around. The center grip point will not change the rectangle size but allows you to move the whole rectangle to a new position.

Note : Any other point on the bitmap can behave like the center grip point. To do that, hold the shift key on the keyboard down when clicking on the bitmap.

 

Grips = CAD lines

 

If this box is checked, the grip lines will temporary behave as CAD lines. This allows you to use all the normal CAD line snapping functions when indication the grip lines.

 

The bitmap displayed on the graphics screen will be outputted along with the rest of the graphic elements when a drawing is produced on a plotter.

 

 

TFW reference file

 

If a reference file is available, Model Maker will also read that and position the image according to the reference coordinates. The reference file must have a TFW extension and reside in the same folder of the image itself.

 

Example of a JPG and TFW files in a folder.

 

 

The TFW normally looks like this.

0.750000000000000

0.000000000000000

0.000000000000000

-0.750000000000000

35009.000000000000000

-2826892.000000000000000

 

The first line being the pixel ratio and the last 2 lines are the coordinates of the top left corner.

 

Then this image is loaded into the program, the settings will automatically be set like this.

The Ratio option is selected, the pixel/unit ratio is set and the top left coordinates are filled in.

 

Ratio calculator

 

 

 

Click the ? to activate the calculator.

A) Enter the theoretical distance between 2 point on the image.

   This is the distance you want to have on the image if the correct ratio was applied.

B) Enter the current measured screen distance between the same 2 points.

         Use the F6 measure function to determine the distances.

C) The ratio will be calculated.

 

To copy the new ratio value to the ratio box, click the OK button.

 

Test the result by measuring distance (A) on the image again. This should now be the correct distance.

 

 

 

INDICATE 2

 

 

This function is used to position the image correctly by matching a visual feature on the image to an existing element in the cad.

 

This approach is often needed when the coordinates of the image corners are not known.

 

1.        Visually identify the feature on the image.

2.        Identify the cadline (or other cad entity) to which the image feature will be matched.

3.        Use the Indicate 2 option and in freehand mode, point to the Base 1 position.

4.        Select the appropriate snapmode (line end point in this example) and point to the line end point.

5.        Do the same for Base point number 2.

 

The image will be moved and stretched to match the entity.

 

This feature does not bend the image but only adjusts the top left and bottom right corners of the image to achieve the result.

 

Always inspect the result visually to verify the match is good for other areas of the image.
