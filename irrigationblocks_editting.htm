EDITING BLOCKS

 

 1 

Use the ? check boxes to specify if the block names, descriptions and numbers must be labeled in the block when plotted.

Block Info - Name & Description

 

Here you may enter the block name and description. These inputs are not important to the operation of the system as such but provides information to the user about the block itself.

The name and description will be printed when reports about the block is produced.

 

Add new

 

This adds a new block to the project and allows you to define the perimeter points. The normal area definition functions are used and up to 160 points can be defined. Set the pen attributes to the desired colour before the area is defined. After the new block is added, enter the new block name and description in the appropriate input boxes.

 

Delete

 

Delete the current block. Only blocks numbering 1 and up can be deleted as block 0 is the main line and is automatically part of the project. The delete function removes the block perimeter and removes the block definition from the block list. Use the delete pipes and delete nodes functions to remove the elements from the block before it is deleted. If a block with existing nodes and pipes is deleted, Irri Maker will after confirmation, remove the elements before deleting the block.

After a block is deleted, all the blocks in the block list following the deleted one will be re-numbered one down.

 

The delete blocks in an area function allows you to delete all the blocks that falls inside a defined a polygon area. A block is deemed to be inside the area if the centre co-ordinate of the block (the position where the block's number appears) is within the defined area.

 

Edit

 

The perimeter nodes of the block may be edited using the normal area definition options. Use this to change the shape of an existing block. Changing the shape of a block that already has nodes defined is allowed but not advisable as some nodes may fall outside the new area. Irri Maker will however warn you if that is the case. Nodes outside of the block area will not influence the system's functionality but you could at some stage run into trouble and find it difficult to keep track of those nodes.

 

Copy

 

The active block (excluding block 0) can be copied to a new position. The block perimeter and any defined nodes and pipes in the block will be copied. The standard rotation functions can be used to orientate the block before it is placed in a new position.

 

Move

 

The active block (excluding block 0) can be moved to a new position. The block perimeter and any defined nodes and pipes in the block will be moved. The standard rotation functions can be used to orientate the block before it is placed in a new position.

 

Colour

 

Use this option to change the colour of the perimeter lines. This is often necessary if the pen attributes were not set correctly when the block was initially defined.

 

Fill

 

Allows you to copy the lateral layout of one block and fill it in another.  Take note that the block to be filled can not already have any nodes or pipes. The fill can only be applied to an empty block with only the boundary defined.

This function will adjust the layout of the laterals after the block area has changed.

Note:

 

Any manifold pipes in the original block will be retained if their nodes have not changed. If some of the manifold pipes were deleted you will have to manually add them again in the new positions.

 

 

The alternative to the adjust function would be to delete all the nodes and pipes in the block and do a Fill Laterals from scratch. The adjust functions does that for you.
