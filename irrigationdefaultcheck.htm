Using the MAL file format to transfer irrigation databases.

When saving data in a MAL file you have the option to include the supporting irrigation databases. These are the databases in the folders as described in the previous paragraph and which are edited in the following dialog boxes.Nodes, Pipes, Parts & clusters.

 

If these settings in the MAL file setup are selected,

the nodes database (IrrNode.dbf),

the pipes database  (IrrPipe.dbf),

the parts database   (Parts.dbf) and

the clusters             (Clusters.dbf) are added to the contents of the MAL file.

 

From version 11.02 all the elements are saved in a single DB file.

 

Loading a MAL file that contains irrigation database files.

When you read a MAL file with irrigation database files a certain method is used to make that data available to the program.

 

Normally, your own irrigation database files are stored in the folder called C:\Program files\MMsystems\MM\IRRIGATION. When Model Maker extracts the irrigation database files from the MAL file it is placed in the folder called C:\Program files\MMsystems\MM\IRRIGATION_MAL.

After the DBF file are placed in the IRRIGATION_MAL the user is then given the option to make use of that data and if selected the program will change your default folder names to the new directory.

If you do not select to use the DBF files from the imported MAL file your settings are not changed.

 

When your default irrigation folders are already set to the IRRIGATION_MAL directory the program is going to warn you that it cannot read the DBF files that is part of the MAL file because it will clash with the ones you are currently using in the IRRIGATION_MAL directory. You must then first use the option and select a different folder name (other than C:\Program files\MMsystems\MM\IRRIGATION_MAL) and then read the MAL file again.

 

Parts, Clusters and Template data base

The second folder name is that of the directory where all the parts, clusters and template DBF files are kept. By default the folder name for the nodes and pipes and the parts and clusters data bases is called . By changing the parts and cluster data base folder name you can work with your own nodes and pipes data bases but make use of different parts and cluster data bases.

As an example consider an office where 2 or more users are working with Irri Maker and that one works with pivots and the other with solid set installations. When the one is working he would like to access a different data base as the other and can easily do that by creating a new folder and let Irri Maker select the parts and clusters from that location.

 

To change the folder names click on the appropriate Update button.

 

Keep in mind that Irri Maker will use the folder name last selected when booting up and it is thus important to check the active folder name on the projects screen when starting a design.

 

A typical example of the files in the irrigation folder.

The MDX files are index files that support the DBF files.

 

The DRP files are supporting files for the IrrDripper.dbf

 

You may edit the DBF files by using Xcel or any other data base file editor. The supporting file are temporary files created by Irri Maker and should not be edited.

 

 

Profile Check

This function will inspect all the pipes of the active block and indicate to you the positions where the pipes are more than a certain distance above or below the surveyed ground model.

Pipes are added to the irrigation design by hooking the two end points onto two existing nodes. Depending on the length of the pipe and how undulated the terrain is, the situation can arise where the 2 terminals of the pipe is on the ground but that somewhere along the pipe the ground can be either over or below the pipe.

 

When a drip line block is developed and Model Maker does the layout of the laterals, a check is done to see if any additional nodes are needed to follow the profile.

If the pipe is really going to be constructed like this, excavated through the hills and a structure build to support it over the valley, then you do not have to inspect and modify it.

If the intention is to have the pipe follow the ground surface more accurately, use this Profile Check function and Irri Maker will alert you to areas of concern.

Irri Maker will place white diamond marks at all the positions where the difference between the pipe and ground profile exceeds the specified maximum value.

Note:

The Check function only draws your attention to problem areas. You must then determine if you want to rectify the problem and which method you want to use.
