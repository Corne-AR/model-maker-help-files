Optimize-Solar

 Top  Previous  Next

This function will take a platform with 4 corner points and change it's elevation and slope so that the pole lengths (represented as nodes on strings) fall within set lower and upper limits.

 

Please note:  In addition to the Basic modules (1 & 2) you also need the Platform module (4), Cad module (6), GXT strings module (11) and the Optimize Terrace module (14) to run this option.

Modules needed. : Mod 1,2,4,6,11,14.  (6 modules)

To access the function go to (teR) Cut & Fill > Optimize > optimize_Solar or use the shortcut key on the left.

The following items are needed to run the process.

1) The ground model on which the poles will be placed.

2) The platform model having 4 corner points and 2 triangles between them, or select the virtual platform

3) General strings that represent the solar lines. The nodes represent the position where poles will be planted.

 We suggest you draw these strings with a pen number higher than 20, the function use pen 1-20

 

 

 

This is a typical image of a solar field on a platform with 4 corner points.

The yellow dots represent poles that are shorter than the spec.

Red dots are poles longer than the specs.

Green dots are poles within the specs.

The single blue dot is at the average position of all the long poles.

The single white dot is at the average coordinate of all the short poles.

The red Circle is the Highest Pole.

The Yellow Circle is the Lowest Pole.

The purpose is to have all the poles to be green, within the specs.

The action would thus be to lower the platform on the side of red poles and raise it on the yellow side.

 

 

 

 

 

 

 

 

       

 

 

 

Setup:

1) The pole lines are represented by GXT strings. The strings must be in a straight line running north south or east west. The number of identified strings display here.

2) The strings related to this design must be identified. Use the Id Strings button to open the DoId function to id strings as needed.

3) Use the “Id by Line” Function to do the same as point 2 with these added benefits. Functions you would normally select in sequence.

·Change snap mode to Free
·ID New
·Id by Line
·Zoom to Id Strings
·Open Project List popup

All you need to do is draw the line through the applicable strings.

4) While the strings are identified you may zoom to them point 3 above does the same.

5) A user defined prefix can be added to to all the nodes. Helpful to name the nodes of this design to something else.

6) You need to specify the ogl project that needs to cover the area below the strings with triangulation done between the points.

 

 

7) Design dtm project numbers. The Ogl will normally always be project 1 but the design platform will change from one field to another. If the background is highlighted red, that means the virtual option is active.

A red warning will show if the design project does not have exactly 4 points.

Another red warning will show if the 4 points are not in a perfect square east west and north south.

Ensure that the platform (4 point & 2 triangles) is big enough so all the nodes will be on a triangle. Remember, changes are made to the 4 corner points, the elevations of the string nodes are then interpolated onto the platform and then compared to the ogl surface to calculate the pole lengths.

 

 

8) Project List displays all the Projects in the Solar Project selector.

A.The OK will be next to the project that covers the area of the Identified strings. Only projects with an Ok can be selected for the function to work. The selected project becomes the Design Platform Project.
B.“Use this project as virtual platform" allows you to highlight/select any project to become the virtual calculation project.

The function will automatically add 4 points below the area of the Identified strings and you do not need to create platforms beforehand. Each time you ID new strings it will create 4 new points and Triangles.

 

C.“After running the Excel report also create dtm points” allows you to convert all the nodes on the Identified strings to DTM points.

Input the project to add the points to.

The Z value will be the Pole Height and the Code will be empty if correct and Low and High if out of tolerance.

Turn your Codes and elevation Display on, to see where you already added points to the selected project.

D.“After running the Excel report also change color of the ID strings” changes the color in sequence from pen 1 to 20 and then repeats the sequence. Because of this best practice would be to draw the original strings in a pen number other than pen 1-20.

Also make sure the Cad display is below the strings display or turn the Cad display off.

 

9) You may enter the maximum slopes in the 2 directions. MM does not use that for any limits, it is just for the user's own reference. Below the Current% column, the actual platform slopes are displayed. This will change actively as the platform is tilted for a better fit. Once the best fit is achieved, the user should check that the current % is less than the specified maximums. The value will change to red if it is outside the limit.

Note: The program does not use the maximums as limits.

10) The user may add or subtract elevations from the 4 corner points by using the options in (11). The amount of the elevations change is specified in (10).

11) Elevations of the 4 sides or all 4 points in total can be changed up or down by these 2 buttons. Select which points (sides) must be affected, set the value in (10) and press up or down. The 4 platform points will be adjusted, and the strings re-draped onto the new platform elevations. The string nodes will then be compared to the ogl and assigned the yellow, green or red colors.

Results:

12) In plan, the string nodes will have one of these 3 colors. The aim is to make them all green, within low and high limits. Poles less than the low limit are yellow and those above are red.

13) Once changes are made to any of the settings, the report is run and the system analyzed. The report has 6 lines containing all the info.

The last line, total pole lengths, is the accumulative length of all the poles, with an average of the pole Hight next to it. (Total Pole Length/#off Poles)

The best design would be where all the poles are green and the total pole length is a minimum.

 

14) “Create CSV file, dtm and color” will execute the selection of point 8 B,C and D. The report will also allow you to save a .csv to open in Excel  a.txt file or to copy the contents to clipboard.

 

The Report includes all the relevant information including:

·The Node Names and Coordinates
·The Ground Level , Top of Pole, Height Above and Depth.
·The Position corners with its Direction like SE (South East)
·The Status of each pole Low = out side of tolerance Low*= Lowest pole in selection. High= outside of tolerance and High*= Highest point in selection.

15) The auto function will internally run through different platform slopes and elevation to try and optimize the design. The auto function will stop when it reaches the maximum number of green points with the lowest elevation. This feature is very handy to get close to the best slope and elevation.

The user can then still try to improve on the results by manually changing the platform slope or height by using options (10) and (11).

16) This information is linked to the report and used to determine if the results are acceptable before proceeding.

17) The Legend displays the legend click on it again to close.

18) The String display to help you choose your display options to set your view the way you want it.
