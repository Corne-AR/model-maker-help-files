Required Hardware:

Suitable MS Windows tablet or laptop with at least 8GB RAM (16GB recommended) and i5 or i7 quad core processor. Wi-Fi Module. SSD Hard Drive.

Dual Antenna RTK Receiver (Rover) with ability to transmit NMEA string data via TCP Protocol via a predefined TCP CASTER and communications port via a Wi-Fi hotspot provided by the receiver.

What Model Maker require from the user:

1)The IP address and port number of the TCP Caster as provided by the rover (GNSS rover receiver)
2)A NMEA GGA string at of 2Hz - 5Hz for a single GNSS receiver antenna position. There are 2 antenna position NMEA strings required. We recommend that both receiver positions should transmit the NMEA strings at the same update rate.
3)A correction signal from a GNSS base receiver or NTRIP caster that is already calibrated and localized should be transmitted to the rover receiver on the machine. You should therefore set up a known point base that transmit a known base co-ordinate.

**Please note that Model Maker will not do any site calibration or job localisation changes to any co-ordinate system apart from translating the geographical co-ordinates as contained in the NMEA string to local co-ordinates. Model Maker will use the predefined co-ordinate system as set up in: Settings > Configuration menu.

4)You will need DTM points in one of the DTM projects for the To Point function
5)For the To String function, you will need to create GXT strings for the GPS to follow
6)The To Surface requires a DESIGN surface of some sort in any DTM project comprising of points and triangles.
7)A CAD Pattern located in the following folder on your PC must represent the vehicle of choice.

C:\Users\YOURNAME\.MMSystesms\MM\PatData

Settings and configuration of the machine guidance module:
Enter the IP address and port number of your GNSS

Model Maker will connect to an IP port that is supplied by the GNSS receiver via LAN cable or Wi-Fi. MM will listen for a NMEA GGA string (the primary GNSS receiver antenna – ANT1) and a second NMEA “ANT2” string (the secondary GNSS receiver antenna – ANT2) that will give position and attitude data to Model Maker.

Model Maker uses the positional movement of the GNSS to determine when the graphics on the main screen will update or replot. Adjust this setting to a bigger value if you want the screen to refresh less often. A reasonable distance is anything between 0.3m to 1.0m depending on the processing speed of your PC, RAM and graphic memory available on the PC. A value less than your GNSS equipment accuracy will be a bad value to use. A typical test would be to use a value that when the machine is stationary, the main screen does not update or replot.

The primary GNSS receiver (ANT1)  is the RECEIVER that provide the main GGA NMEA string. Model Maker will place the pattern origin at this position. If the pattern origin is therefore not in the desired position on screen, you can use the GPS offset from machine centre values to change the virtual pattern origin to any position the user require. The offset values must be entered in meter values.

You have the option to use a single GNSS receiver or two receivers. Clicking the GPS Format button will toggle between One or Two receivers. Depending on the machine setup, you can choose to have ANT2 in front and ANT1 at the back (viewed from the main direction you are moving in) or vice versa. We recommend the primary ANT1 to be in the back for the To Point function.

Plan orientation can be North Up which mean the vehicle will rotate and the main graphics will be Nort Up always. Machine up will rotate the main graphics, but the vehicle will remain stationary pointing north.

The “Copy points to DTM project 10 at interval” option will save the GNSS position (after adjustments) to Project 10 at the interval you specify. A value of 0 (zero) will disable this function. When an interval is entered, this will create an as-built survey while the machine is moving at the interval specified. DTM points will only be recorded while the vehicle is moving in a forward direction because some machines, like dozers or graders, will raise the blade while reversing and this may produce false heights.

With a value entered in the ”Remove previous point within radius” box will prompt MM to actively search for previously placed DTM points and remove them. This happen so that the as-built survey continually updated with points at the new as-built elevation or position. The function is intelligent and will not remove points directly after placing them. The machine must move away from a point and then revisit the point proximity a little while later before an existing point will be removed. This means you can use the same distance for placing points as for the remove radius.

About pattern dimensions

The pattern you provide must be drawn at a scale of 1:1000. Model Maker will use the origin of the pattern as the centre position of ANT1:

The CAD pattern must be scaled at 1:1000 in Model Maker where 1m in Model Maker will represent 1mm on paper when printed. Lets assume a bulldozer will have a blade width of say 3.5m and should therefore match exactly 3.5m in Model Maker when you measure the width of the blade using the F6 button.

 

TO POINT

 

 

You can set the main position of your machine focus area in relation to ANT1 position bearing in mind that your ANT1 may be in front or at the back. This will influence the Left, Right, Front, Back values. Left and back is always a negative value.

The default position of ANT1 is back and ANT2 in front and we recoomend a minimum distance between these antennas of at least 1.5m.

This option is a proximity tolerance value. As soon as your machine focus point reach the targeted position within this tolerance value, a green tick button will appear on the right-hand side of the screen:

 

When the user presses this button, the software will draw a circle on the point you just placed and the next point to navigate can be selected on the main interface. If two points are selected (one after the other is placed) the program will automatically locate the next point on the same direction as the previous two points and manual selection of points will not be needed any longer. Two points must be selected before a new direction will be automated.

TO SURFACE

The Design Tolerance value for the To Surface function is used to determine the height accuracy that is required. Some RTK systems can only achieve heights accurate to 4cm so please be sure to choose an accuracy tolerance within the ability of your GNSS system.

The Ground to gps value is the target height of your GNSS ANT1 receiver. Measured the mounting height vertically from the bottom of the bulldozer blade tip to the bottom attachment the ANT1 receiver that should already be mounted securely to the blade of the machine.

Tick the “Using ellipsoidal heights” if the GNSS system should use GPS/ellipsoidal heights instead of orthometric/mean sea level heights. It may be that your design surface was designed based on ellipsoidal heights but your GNSS system by default applies a GEOIDAL correction to the heights which in certain parts of the world will give a height separation that is quite substantial. In South Africa the difference can be in the region of +-30m difference between the ellipsoidal and geoidal

heights.
