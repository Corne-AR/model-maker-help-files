MENU : File - Import ASCII & Other - Define format

 

 

Import ASCII Data allows you to load co-ordinate data from files that have a different format to that of the standard Model Maker yxz format.

 

These files usually have a different column sequence or has a file extension other than the normal YXZ.

The options in this dialog box will be used to specify the specific order of the co-ordinates in the file that you load. It is important for Model Maker to know which of the characters in the file must be loaded as co-ordinates, the elevations and the code or name of each point.

These type of files normally consist of data lines, each defining the co-ordinates of an individual survey point.

The functions in this dialog box are also used when geographical co-ordinates are imported in the electronic menu.

 

Examples of an ASCII files.

 

Example 1

         0           00001       -3385834.053 -156131.245     1214.950

         0           00002       -3385832.145 -156127.499     1215.160

         0           00003       -3385857.061 -156125.689     1215.050

In this example the columns in the file are separated by one or more spaces.

 

Example 2

         1 , 2000.000 , 2000.000 , 200.000 , B

         2 , 2010.000 , 2000.000 , 200.000 , D

         3 , 2002.405 , 1994.931 , 200.466 , WAL

         4 , 1995.640 , 2013.106 , 196.676 , LAND

Commas are used to separate the columns in this example.

 

This import ASCII dialog box consists of three main panels.

The defaults panel is used to specify the column delimiter and decimal notation. Use the options on the read format panel to specify the column format and use the display part of the dialog box to view the first few lines of the ASCII file.

 

 

Defaults

 

 

Column delimiter

Click on the character used by the data that you load to separate the columns. The comma, space, and tab characters are standard formats. Use the OTHER option if the data is separated by any other type of character.

 

Decimal notation

Click on the point or comma options. If the decimal position in the data is specified by a different character use the OTHER option and enter the character in the edit box.

This line uses a point as decimal notation.

PI48                                52813.018 3388983.114      168.609

 

This line uses the comma as decimal notation.

CL245                                52791,283 3388962,406      169,139

 

Defaults

The current combination of defaults and read formats can be saved and later recalled.

To save the defaults, enter a name in the list box and click the Save button.

To load a default combination, select the name in the list box and click the open button.

 

Original

Click this button to view the effects of the settings on data of the file.

The first 11 lines of the file are loaded and displayed on the screen.

If the button caption is 'ORIGINAL,' Model Maker will display the 11 lines in their original form.

If the button caption is 'PREVIEW,' Model Maker will display the 11 lines in the adjusted form. This means that the column delimiter will be replaced by a space character and a point will replace any other decimal notation. The preview option is a handy function to check the adjusted format of the file and to see if Model Maker interprets the format correctly.

 

An example of the most common format of an ASCII coordinate file.

         0           00001       -3385834.053 -156131.245     1214.950

The data in this file uses spaces as column separators and a point as the decimal notation. The preview option will have no effect on a file like this as the format is already in the required standard.

 

In this example commas are used to separate the columns.

         1 , 2000.000 , 2000.000 , 200.000 , B

In the preview format this line will be displayed as:

         1   2000.000   2000.000   200.000   B

 

In this example commas are used to separate the columns and as the decimal notation.

         4 , 1995,640 , 2013,106 , 196,676 , LAND

A file like this can be very confusing and should be read with read format 1, discussed below.

 

 

Read format

Two different options can be used to specify the co- ordinate positions in the ASCII file. Select one of the two formats by clicking the Format button. Depending on the selected format, enter the required column positions or co-ordinate sequence.

 

FORMAT 1 - Columns

If this format is selected an editbox will be visible on top of the listbox that displays the first few lines of the ASCII file. Use this editbox to specify the column positions by spacing the characters in the appropriate columns.

Place 'C' characters over the characters in the columns that are used as the CODE.

Place 'T' characters over the characters in the columns that are used as the TAG.

 

When working in the southern hemisphere use 'Y' characters to indicate the Y co-ordinate and 'X' characters to indicate the X co-ordinate.

When working in the northern hemisphere use 'E' characters to indicate the Easting co-ordinate and 'N' characters to indicate the Northing co-ordinate.

When loading Geographical co-ordinates use 'A' characters to indicate the lAtitude and 'O' characters to indicate the longitude value.

 

Use 'Z' characters to indicate the elevation.

 

 

The example illustrates how to place the characters in the edit box.

The ccccc characters indicate the code of the point.

The tttttt characters indicate the tag of the point.

Place enough yyyyy characters over the Y co-ordinates. Do the same for the X and Z co-ordinates.

 

Note:

This format of reading co-ordinates can only be used if the columns in the ASCII file maintain their relative positions on all the lines.

 

The blue block indicates the extend of the Y characters and the red block the extend of the X characters. The black circle points to 2 text lines that do not have the same column positions as the other text lines and will be translated incorrectly or skipped.

For files that have co-ordinates in irregular column positions use the sequence format to interpret the data columns.

 

 

FORMAT 2 - Sequence

With this format more emphasis is placed on the relative position of the column within the text line rather than the specific position of each character.

Use the edit box and specify the order of the columns. The specified combination of 'C', 'T', 'Y', 'X' and 'Z' characters in the input field is used to determine the column positions.

Note that, depending on the system settings, 'Y' is used to identify the Y co-ordinate, 'E' for the Eastings and 'A' for the lAtitude value. Use 'X' to identify the X co-ordinate, 'N' the Northing and 'O' the longitude value.

 

Check either the Left or Right selection which indicates that the CTYXZ column indicator should be implemented starting from the front (Left) of the text line or from the end (Right) of the text line.

 

 

EXAMPLES:

 

If the Left option is used the CTYXZ combination indicates to Model Maker that the code is the first column, the tag is the second column, then follows the Y, X and Z co-ordinates.

If the Right option is used the CTYXZ combination indicates that the Z column is first when the line is scanned from the end, the X is second and so forth.

Trouble Shooting

 

After importing an ASCII file, use the Normal plotting function to display the points on the screen. If the points are not correctly displayed on the screen, check the Status and note the number of points in the system.

If the number of points in the system look correct use the Data base, Points menu and inspect the co-ordinates of the converted points.

 

The first few lines of imported ASCII files often consist of header information. Numbers in header lines often get converted to DTM co-ordinates which is then not related to the rest of the co-ordinate list. Locate these points on the Points menu and use the Delete button to eliminate them.

 

If no points were converted and loaded into Model Maker, inspect your logic when defining the read format.

Is the decimal notation correctly specified?

Is the column delimiter correctly specified when using Format 1?

Is the CYXZ sequence and the Left or Right option correct when using Format 2?

 

Model Maker only displays the first few lines in the preview panel. Check that the text format that you see on the first few lines are really maintained throughout the file. Columns may be missing on other lines further down in the ASCII file. Use Notepad or Wordpad to load and inspect the ASCII file.

 

Sign change

Check the relevant check box if the sign of the Y or X co-ordinate must be changed.

In this example the sign of the Y co-ordinate will be changed.

If the Y co-ordinate in the ASCII file is 123.225, it will be recorded as  -123.225.

If the Y co-ordinate in the ASCII file is -500.225, it will be recorded as 500.225.

 

Add constant

Use this input and check boxes to automatically add a constant value to the imported co-ordinates.

 

In this example 2 800 000 will be added to the X co-ordinate.

If the X co-ordinate in the ASCII file is 500.225, it will be recorded as 2 800 500.225.
