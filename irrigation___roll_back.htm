IRRIGATION - Roll back

 Top  Previous  Next

The feature to roll back the irrigation project to previous stages is described here.

 

 

The button to activate the roll back (undo) features is located at the top of the main irrigation dialog box.

 

 

When the Roll back button is pressed, the following dialog box is shown. (this is just an example the actual contents will look different)

 

The time at which the stages were recorded is shown along with the number of seconds since it was last saved.

Each stage is recorded before a new function is executed. A description of what the action was that initiated the recording of the stage, is also shown.

The number of the active block (when the action occurred) is also shown.

 

How it works:

 

A copy of the current irrigation project is saved in the roll back list when a function is pressed on the main irrigation dialog box. The roll back entry records the time when it was done and also a description of the function that made the entry.

Each time a function is pressed, a new roll back is saved. A maximum of 10 roll backs will be saved with the newest action at stage 1 and the oldest at stage 10. When all 10 stages are occupied, the program will drop the tenth stage (cannot be retrieved again), stages 9 to 1 move down one and the latest roll back is placed on the top line.

 

This is not a full fledged UNDO system for the following reasons:

·A roll back entry is created at the stage when the function is selected. If for instance the function is to add pipes and 21 pipes are added and afterwards a roll back is done then all 21 new pipes will be removed.
·A regular UNDO would have been able to undo each individual action after a function is started. The roll back reverts back to what the system was before the function started. It does not check for each individual action within the function.
·A regular UNDO/REDO would undo the actions in a chronological order. The roll back shows you all the roll back entries and you can select to which one you want to revert to.
·The roll back system operates more like a backup system rather than a true undo/redo option.

 

Undo roll back:

 

When the roll back dialog box is opened and the project is rolled back to another stage, MM first makes an image of the current project and this button then becomes available.

This allows you to select any of the roll back stages and in the end, if you do not want to commit to any of them, just press Undo roll back and the system will be restored to what it was when the Roll back button was pressed on the main irrigation dialog box.

 

Note: The roll back images are all kept in memory and all the stages are lost when you exit MM or a crash occurs. The roll back functionality is intended to undo unintended actions not to replace the benefit of the general MM backup system.
