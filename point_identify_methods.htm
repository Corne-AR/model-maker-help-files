Choose individual points that cannot otherwise be grouped together by some common attribute. Using the mouse to select the point you want to identify or mark. The point closest to the cursor, when clicked, will be selected and will then turn red, indicating it has been identified.

Your cursor snap mode is irrelevant at this stage and cannot be changed. It will appear like . You can only snap to a tachy point if you are identifying points individually. Any attempt to change the mode will fail. Watch the prompts at the bottom of the screen. They are there to instruct you.

 

 

 

Identify points by using their code. Typically, you could identify all the points with a code of F (Fence).  The code menu explains the action in more detail.

 

 

Identify points by using their tag. You could identify all the points with a tag of SEWER.  The code menu explains the action in more detail.

 

 

 

All the points are stored in a database and each is allocated a number. The points number represents its position in the database. For example, the first 20 points collected would have numbers 1-20. Use this option to identify a group of sequential points. For example, if you combined project #2 with project #1 and then discovered you made a mistake, this option would help you correct the problem. Using status you would be able to determine how many points were in project #2, for example, 100. If project #1 now had 500 points in it, you would know that points 400-500 had come from project #2. You could make project #1 active, and identifying by number, points 400 -500, the points could be deleted.

 

 

Identify all points within a specified elevation range. For example, you could identify all the points within the elevation ( Z  ) range  151 to 253 and delete all the triangles from these points or add a constant to the elevation or Z co-ordinate.

 

 

Identify all the points with triangles connected to them. Model Maker will scan through all the triangles and the points forming these triangles will be identified. Once again, this applies only to the triangles in the active project.

 

 

Identify all the points not connected to triangles. This function is often used to eliminate old points in a project that are no longer relevant to the model. A good example is when 2 projects are combined and the triangles are brought down and overlap onto the existing model. Overlapping triangles will be deleted, meaning that we now have points not connected to triangles. This is an easy way to get rid of redundant points.

 

 

Identify the points of all the triangles on the perimeter (side) of the model. View the side of the model under the triangle menu.

 

 

Identify points within a project that are not linked by one continuous triangulated model. A good example of this is a survey of many individual mine dumps all within the same area. Correct triangulation will result in islands of triangles being formed.

 

EXAMPLE of triangles forming groups.

 

Calculating the dump volume of this model will result in a combined answer of all the mine dumps. The dumps must be isolated one at a time and copied into another project before Model Maker would be able to determine the volume of each. The group function allows you to point to an individual set of triangles (eg., a single dump) and all the points connected to that island of triangles will be identified. This saves you from having to identify the points by area and then defining the area. The same can now be achieved in one keystroke.

 

 

  Hot Key selection is A

This option gives you the freedom to define any regular or irregular shaped area and identify the points inside or outside the figure. The area as defined by the triangles of a specified project may also be used.

 

Triangle area

Identify all points to the outside of the triangle. Select the project to use when determining the area covered by triangles. Use this option when 2 projects are to be merged and you want to manually delete the common points below the design project.

Example : Project 1 is the ground and Project 2 the design. The design must be copied to the ground.

Activate project 1 and Identify by triangle area. Select project 2 as the one to determine the area from.

Result: All the ground points directly below the design data will be identified. You may now delete the identified points resulting in a gap being formed in the ground data ready to accommodate the design.

 

Define area

Defines a window or polygon inside, or outside of which all points must be identified.

 

 

Enter a grid interval.

This interval must be in meters and will be used to identify all the points closest to every imaginary grid intersection. This function will identify a fairly even distribution of points within the project. A good example of when it can be used is during final plotting. When the project is plotted along with the codes and/or elevations, of only the identified points found using the GRID option, the result is often a neat display of points. It also helps eliminate over plotting and crowded looking drawings

 

 

Identify all the points falling on the endpoints of a continuous CAD line. Indicate close to the start of the CAD line you wish to follow. Model Maker will follow the sequence of lines and identify any DTM point with the same Y&X co-ordinate. Use this option to extract all the points attached to the edge of a road that are defined by a continuous CAD line. You may wish to rename or save these points.

 

 

Identify points at nodes of indicated string.

 

 

In the process of manipulating a DTM project, duplicated points may be generated. This is not a serious problem, but should be rectified as soon as possible. Before triangles are generated, Model Maker searches for duplicate points. They can cause problems with triangles overlapping and models getting corrupted. Should any be found, use this option to identify and remove them. You will be presented with another dialogue box where you may specify whether the first or second of the duplicated points must be identified. A third option allows you to see which points are duplicated and select the one you wish to identify.

 

 

Identify points either above or below the triangle surface of a selected project. This is useful for comparing different survey data of the same terrain.

 

EXAMPLE: Load the points and triangles of the first survey in project 1. Load the points of the second survey in project 2 and make project 2 the active project. Now identify the cut points (points below the surface) of project 2 against the surface of project 1. The identified points represent the area where the surface as depicted by project 2 are lower than project 1.
