CAD – Hatch - Report

The hatch report will calculate all the areas (Hectares) of hatch elements on the same CAD Layer. A proportional report for all the hatch areas will also be given. This is to determine how much of the total area is used by each layer of hatching. The CAD layer names will be used as the description for each hatch zone. If CAD text (for example property or erf numbers) are present within the hatched areas, the resulting table will give the number of text elements contained within the boundaries of the hatched areas specific to each zone.

 

The requirements for the report are:

1)CAD HATCH elements on uniquely described CAD Layers. Only hatch areas that are that are ID’d will form part of the report.
2)CAD text that are ID’d. The ID text is optional, but must be within the bounds of the hatched areas. If there are no text identified, then you should select the “Hide text column if zero” option to hide the text count column to produce this result:

 

The text shift value is normally the same size in mm as your Text size as set in your text parameters. The text size in mm is the printed size on paper after printing the plan to your printer.

It is useful to know that each individual erf need not be hatched, but rather that the larger areas can be hatched and that it is the Id text that gives the counted number of erven. Be sure to eliminate any duplicated hatch and text values before you run this report to avoid possible errors.
