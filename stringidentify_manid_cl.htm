STRING : Id - Manid - Clash

 Top  Previous  Next

 

MENU : gXt - Id - Man-id - claSh

 

This function checks the intersections of all the identified strings with each other and reports any clashes if the one string does not clear the other.

The elevations of the strings are assumed to be the invert levels if the string models a pipe. The diameter of the string must be specified in the string name in millimeters. Eg. ST10_200, W1_250.

 

The ST10 or W1 part of the name is for your own information. eg. ST for storm water and W for water. The 10 in ST10 could be the pipe number.

Model Maker assumes that any value after the underscore '_' (eg. 200 and 250) specifies the diameter of the pipe in millimeters. Clashing takes the diameter of the pipe into account.

 

The string name can be 10 characters long. The clash report can be outputted to an ASCII file (.ZZ2) or may be displayed on the screen.

Specify a clearance height. For example, 100 means that the clash will be reported for any 2 string going over each other with less than 100mm of clearance. A negative value will report all the crossings of strings irrespective of the clearance.
