MENU : Pnt - Id - Man_id

 

All these functions may be applied to the points previously identified.

 

·Move

 

This option moves all the identified points to a new position. The Y & X co-ordinates of the identified points will be changed.
·Copy

 

Copy all the identified points to a new position
·Rotate
·

Rotates all the identified points around a fixed point.
·Match

 

All the identified DTM points of the active project can be stretched between two co-ordinates

 

·Interpolate

 

The elevations if the identified points will be interpolated to the surface of a selected dtm project.

·Report

A report listing the minimum and maximum values of the dtm points will be shown in the editor. If the checkbox next to the report button is checked a new dtm points will be added to the project. This new point will be placed at the center of the identified points and the Z value is equal to the average elevations of all the identified points.

 

·Incorporate

 

This function will inspect each identified point and if not connected to triangles but on top of a triangle, the triangle below will divide itself into 3 new triangles and incorporate the ID point into the contour plan.

 

·Constant

 

You will be taken to the database functions where coordinate constants can be added to the identified points.
·Code

 

The codes of the identified points can be changed in various ways.
·Tag

 

The tags of the identified points can be changed in various ways.
·Elevation

 

The elevations of the identified points can be changed.
·Morph

This function will stretch the elevations of the identified points.

a) The lowest elevation of the points will be shown. You can either except the value or enter a different value.

b) You will be asked to indicate the second base point. You can keep the elevation of the second point the same or enter a new Z value.

The procedure works as follows:

1) The original height difference between points a and b is calculated.

2) The new height difference between a and b is calculated.

3) The stretch difference (ratio) between the 2 sets of values are then applied to the elevations of all the identified points.

 

·Invert ID

 

All the identified points will be un-identified and the others will be identified.
·Copy to project

The identified points will be copied to another project.

 

·Cad

 

The points will be copied to cad as lines and text elements.
·Join

Joins (Inverse) between an indicated point (base) and all the identified points will be calculated.

You will be prompted for the base co-ordinate. The results of the calculations will be printed to the editor.

 
·Update Code

The codes of the identified points will be changed to the text of selected cad text elements.

1) Indicate : Here you will point to the dtm point and then the desired cad text element. The indicated dtm point will change it's code to the cad text value. This is not a truly ID option since the user manually selects both the dtm point and cad text.

2) Identified : All the id points will be inspected and for each, if a cad text entity shares the same base coordinate, the code of the point will change to the cad text value.

 

 

·Update Z

This is similar to the previous (update code) function but the cad text will effect the Z value of the dtm point. This function is only useful if the cad text entity can be converted to an actual numeric number.

 

·Delete

 

The points will be removed from the project.
·Save

Save the co-ordinates of the identified points to disk using the normal File dialog box and the YXZ co-ordinate format.

Remember that only the identified points are saved, the whole project should still be saved in the normal format.

See also: Total

 
·ediTor

 

The coordinates of the points will be listed in the editor.
·Sort

 

This function can work in 2 ways :

1) Manually : Will be available if all the points were manually (individually) identified. The position of the points in the dtm database will changed to the order in which the points were selected.

 

2) By GXt strings.  If there are GXt strings identified, the points will check if they are within 100mm from the string line and if true, they will be sorted according to their position on the string chainage.  Use this option when you want to reorder the points to follow the strings so when saving the points for setting out purposes, they will be in order, one after the other according to chainage distances.
