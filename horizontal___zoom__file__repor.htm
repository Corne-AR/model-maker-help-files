HORIZONTAL : Zoom, File, Report

MENU : Hor - Zoom_ch

 

Chain

Calculate the chainage on the horizontal alignment at a specified co-ordinate. Pick any screen position. Model Maker calculates the shortest distance between the indicated point and the center of the alignment. The chainage at that intersection point will be reported.

 

Zoom chainage

Zoom into the alignment at a position defined by the chainage value that you entered. There are many instances where you have a long alignment (10km+) loaded and you want to see it at a certain chainage on an enlarged scale.

Rather than zooming in and tracking the alignment with the shift function until the chainage comes up, use the zoom chainage function and Model Maker will automatically define a zoom window.

 

Chain CAD

Calculate co-ordinates all along the alignment and produce a list that can be added to CAD in table format as regular text elements.

 

Enter the start & end chainage values. The chainage between these limits will be calculated.

Use the INDICATE button if you don't know the exact chainages and you want to graphically indicate the positions.

 

Offset : A zero value calculates the co-ordinates exactly on the center line. A negative value will offset it to the left and a positive value to the right.

Interval : Specify the incremental steps on which to calculate the values. Normally this would be 20 meters.

 

Include PI data

If this check box is selected, all the curve chainages (start curve, end curve) will be included in the list along with all the regular specified chainages.

 

       

EXAMPLE of CAD text;                Chainage, Y co-ordinate, X co-ordinate

         

Report

Model Maker will produce a report containing information of the position of certain points in relation to the horizontal alignment. This function is the same as the first CHAIN function described above. However, it works automatically and reports more information to the editor.

In addition to reporting the chainage value at the intersection with the centre line, the report also holds the offset distance from the alignment (negative = left;  positive=right).

The report could either include all the identified points or only those you indicate manually.

Points indicated by the user do not only have to be DTM point. Use the snap modes to indicate any position on the screen.
