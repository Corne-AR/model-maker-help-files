MENU : teR - Hatch cut-fill

 

When doing terrace cut and fill the user can specify that the cut points and fill points, generated on the regular grid must be copied to a selected project.

See the Output page of the terrace volume dialog box.

 

In addition to the regular cut and fill calculations, when using the autostring option to do multiple platforms as specified by strings, one can also output the cut and fill grid points to project number 4.

 

A typical result of the cut and fill grid points added to a project will look like this.

 

 

The hatch cut-fill function will scan all the points in the active project and create hatch elements on top of the cut and fill points. Hatching the different areas allows you to distinguish and identify the different earthworks operations much better.

 

After the hatch cut-fill option is selected the program will scan all the points in the active project to ensure that they are all on a regular grid. This means that the function will only work on a dtm project that solely consists of even points on a regular grid.

 

The following dialog will be displayed.

Use the hatch parameter buttons to set the hatch attributes of each code.

Once these attributes have been set, the program stores it and you only have to use the buttons again if you want to make changes to the default settings.

 

If the OK button is clicked the program will proceed to inspect the point codes and create the hatch areas.

NOTE : Since the original cut and fill points are on a regular grid, the resulting hatch area will also have a slightly uneven perimeter.
