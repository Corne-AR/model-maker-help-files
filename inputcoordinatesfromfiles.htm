MENU : Input Co-ords - From files

 

One of Model Maker's strengths is its ability to communicate with other systems via ASCII files.

 

For this example, save the  OX  tachy data into an ASCII file using  FILE - SAVE, and then select ASCI. The file extension will be .YXZ.

 

Loading ASCII co-ordinate files is easy. Use the FILE - Import - ASCII option. Select  FILE - IMPORT - DEFINE to load other ASCII files.

 

Remember  Model Maker  can save any of its own survey coordinates in an ASCII file. The OTHER ASCII data is to let Model Maker know that the format of the data structure in the file is unknown to Model Maker and will be specified.

 

Model Maker uses a easy method to specify the ASCII format. An ASCII file must consists of the CODE, Y, X, Z co-ordinates.

 

After specifying the filename (ZX) and the extension (YXZ) choose to read the data in a FIELDS or COLUMN format. For this example, use the COLUMN format.

 

You would be presented with a line containing groups of  CCC  TTT  YYY  XXX  ZZZ  characters. Place the relevant character over the column containing the specific co-ordinate information.

 

Model Maker will read the  ASCII  file and line by line pick the correct co-ordinates from the columns and record each point into its own database (CODE, Y, X, Z).

 

A .dxf file is intended to transport CAD drawings between systems. Model Maker read s DXF files into its CAD system. It can also read and translate .dxf data into survey points.

 

Model Maker can also read  MOSS GENIO  files.
