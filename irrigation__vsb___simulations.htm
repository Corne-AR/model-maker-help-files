IRRIGATION : VSB - Simulations

 Top  Previous  Next

The simulation exercise is intended to help the designer in a scenario where no fixed shifts exist and any or all the valves can be open at any time.

 

After running the simulations the results can be loaded into Excel and a graph of flow/pressure can be drawn to produce a band of operating possibilities in order to select the correct combinations of pumps to service the flows.

 

Since one of the possibilities would be that all the vales are open, it is important that the pipes must already be sized to handle the flow of all the valves. This is an easy scenario and the user would need to first assign all the valves/blocks to a single actual shift and do the pipe sizing for that.

 

Note: This simulation will assign a virtual shift to the valve elements as it calculates the options. The actual shift numbers of the valves are not affected and the design is not changed in any way. It only gives you the needed information to help you make a better design decision in regards to the number of pumps to be used.

 

 

 

1) All the valves in the mainline (Regular,Blocks,Pivots) will be counted and assigned a character. (A-Z  a-z and 30 special characters). The system can therefore handle up to 84 valve entities. The text representing the valves is displayed here.

 

2) Based on the coded text the Build Combinations will determine all the combinations and list them in (4)

Press the Build Combination button after setting the selections in (3).

 

3) There are 4 building selections.

 

·All Single : Only the individual valves will be listed.
·Include All : You may define which valves must be in the simulation by specifying their character string.
·Include Any : The combination list will be build including at least any of the selected, coded valves.
·All :  All the combinations will be listed using all the valves. (Only possible with 12 or less valve elements)

 

4) All the combinations of characters are listed here.

 

NOTE: MM can do a maximum of 20000 combinations and you can quickly determine when that will be exceeded.

A design with 12 valves will have 2000+ combinations and 13 valves will have 4000+ combinations. A design with 27 valves (the full alphabet) will have 67 000 000+ combinations.

This illustrates that MM can simulate all the valves combinations for a design of 15 valves.

If the number of valves combinations exceed the combinations limit then MM will automatically start to build the combinations by inserting the new possibilities to create an even distribution.

Use the INCLUDE ALL or INCLUDE ANY options to include or exclude specific valves.

 

5)  After the combinations have been listed in 4, press the Run Simulation and MM will activate the valves for each combination string (4) and run the hydraulic calculations. The needed pump pressure will be determined and the results listed in (7)

Internally, MM will create a virtual Shift with the string valves operating and then do the hydraulic calcs.

 

6) The maximum pump pressure is shown here as an extra piece of information. This will normally occur when the most/all the valves are operating.

 

7) The list with results.

 

8) The results can be saved as a csv file which you can load into Excel and draw graphs from it.

 

9) Here you can limit the combinations and also specify a design pressure margin and flow limit.

The design pressure limit will be added to all the calculated pressure (eg.  in 6)

The Flow limit will only show the combinations with flow less than the specified % of the total flow of all the valves.

If the total flow is for example 1600 l/s and the Flow limit is 50% then only combinations with max. 800l/s will be shown.

 

General Notes:

 

When the dialog opens the first time, the ALL single valves option will be run. You can use that info to see which are the critical valves if you need to Include a certain number of valves to limit the combinations.

 

A) As indicated previously, a character is assigned to each valve. This column shows the combination of characters for the open valves. (Each line of info can be regarded as a shift in which the listed valves are open)

 

B) If the design has regular valves (not representing a block or pivot) they will be listed here. The numbers in this column will be actual node numbers.

 

C) Valves representing blocks will be listed here. This number will be the actual block number.

 

D) Valves representing pivots will be listed here, This number is the actual pivot number in the design.

 

E) The combined flow of all the open valves.

 

F) the lowest pump pressure needed to provide the required pressure to all the valves that are open.

 

 

 

 

 

 

When multiple valves are open, the range of numbers will be listed in the appropriate columns.
