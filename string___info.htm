MENU : gXt - infO

 

Display the global information of a string.

 

 

Fixed info

 

 

# of nodes

The number of horizontal nodes on the string.

Area (m2)

That is defined by the string.

Length (m)

The perimeter length of the string.        

Slope length

The length between nodes taking the elevation into account.

 

 

String type

 

The type of string. The format is fixed and can only be defined when the string is created.

 

General string

The length between all the nodes

Alignment string

The actual calculated chainage length of the horizontal alignment.

   

Pen & line type

The default pen dialog box will be presented. Any changes will be applied to the current string.

 

 

Calculate new area

Calculate a new area as defined by the string. Model Maker will take an indicated leg of the string and move it in or out as to create a new requested area.

 

 

 

Move parallel

The indicated leg will keep the same length and direction, but will move parallel to the original either in for a smaller area or out for a larger area.

Move horizontal

More appropriate for strings that define an area on a cross-section.

 

The leg will keep its original length and angle but the iteration process will move it on a horizontal line only.

Use the iterate button to perform the iteration process.

 

 

Fill/Hatch

Enter a fill percentage to let Model Maker fill the area as defined by the string with the selected colour. Use a value of zero (0) if it is not to be filled. Click on the black box to open the colour definition box and define a color. Click on the ? button to point to another string from which the colour must be taken.

 

Volumes

This option calculates volumes between strings where these strings represent areas of cross-sections.

It would usually be applied to the following situation.

 

 

Fig 1 & 2 represent two sets of cross-sections as displayed in the Model Maker's CAD. These sections could have been drawn manually or loaded from DXF files or read from Road Maker etc.

Fig 1 shows the sections at chainage 10 and Fig 2 that at chainage 20. The areas of interest within each section are defined by strings A, B and C in the figures.

It is important to note that all the strings of fig 1 are drawn on layer 10 and that of fig 2 on layer 20.

The chainage value of the section is defined in the labeling of the string layer.

The string names are given as A,B and C.

 

Model Maker now has all the necessary information to calculate volumes using the following logic.

 

1) String name A in figure 1 is identified on layer 10. The area is calculated and the chainage is defined as 10 meters.

 

2) The string data base is scanned to see if any other strings with the name A exists.

    The string A on figure 2 is found on chainage 20. The area is calculated and the volume between the A strings is calculated with the end area      method.

 

3) The same is done for strings B and C.

 

4) A volumes report is printed.
