Coordinate System

 

Select the coordinate format.

 

Unchanged : Output as is. Y=Y and X=X

 

Swap : The Y and X will be swapped

 

Invert : The sign of the coordinates will change (multiplied by -1)

 

Swap and Invert.  Both changes will apply.

 

TEXT PARAMETERS

 

 

Standard font

 

If this box is checked Model Maker will output all the text as standard font. This is often necessary since certain defined fonts in Model Maker may not exist in receiving CAD's program environment. In such a case, you will lose the well-defined CAD drawing, but the receiving program will be able to import the .dxf.

Text size factor

 

This option is typically used when sending a .dxf file to another package.

It will multiply the DTM, CAD text, codes elevations values etc.

Text font ratio

 

The default value is 1. However, Model Maker defines its text as true-type font. Other programs may interpret a font not to be true-type. Changing the ratio to a value larger than 1 will make the text to appear longer. A value less than 1 will shorten the text. In the below example the ratio is set to one. The height is set to 5mm and the width is set to 3.

 

 

In this example the height of the text is set to 5mm and the width set to 3mm. Using text ratio 1, the height and width will remain the same in the .dxf file.

USER LINES      

 

 

Solid lines

 

Model Maker will output the user defined CAD line types as solid lines in the .dxf file. You will lose some of the detail, but the receiving program will be able to load the .dxf.

The example below shows a fence line which has been read in as a solid line. The length of the line will remain the same.

 

 

 

Defined as

 

The user defined line types will be outputted to the .dxf file with their line type names as defined. The receiving CAD might not know how to handle the lines if similar lines types are not defined.

 

 

 

Exploded

 

The defined lines will be drawn with solid lines but all the patterns on the user line (eg, the cross of a fence line) will be drawn with individual lines. The picture on the receiving side will look like that in Model Maker but the exploded lines will increase the number of lines in the .dxf file considerably. The example below shows a small portion of a fence linetype. Using the explode option will result in the output having 4 components instead of 1. When the .dxf file is read into the receiving package the result will look like that in Model Maker. It must be remembered that once read into the receiving package the length of the line (because it is no longer one entity) will not be the same.

 

 

 

 

3D

 

Check this box to define the elevation in the .dxf file. Normally, .dxf files are defined as 2D. In some cases, the receiving program can use the elevation if the 3D co-ordinates are defined.

 

 

Co-ordinate system

 

Select output co-ordinates in these information boxes.

 

 

Default

 

Click this button to select the Model Maker default format.

 

 

Constants

 

Adds or deducts these constant values from all the co-ordinates in the new .dxf file.
