MENU : Irr - Nodes

 

Model Maker defines an irrigation network by nodes and pipes. Nodes are needed as hookup points for the pipes and may be either pumps, sprinklers, valves, fittings or others.

 

A separate database holds the information regarding all the pre-defined node type. Before nodes can be added to the irrigation project, at least one default node type must have been defined in the database.

 

Full control is provided over the graphical manipulation of the nodes.

 

 

 8 

Plot Parameters

 

 

 

Apart from the general plot parameters that can be set, specific properties for displaying the nodes on the graphics screen can be defined.

Display nodes

 

Use this check box to show or hide the plotting of the nodes. If the Display Nodes is selected, all the nodes will be shown according to the following setup.

 

Only Active block

 

In general all the nodes of all the blocks are displayed on the plot. This can sometimes be confusing when you want to identify the position of the active block. By checking this check box, only the nodes of the current selected block will be displayed after doing a replot.

Remember to un-check this box before doing a printer plot unless you desire only to display the nodes of the active block on the paper plan.

 

Shift Colour

 

Normally the nodes are drawn in the colours assigned to it when the nodes were added to the project. This is the color specified in the node data base. Check the Shift Colour box if you want the nodes to be displayed in the colour of their allocated shifts.

 

Pump, Sprinkler etc.

 

Use these 5 check boxes to individually select the display of the individual components. Using this feature can be quite useful when you only want to display nodes of a certain type.

Keep in mind that this can also be very confusing when you work on a new project and certain nodes are not displayed because of the settings which you have forgotten to reset.

 

Radius

 

The defined spray diameters of sprinkler nodes will be drawn with a circle around the node.

 

Numbers

 

The node number as defined in the database will be displayed in the defined text size.

 

User Name

 

The name as defined by the user in the node edit dialog box, will be displayed.

 

Patterns

 

When a node is defined in the data base, a pattern name may also be specified. This name refers to patterns as defined in CAD. Check the patterns check box to display the defined pattern at each node if the pattern is available.

 

Shape

 

By default the nodes in the irrigation project are displayed by means of a pre defined shape. These shapes are placed in the MMW.SHM file to be found in the Model Maker directory. Each shape is identified by the descriptive name. PUMP, SPRINKLER, VALVE, FITTING and OTHER and can adjusted to your own needs. Use the Default, Nodes, Edit shape option to change the format and size of the shapes.

 

Text Sizes

 

Set the height and width in millimeters of the node numbers size.

Add Node

Single

 

After selecting the node type to add, place the node at any position using the cursor mode functions. These nodes will automatically interpolate the elevation from the DTM model at the specified co-ordinate. If no triangles are available or the node is placed outside of the triangle model, it will be allocated a zero elevation.

 

Junction

 

Nodes will be placed at all the endpoints of the cad lines.

 

DTM-points

 

Nodes will be placed on all the DTM points. This is usually done after a set of DTM points have been defined with the Layout functions.

 

Grid points

 

Nodes will be placed on all the grid points within a defined area. Use the cursor mode dialog box to specify the grid interval and position. The area definition functions will be used to define the area. The new nodes will be added to the current block.

 

 2 

Move Node

 

Move a single indicated node to a new position. Note that all the pipes connected to the nodes will also have their one terminal updated accordingly.

 

 3 

Elevation

 

 

All the nodes of the active block will be re-elevated by interpolating the elevations from the triangle model of the active DTM project. This function is often used when some nodes with a zero elevation is present.

 

If this check box is checked, all the nodes of all the blocks in the irrigation project will be updated.

 

 4 

Database

 

The pre-defined node database will be displayed. Use the available functions to edit the supplier items.

 

 5 

Edit Node

Single

 

Select the node to be edited. The node info dialog box will be displayed where you can make changes to the data or select a different node type.

 

Type

 

After selecting the specific node type to be edited, all the nodes with the default type will be changed to the new defined type. When the node dialog box is shown use the Get Type function to pick an existing node.

 

Area

 

All the nodes in the defined area will be changed to the new selected node type.

 

 6 

Delete Node

Single

 

Select the node to be deleted. Note, only nodes that do not have any connecting pipes can be deleted.

To delete the pipes connected to a node use the Delete function on the pipe page.

 

Type

 

After selecting the specific node type to be deleted, all the nodes of the specified type that belongs

to the active block and that does not have any connecting pipes, will be removed from the project.

 

Area

 

All the nodes in the defined area, belonging to the active block, will be deleted and removed from the project.

 

No Pipe

 

All the nodes that have no pipes connected to it will be deleted.

 

Delete All

 

All the nodes will be deleted

Number of nodes - Zero nodes

The panel at the top right of the dialog box indicates the number of nodes in the active block.

The zero field indicates the number of nodes with an elevation of zero.

When you are doing an irrigation design and have surveyed and modeled the terrain correctly,

you would expect all the nodes to have an actual elevation as interpolated form the triangle surface.

However, it often happens that some of the irrigation nodes are placed outside the surveyed model

with the result that Model Maker cannot interpolate an elevation for them.

 

These nodes, falling off the model, then gets a zero elevation.

This will result in the incorrect calculation of the hydraulics and measured length of the pipes and

should be avoided.

To see which of the nodes have zero elevations, click the Num. of nodes panel.

The graphics menu will be displayed and the zero elevated nodes will flash on the screen.

 

Press the escape button. The normal node edit function is activated allowing you to edit

the nodes and to assign them the correct elevations.

Another method to update the node elevations is to correct the triangle model by adding more

survey points and triangles to cover the areas where the nodes are falling off the model and then

to use the Automatic elevate option.

 

Note :

All the delete functions only operate on nodes in the active block that do not have any pipes connected to them.

 

Densogram

 

 

The densogram button on the display toolbar will draw the wetted area of the sprinkler in a user defined hatch image.

The densogram parameters are set on the defaults page.

 

 

 

 

Full fill : A very solid image is produced but takes a little longer to plot.

 

Quick fill : The densogram image has a dithered pattern, it plots faster and will be lighter.

 

 

 

Select the color band to produce the flooded fill.

 

 

The KD factor

The KD value refer to the secondary pressure losses created by emitters inside the pipe and is only relevant when the pipe is used as drip lines.

In drip lines, the kd factor of the pipe is used when secondary losses are calculated.

If the kd value of the pipe is zero and the drip emitters have a kd value, then that value is used.
