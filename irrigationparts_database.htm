MENU : Irr - Parts - Database

 

This menu puts you in control of the parts and cluster data base.

               Any part number may be defined along with a description, cost a and part type.

               A combination of parts can be grouped together as a cluster.

               Items from either the parts data base or the cluster data base can be added to any of the nodes making up the irrigation                project.

 

 

Parts

 

The maximum length of the part number is 15 characters.

 

This grid editor gives direct access to the items in the parts data base. Any line may be selected and updated.

 

Use the control bar to traverse the data base, update and delete items.

To view the function of each button,  look at the hints that are displayed when moving the mouse over them.

 

 

 

 

 

 

 

Parts in cluster

Up to 10 part numbers can be added to a cluster. For each entry, enter the part number and how many are used in the cluster. The cost of the part will automatically be recorded from the parts data base. The total cost of the cluster is also calculated and displayed at the bottom right.

 

 

 

The cluster editor displays the cluster data base. Any of the cluster number or description may be edited. Use the parts input boxes to add up to 10 parts that make up the cluster.

Use the control bar to add, edit or delete an item.

Clusters

 

The maximum length of the cluster number is 15 characters.

 

 

The cluster data base is a bit more complicated than the parts data base.

A cluster is compiled from a selection of up to 10 part numbers taken from the parts data base.

Note:

Each irrigation node may have up to 4 part or cluster numbers allocated to it.

Take for instance an example where a sprinkler node is placed on the manifold line and connects 2 lateral pipes to the manifold.

 

EXAMPLE: the manifold pipe reduces from 140 to 100 and a 40 lateral and 20 lateral connects to it.
