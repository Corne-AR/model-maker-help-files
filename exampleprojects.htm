MENU : Example - Projects

 

Model Maker's 21 project system offers enhanced performance.

 

The two main objectives of using projects are:

a) Viewing 2 parts of a survey simultaneously, but keeping it separate for manipulation purposes.

b) Calculating terrace volumes between a before and after survey or, for example, calculating volumes for an open cast coal mine.

 

Most of the time, you will only need to use one project, such as in ordinary tachy reductions and contour plans. Nevertheless, familiarize yourself with the advantages of the projects system and apply it when necessary.

 

This example illustrates the basic concepts of projects :

EXAMPLE:

Two surveyors have surveyed an adjacent area. One did the northern half and the other the southern half. You have downloaded, reduced and saved the data of each surveyor. Now you would like the 2 surveys to be combined to form the complete picture.

 

a) Load the data of surveyor 1 into project one.

b) Load the data of surveyor 2 into project two.

c) The DTM points of each project are displayed in the colour of the project. If the default project colours are used, the points, codes, elevations etc in project 1 will be plotted yellow on the screen. Those of project 2 in green and those in project three cyan.

d) Combine the two projects by making project # 2 active and Id all the points.

e) Use the PROJECT COPY  function and copy all the points of # 2 to # 1.

f) All the data is now combined in 1.

g) Save the points of # 1 to disk.

 

Notes about projects

Buttons on the top right of screen.

a) The following default colours are used for the projects :-

    # 1                :                white

    # 2                :                green

    # 3                :                cyan

    You may change colours in the colour menu.

b) The cursor on the graphics screen, when displayed as a cross, always has a small 1, 2, or 3 next to it according to the active project.

c) The DTM points of each project are displayed in the colour of the project.

d) The active project refers to the project which would be used when information like triangles, contours, points, editing, getting cursor co-ordinates from points etc. is requested.

e) Only one project can be active at a time. The other two will then be in background mode, meaning they will be displayed and cannot be manipulated.

f) Every project may hold the same maximum number of points and triangles. If your system is activated for 9000 points, each of the 3 projects may hold 9000 for a total of 27000 points.

g) Every project has the same status, meaning that no project is reserved for any particular purpose.
