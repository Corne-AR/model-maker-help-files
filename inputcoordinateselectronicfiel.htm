MENU : Input Co-ords - Electronic field book

 

Model Maker has a comprehensive set of routines that make it possible to load and reduce survey data taken by means of electronic instruments or hand held computers.

 

To access the functions do the following:

               On the top menu use the UTILITY option and select ELECTRONIC DATA.

 

Background

Reduction of field book data involves 2 components, the stations and the observation points.

               The stations refer to the positions where the survey instrument is placed each time a new setup is made.

               The points refer to all the surveyed measurements taken from each of the station setups.

 

An example of what a typical electronic survey file looks like.

 

L1                                2                7.0394                                11.3314                0.000                1.582                                This line describes the station

1                DRD                5                145.0634                91.0932                89.172                1.300                                   These are the angles and distances to the points

2                DRD                5                142.0214                91.0555                81.165                1.300

3                DRD                5                137.5519                91.0318                72.669                1.300

4                DRD                5                133.0726                90.5321                65.090                1.300

 

Overview of the procedure

Model Maker will reduce the all the surveyed points in the electronic field book file in the following manner:

               The data in the field book will be read and when a station is encountered, a new station record will be created in the Model Maker data base.

               The trig data base will be checked to see if co-ordinates are available for the specific station name.

               The station input box is presented with the co-ordinates from the trig data base or if none were found, zero co-ordinates will be displayed.

               If a trig beacon was surveyed for orientation purposes, the Correction Angle Dialog box is displayed. (Explained below)

               All the spot shots surveyed from the station will be reduced according the station co-ordinates and correction angle.

 

Procedure

Step 1

Download the observation and co-ordinate files from the data capturing device into the computer using a COMMS link or another program supplied by the survey instrument dealer.

Normally the survey data will consist of the observations and co-ordinates files.

               Observations                : A file like the example above - containing the setup information and survey data to all the spot shots.

               Coordinates                : This is a file that contains the co-ordinates of the positions where all the stations were setup.

         Many data capturing devices have build in programs that calculate the co-ordinates of the stations by means of traverses at the time                                when the survey is done. If you do not have such a list of coordinates, use the traverse function to calculate the co-ordinates of the                stations.

 

Step 2

In Model Maker, use the  UTILITY - ELECTRONIC DATA option and select the Trig Data Edit option.

In the trig editor, load the co-ordinates from the surveyed coordinate file or enter the co-ordinates of the stations.

               Model Maker uses these co-ordinates to place the stations correctly when reducing the field book.

               These co-ordinates are stored in a database specifically designed to hold the trig / station data for the electronic field book reduction                  process.

               This database has no relation to any of the Model Maker DTM projects.

               The trig file menu has its own data editor and the coordinates may be saved in a normal MODEL MAKER ASCII co-ordinate (YXZ) file.

         Use the FILE - OPEN - ASCII  option if you want to load this trig / station data into one of the actual DTM projects.

 

Step 3

               In the Electronic data menu, select the button that will allow you to reduce the data of your specific format.

               Enter the file name - Model Maker will read the file and stop at each station.

 

The station name and its co-ordinates will be displayed. Inspect the co-ordinates and edit it if necessary. Whenever a station or back sight reading to a trig point is encountered, Model Maker will first inspect the trig database to see if co-ordinates have been defined for that point and use those co-ordinates. If you are reducing a field book file without having loaded the trig data beforehand, Model Maker uses zero co-ordinates as initial values for the station / trig. You must then manually type the correct values.

 

Whenever an orientation reading is encountered in the observation file  Model Maker presents the orientation correction menu.

               An orientation reading is used to calculate the correction angle of a station. Normally, when doing the survey using electronic instruments, a survey measurement is taken to one or more trig beacons (or any other known point). These measured directions are then compared to the calculated directions and the difference is then added to all the horizontal angles measured to all the spots surveyed from the station.

 

Correction Angle Dialog box

Move the cursor over the graphic and when the appears press the enter button on the mouse and a more detailed explanation of the function in question will be given.

 

This menu will calculate a join between the co-ordinates of the station and that of the trig point. It will compare that to the actual direction that was surveyed in the field. The difference between the 2 directions will be taken as the correction angle. The correction angle will be added to the horizontal angle reading of all the observation points surveyed from that station before the co-ordinates are calculated.

 

The correction angle menu has some additional features.

 

The name of the trig point can be changed in this menu. The co-ordinates of the new trig will be looked up in the trig database, and recorded into their respective positions on the menu. A new direction will be calculated from the station to the new co-ordinates and then compared to the actual survey direction. The correction angle will be updated accordingly.

 

The following example illustrates the usefulness of this function.

 

1.                Your station is called STA.

2.                Your back sight is to TRIG 150 and you know you were working in orientation meaning that you expect a 0 correction angle.

3.                The correction menu calculates the correction as eg. 25'.

4.                You realize that the trig should be TRIG 151 and not TRIG 150.

5.                All you need to do is to change to the correct trig name.

6.                Model Maker retrieves the new coordinates and updates the correction angle.

 

The last option on the correction angle menu is: Accepted Correction.

This allows you to override all the calculations of the correction angle by manually entering the angle you think is more accurate.

 

Note:                

The reduced field book data is recorded into the selected project. This survey data is now in Model Maker in the normal format and should be saved to disk using the FILE - SAVE - TACHY option.

 

 

Although the program calculates the co-ordinates of all the survey points. The original field book, (prism readings, distance and vertical angle) is also recorded in the project database. The points and stations may be printed and edited in the  DATA - POINTS / STATION  menus.

 

Setup at station

These input boxes holds the station information and are normally not changed by the user.

               If the dialog box was accessed from the data base, the co-ordinates will be those of the station that you have selected. If you have to make changes to the data, do that in the

               data base and not in these boxes.

               If the dialog box was accessed from the electronic field book reduction process, make changes if required. The trig editor will be updated accordingly.

               Use the UPDATE button to do the recalculations after the changes.

 

Using multiple stations

Model Maker can handle 50 survey stations per project. Make sure that any survey site consisting of more than 50 stations is recorded to observation files in groups of 50 or less stations.

 

You can also use any editor program to load large observation files and split them in smaller files of not more than 50 stations.

               Please note however that some editors change the TAB characters in a BOOKER file to spaces and  Model Maker  will not be able to interpret the data correctly.

 

Model maker is able to produce a contour plan of a site if for example 75 stations were used in the actual survey.

 

STEP 1

Make sure that all 75 stations are recorded in different .BKR files with each having less than 50 stations.

 

STEP 2

Reduce every file in the normal manner. Go to the  FILE menu and save the data for every file as a TACHY as well as an ASCII file before the next file is reduced.

 

The data is saved in a TACHY file in order to retain a copy of the data in the full field book format. This is useful because this format allows the user to make changes to a station if necessary and all the survey spots from that station will be recalculated.

 

Saving the data in the ASCII file has the following effect. It is saved only as a list of co-ordinates meaning that the distinction between the station and the observed points, no longer exist.

 

STEP 3

After reducing all the observation data , go to  FILE - OPEN - ASCII  and load the first ASCII file. Although the data may consist of say 50 stations when the file was initially saved, The data will be read back into  MODEL MAKER  as one KOORD  station. Remember a KOORD station is a dummy station capable of holding only data in Y X and Z co-ordinate format.

 

Now load the second and third ASCII files use the ADD to file option (J) in the file dialog box..

 

Note:                

When a file is loaded into  MODEL MAKER via the FILE - OPEN menu, MODEL MAKER  automatically replaces any current data with new data. Checking the ADD TO FILE box instructs the program to add the new data to those already in memory.

 

Our 75 station project is now loaded back into  MODEL MAKER  in the database as 2 KOORD stations.
