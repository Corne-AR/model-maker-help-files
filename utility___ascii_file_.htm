MENU : Utility - Ascii files

 

 

MM can only accommodate a certain number of points. Use this function to split a large number of points into workable sections.  A single rectangular area between given co-ordinates can be extracted, an area can be divided into blocks of a given size or a strip of data could be extracted over a defined horizontal alignment. The data must be in ASCII format (code, Y, X, & Z) with the different values separated by one or more spaces. Refer to the Load ASCII option in this section on how to load this file.  

 

Get Limits

 

Scan a file to find the maximum and minimum co-ordinate values in the file. This represents a rectangular block around all the data in the file. Beware of points with a 0 co-ordinate while the average is a greater value.  

 

 

Extract one block

 

Extract a block of data from a larger area by specifying the co-ordinates for the bottom left and top right corners of the block.

You will be prompted for the filename to read the data from and the co-ordinates of the block that must be extracted.

 

 

Extract sequential

 

Extract the data between two data line numbers. You will be prompted for the start and end number of the data points. All the lines of data preceding the start number and all the lines of data after the end number will be ignored.

 

 

Extract strip

 

Extract a strip of data along the center line of a road.

This centre line file must be created and saved with the Horizontal alignment - Cenf option on the main graphic screen.

The file extension must be .CEN. You will be prompted for the width within which the data must be extracted.  

 

 

Print ASCII file

 

Print any text (ASCII) file to a printer.

 

Convert ANSI to ASCII

 

This function will read a regular ANSI text file and convert it to a regular ASCII text file.

An ANSI text file consists of one long line of data with a certain line termination character embedded in the line to mark the start and end of the actual individual strings. This specific character is normally character number 10.

MM will read the ANSI line and break it up in multiple lines, each occupying a single row terminated by a carriage return value.

The original ANSI file will not be changed but a new file with an underscore prefix to the original file name will be created to hold the new ASCII lines.

 

EXAMPLE:                ANSI file name                :                .dxf

                         ASCII file name                :                -Township.dxf
