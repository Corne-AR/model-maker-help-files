MENU : Tri

 

Most of the functions relating to triangles are found under the Tri  menu on the left of the main screen.

 

 

Form a triangle by connecting 3 points and form the surface between the 3 points.

 

Visualize a triangle, not as 3 lines, but as a triangular surface between 3 points with the boundary of the triangle being formed by the legs.

 

In memory, a triangle is stored as an object with six variables. Variables 1 to 3 hold the database number of the DTM points on the corners of the triangle. Variables 4 to 6 hold the number of the neighboring triangles on the side of the triangle. The term open side refers to a triangle that has a side not connected to a neighboring triangle.

A model consisting of only 1 triangle has three open sides and, if the model has 2 triangles next to each other, each triangle will have 1 closed side, (the side where the 2 triangles are joined) and each will have 2 open sides.

 

About open sides:

 

1)        Adding a new triangle to a triangle model can only be done from an open side of the existing triangles.

 

2)        The open sides of the triangle model will always indicate the boundary of the model.

 

         

 

3)        Open sides can be inspected with the SIDE  option in the  TRI  menu.

 

4)        Contours originating from an open sided leg of a triangle will result in a contour that does not form a closed area. Therefore, it  will        always stop on an open sided triangle  again.

 

5)        The perimeter of a triangle model, as depicted by the open sided triangle, can be copied to CAD lines via the  CAD - GEN - COPY -        SIDE  function.

 

6)        It is good practice to use the  TRI - SIDE  function after a triangle model is created in order to ascertain that the triangles cover the        whole model and that no gaps are left within the model. If you find any sides flashing inside the model, zoom or magnify into those        areas and manually add new triangles in the open areas with the TRI - ADD  function. It is sometimes useful to use the  CONTINUE        SEARCH  (CSEARCH) option to have  Model Maker closes those gaps automatically.

 

7)        The  CONTINUE SEARCH  function will always inspect the open sides and try to add new triangles from them.

 

 

THINGS TO WATCH OUT FOR :

1)        Triangles should never overlap each other. Model Maker takes all precautions to  prevent triangles from overlapping or, in other words, model the same area with 2  triangles.

 

Some situations result in an incorrect triangle model:-

 

a)                Where triangles have been connected to a point and the point is then moved so far away from its original position that it drags the triangle over the legs of others.

 

         

 

       

 

b)                Model Make  does not allow triangles to be built on a set of points that have duplicated points. Although the program checks and tries to accommodate duplicated points before triangles are generated, confusing situations arise and Model  Maker could lose control over the triangle model and overlapping triangles may be formed.

 

Use the  TRI - WIPEOUT - LAST  function to backtrack out the situation if possible. The bottom line is to try and avoid a model with duplicated points. From        the top menu, use the  DATA - GENERAL - AUDIT option to deactivate duplicated points.

 

2)        Model Maker cannot model a solid surface, overhang or vertical surface.

 

3)        On average, you will have twice as many triangles as points on a model.

 

4)        Triangles are normally saved separately from the DTM points in a  .TR2  file. The 2 file formats that save the triangles and DTM points together in one file is the TOT and MAL formats.

 

5)        Often,a single project of points may have different triangle models that fit over it. For example, you have a set of DTM points                modeling a coal seam having points coded TC for top of coal and BC for bottom of coal. Separate sets of triangles may be                formed to all the TC points and saved to disk and triangles to BC points and saved under a different file name to disk.

 

6)        Because triangles are referenced to coordinates via their point numbers in the database, it is important to note that a set of                triangles can only be useful if loaded together with the points from which it was originally created.

 

7)        Number 6 implies that triangles data is of no use if the original points data is not available.

 

8)        Number 6 implies that you should not alter the structure of your points database after a triangle model have been based on it,        meaning that deleting or adding points to the database in between the original points would jeopardize the integrity that exist        between the triangles and the points.

 

 You can add points to the database if the new points follow sequentially after the last points of the original database.

 

9)        The only file type that saves points and triangles together in one is the TOT file.
