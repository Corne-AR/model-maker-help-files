MENU : teR - Autostring - Parameters

 

Calculate volume areas  between 2 surfaces using the terrace options.

 

The parameters control the calculation method.

Grid interval

This is the grid size used for the calculation.

A suggested grid value is displayed as a guide line.

In general, use a grid interval smaller than the suggested value. This will ensure an accurate volume result.

Also remember that the calculated results are going to differ with each new grid interval because the measurements between the 2 surfaces are based on the height difference at each grid point. If the position or number of grid points are changed, the answers will be slightly different.

 

Grid direction

Model Maker will calculate the grids in horizontal and vertical lines across the model if a zero value is specified.

In a case where the design or ground data is on a grid format, a zero grid direction is not recommended since it could lead to answers not representative of the models.

A grid direction of 5 degrees is normally used. Changing the grid direction will also result in different answers for the same reasons as stated above for the gird interval.

 

Note:

In general, a smaller grid size should result in a more accurate volume as a larger size because more grid points have a better chance of taking all the topographic changes into account.

A grid direction other than zero will also have a better chance of getting a more accurate answer because the points will not coincide with platforms and other shapes that are directly orientated east-west and north-south.

If the result is saved as an ASCII file to be used in the mine rehabilitation module, a zero value must be entered because the mining module would suspect the data to be in a random format when calculated at an angle.

 

Topsoil depth

This is the value (in millimeters) taken from the ground model before the mass earthworks is calculated.

The topsoil is removed from the ground model and reported separately.

 

Layerwork depth

This is the value (in millimeters) taken from the design model before the mass earthworks is calculated.

The bulk cut & fill answers are calculated from the ground surface less the topsoil to the design surface less the layerworks.

 

Output or record the gridded depths using the following options.

 

 

Save depths to ASCII file

The elevation difference between the ground and design surfaces will be recorded and save to a user defined ASCII yxz file. This file has the normal Model Maker yxz co-ordinate file format except for the elevations that are actually height differences. This file may be used in the mining module.

Add depths to project

The height differences on the grid will be added to the selected (1,2 or 3) project as normal dtm points with either a CUT or a FILL code. The elevation of the point will be the height difference between the design and the ground projects.

 

 

The following information is added to the CAD as normal elements.

None

Nothing is added to the CAD.

As lines

Lines will be drawn on the grid points. The lines will have different directions, layers names and colours depending if the grid is a cut or a fill position.

Crosses & circles

Crosses will be placed on all the cut points and circles on the fill points.

Pattern

This option will only be active if a pattern named CUT and another named FILL exist in the pattern directory. These 2 patterns will automatically be placed on the appropriate positions.

Text

CAD text on cut and fill layers with different colours will be placed on the grid positions. The text value will be the depth between the surfaces.

 

This input box allows you to set the relative size of the lines and circles when outputted to CAD. This size works hand in hand with the selected grid size. If, for instance, you have a grid size of 10, then the crosses will be 10 wide and the circles will have a radius of 5. If the relative size is set to 0.5, then the crosses and circles will be half the size of the grid interval.
