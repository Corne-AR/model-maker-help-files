MENU : Irr

 

The hydraulics function applies water to the system and calculate the respective pressures and flows in all the pipes and nodes. Automatic sizing of pipes may be done and a report of the available flows and pressures can be generated.

This page displays the graphical picture of the hydraulic gradient of all the pipes when the flows and pressures are calculated. The relevant information is also displayed in a table format for numerical analysis.

 

Input Values

The following are displayed as indication of the flows and pressures:

 

Approximate flow required :        If a block is calculated the flow indicates the total discharge of all the sprinkler nodes in the network when operating at their design pressures. If the block is defined as a micro/drip system, the flow is the total discharge of all the emitters in the block. On the main line, the flow is the total consumption of all the valves operating in the selected shift.

 

Average pressure @ nodes :        If a block is calculated the indicated pressure is the average design pressure of all the sprinklers. On a micro/drip system, the pressure is equal to the global design pressure for the block.

 

       

Enter the available pressure at the source node.

If a block is calculated, this is the pressure at the valve (source) node.

If the main line is calculated, enter the pump operating pressure. If 2 pumps are used, please check the help topic for dual pumps.

 

More about pumps.

 

Click the Calc flow along pipes button to activate the calculation process. Irri Maker will apply water to the system with the defined source pressure and balance the flows to all the nodes.

If for any reason the calculations take too long, you may press any key to abort the operation. This could happen when one or more pipes in the system are too small preventing Irri Maker from finding a suitable convergence value. In such a case, inspect the pipe sizes and elevations of the nodes.

 

Error messages

In some cases Irri Maker will post a warning if it finds that the system cannot be resolved.

This could happen if :

       the pipes are not sized correctly, or;
       the slope of the terrain is very steep, or;
       the change of flow based on the different pressures of the selected emitters is too big.

         The last item can often be the reason that you cannot get a micro/drip system to converge correctly. In the micro/drip design method, Irri Maker makes certain assumptions and if the specified emitter has a very wide flow range (relative to different pressure values) it can cause the system to post the warning that the flow differences are too big. In such a case, try using the sprinkler method to do the design.

The following options has an effect on the hydraulics.

 

 

Apply secondary losses

If this box is checked, the secondary losses at each node will be calculated and deducted from the available pressure in the system. This will effect the regular hydraulics results and could also have an influence on the pipe sizing routine, depending on the magnitude of the secondary losses.

The k values are defined in the node data base  for each node type.

 

Note:

Remember that the k values are added to the node data base but only transferred to an irrigation design when new nodes are added to the design. This implies that older projects, designed before the secondary loss option in Irri Maker has been in place, will not automatically inherit k values from the node data base just because they are now available. Existing nodes will have to be redefined (re edited) to have them make use of the secondary loss option.

 

Regulate node pressure

If this box is checked all the sprinkler nodes will behave as if a regulator is placed at the node. That is, if a pressure higher than the design pressure of the sprinkler is available at the node, the sprinkler will perform at the design pressure level. The sprinkler will operate at the available pressure when that is less than the design value.

If the pressure to each lateral is to be regulated at the entrance of each lateral pipe and not at each node, introduce a pressure regulator node (node type=OTHER) at the start of the lateral.

 

Update pipe classes

After the flow to all the pipes have been calculated, Irri Maker calculates the maximum pressure in each pipe segment based on the available pressure at the valve or pump and taking into account the friction losses and elevation differences. These pressures are then compared against the selected pipe classes and the lowest pipe class that can accommodate the actual pressure is selected from the data base. If only one pipe class for the specific pipe diameter is defined and that is less than the actual pressure, a warning will be posted. If multiple classes are defined in the data base for a specific diameter, the appropriate class is selected. If the actual pressure exceeds the highest available class, a warning is posted.

Normally a pipe with the same nominal outside diameter will have different inside diameters for the different classes. The hydraulics are calculated using the ID values. If Irri Maker updates a pipe class, it results in a different ID value being allocated to the pipe which in turn has an effect on the hydraulics. For that reason a warning is posted when classes are changed, prompting you to click the Calc flow along pipes button so that the new hydraulics is calculated based on the revised ID values. Normally the hydraulics would not change much when the ID's are updated. It is however advisable to perform the recalculations.

Leave this option unchecked if the pipe pressures do not have to be checked against the data base values.

 

Warn : Overload pump

When the main line is calculated you would enter the available pressure at the pump under which the system will operate. Irri Maker will determine which values and sprinklers, if any, are open in the shift and calculate the amount of flow through the system. When the calculations are done, the pressure and flow at the pump node will be checked against the specific pump performance as specified in the node data base.

If the required values exceed the maximum pump capacity, a warning will be posted. This does not influence the hydraulics but only serves as a reminder to you that the pump would not be able to physically supply the demand.

Leave this option unchecked if you are not concerned about the pump's real performance.

 

Warn : Nodes on zero elevation

Irri Maker uses the DTM survey to assign the appropriate elevations to the nodes when they are generated. Often, when the survey model is not covering the total irrigation area, a situation may arise where some nodes fall off the model and can not be elevated resulting in huge elevation differences between some nodes. A situation that is definitely undesirable and may cause incorrect hydraulic calculations.

When this option is checked a warning will posted if any or all the nodes are on zero elevations.

You would probably deactivate this warning if you are not working with an actual surveyed project and do not want to be reminded about the nodes being on zero elevation each time.

 

Velocity Limits

The velocity in each pipe is calculated after the hydraulic analysis is performed. These velocities can be compared against certain user defined limits and a warning posted if the velocity of any pipe falls outside the criteria.

 

Click on the Warnings page and set the desired velocity limits.

If the velocity warning is required, Irri Maker will mark all the pipes with a velocity outside the limits, with a ( * ).

Graphics

 

A graph displaying the nodes on the horizontal axes and the pressures on the vertical axes is displayed on the top part of the dialog box.

Pressure lines will join all the nodes and is an indication of the pressure gradient.

The node numbers are displayed along the top of the graphics screen.

Blue horizontal grid lines indicate specific pressure grid values. The actual pressure value is written to the left and just below these lines.

The red horizontal line indicates the bottom (%Press-) pressure limit as specified by the envelope values.

The green horizontal line indicates the top (%Press+) pressure limit as specified by the envelope values.

 

Envelope limits. Only used for visual display purposes.
