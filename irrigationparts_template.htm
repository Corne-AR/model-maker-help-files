MENU : Irr - Parts

 

The parts template provides an interface to place part numbers automatically on the nodes.

The links between the nodes, pipes and part numbers are defined by the user by means of the following menu structure.

 

Notice that the dialog box consists of 4 main parts:

 

Yellow

 

Red

 

Blue

 

Purple

 

The definition of an individual node combination.

 

The data base - holding all the node combinations.

 

The parts data base - showing the available parts that can be used in the node combination.

 

The cluster data base - showing the available clusters that can be used in the node combination.

 

This template data base consists of a part number, the applicable node type, the diameter and types of the 4 pipes connected to the node. These are the necessary variables that must be defined to uniquely define all the combinations.

 

Data bases

 

 

Up to 5 different data bases can be used to save the template information. At the top left of the dialog box, select the database number you want to work with. All the template actions like teaching the template or filling the design from the template is done with the selected template database.

 

The purpose of the different databases is to separate different irrigation design types from each other and to make the application of the template functions more related to certain designs.

If this box is checked, more than one of the same part may exist on this node.

Example 1

Two lateral pipes with the same diameters connect to a manifold. If it is a micro/drip project then the 2 pipes will connect independently to the manifold each with it's own grommet. In this case same grommet part number is used twice, one for each side.

In this case the Allow duplicates should be checked.

Example 2

The same 2 pipes as in example 1 connects to the manifold in a sprinkler block design. In this case the 2 pipes are not independently connected to the manifold but are joined to a regulator which in turn connects to the manifold.

In this case the Allow duplicates should be off, allowing only a single instance of the cluster describing the connection.

 

Below is a step by step explanation on how to define a template entry.

 

Step 1 :                Use the data base control bar and click on the '+' button. This creates a new entry in the template data base.

 

Step 2 :                 Select the node type applicable to the combination.

 

Step 3 :                 Select the appropriate pipe types for the 2 connections. Also enter the pipe diameters. A zero pipe diameter will substitute for any other diameter.

                         The pipe type can be set to either the main line, manifold or lateral types. Additionally the following types are defined.

                         Any    : The pipe type may be either a main line, manifold or lateral.

                         None: Use this type to specify a node on the end of a line.

 

          EXAMPLE:  end point definition.

         

Step 4 :                Decide if a part or cluster number must be used and double click on the correct item from the selected data base displayed on the right hand side of the screen.

          or

 

Step 5 : Press the button to update the entry in the data base.

 

Note:

The Teachoption on the parts dialog box will automatically add any new combinations from the irrigation design project, to the template data base.
