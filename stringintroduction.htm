GEOMETRICAL ALIGNMENT STRINGS IN MODEL MAKER

 

Strings have been introduced as a new module in Model Maker to assist in manipulating lines (strings of data) in horizontal and vertical planes. Strings, in contrast to normal CAD lines, may consist of one or more segments which operate as a unit. The segments as defined by the nodes at the endpoints may be moved, copied, rotated, re-elevated, deleted etc. as one entity.

 

A string will have a pen, line type and layer name associated with it. These attributes may be changed over the whole string by just indicating any segment. A single string may be broken up into separate strings and different strings may be combined to form new strings.

 

You can add new nodes to an existing string. They may be added to the start of the string, in between existing nodes or at the end of the string. A string also has a starting peg distance value and the intelligence to calculate its length with peg distance values for every node.

 

Strings are generally viewed in plan mode, but the unique vertical view option easily flips any string to the mode where the peg distance and elevations are displayed. This is similar to viewing the string as a long-section.

 

All the necessary functions are available to manipulate the components of the string in the vertical mode. For example,a node can be moved to a new position by entering a new peg distance value and elevation or by graphically pointing to a new position.  Add or delete nodes while in vertical view mode. A powerful function is available to regrade the string as a whole or partly between indicated nodes to any specified slope.

 

Strings may also be regraded by the vertical alignment of any indicated base string, making it invaluable for road, ramp and intersection designs.  Strings may present themselves as horizontal alignment definitions for a network of roads, pipes or any other service detail. The vertical design data of the service is incorporated in the elevation of the string nodes, allowing the strings to determine whether services, when crossing, clash or follow freely over each other.

 

A detailed report of all the crossing services is produced along with the graphical indication of any conflicting situations.  Groups of identified strings may be used to indicate areas of volume calculations or to extract cross and long sections automatically.

 

Contours can be simulated with strings making it a powerful landscaping tool to design irregular shapes. The mining module in Model Maker makes extensive use of strings to perform complicated rehabilitation functions with great ease.

 

Strings have the added ability to generate DTM points at every node. This allows you to triangulate these new points and view the contours as defined by the strings. Changes in the vertical alignment of the string are easily translated to the newly defined DTM points and are immediately updated in the contours.

 

 

Three types of strings are defined:

 

Alignment string

 

This string is unique because the horizontal component works exactly the same as the definition of a road horizontal alignment.  The horizontal pi's are defined by pointing to the coordinate positions. Enter the curve radius and transitional length.  These horizontal pi's (plan nodes) have no elevation defined. The vertical component of the alignment string is defined in the Vert option of the string menu. This vertical component of the alignment string works independent from the plan nodes, meaning that the alignment string may consist of a different number of plan nodes and vertical nodes. Each component must have at least a start and end node (2 nodes).

 

Two vertical nodes, one at the start peg distance of the horizontal alignment and the other at the end peg distance, are defined automatically. The Z value of these initial nodes are undefined and must be updated by the user using the Vert>Vert>Z-Node option. New vertical nodes can be added between the original two nodes with the Vert>Vert>Special function.

 

General string

 

This type of string is also defined by pointing to the plan nodes.  The line defining the string will be the segments joining the plan nodes in order of input.

 

The horizontal alignment is therefore only defined by the number of plan nodes and not by pi's, curves and transitions.

 

The vertical component of this string consists of the elevation of every defined plan node. The number of plan and vertical nodes will always be the same. Additional vertical nodes are thus defined by first adding a plan node to the string. The Z value of the new node may then be entered.  The elevation of a node may be interpolated directly from an existing model.

 

Pipe string

 

This string type is very similar to the general sting type but it also allows for pipe diameters and invert elevations at the nodes.

 

Functions applied to strings

 

Strings are defined by the following means:

 

1)                Indicate the plan nodes. Use the Nearline mode to have a string follow a pre defined CAD poly line.

 

2)                Copy a string to a new position.

 

3)                Copy a string parallel to another. The new string will always be of the general type. If the base string is of an alignment type the following will apply :

 

The new general string will follow the alignment string along it's defined horizontal alignment position, taking curves and transitions into account. the user will be asked for the interval between the plan nodes of the new general string. The elevations on the new string will be interpolated from the base string.

 

The vertical component of the new string is manipulated in the Vert menu.

 

4)                Generate the toe-line string by calculating toe-points along an indicated base string at a set slope value down to the ground surface as defined by the active DTM project.

               Toe-Lines are defined without first creating a design surface with triangles.

 

Vertical menu

 

View one or more strings in the vertical mode. The ground long section below the string may be displayed with the string.  

 

You can apply the following vertical manipulations to the strings while in vertical mode.

 

1)                Regrading a segment or all the segments between indicated nodes.

 

2)                Grading a string relative to another.

 

3)                Adding a Z constant to the whole string.

 

4)                Moving Z nodes graphically up or down in space.

 

 

Strings may be used :

1)                as a substitute for a CAD poly line.

 

2)                to define design lines for parking areas, roads, platforms, landscape forms, rehabilitation etc.

 

3)                to generate new DTM points at the nodes of the strings.  (This applies only to general strings.)

         These DTM points may be manipulated by the normal menu's.

 

4)                to update the elevations of DTM points at the nodes using of the string.  This is used mainly after

         the vertical component of a string has been changed.

 

5)                to model the top and bottom part of an embankment.  Embankment hatch lines can then be drawn

         automatically between the strings.

 

Example workshop

 

1)                Remodel landscape area to new design contours.

 

Load the OGL model and triangles into project 1.

 

Switch the active project to #2.  Add general strings graphically by adding nodes along the intended new landscape contour. These strings will initially all have a 0 elevation.  Use the Vert options and elevate each string to the desired level.

 

Use the Utility option and generate new dtm points onto project #2 by pointing to each string individually. Search for triangles between points in project 2. Draw contours and calculate volumes with the Terrace option. Redesign the vertical part of the strings if necessary. Update the elevations of the DTM points along the new designed string. Use the Utility option.

 

The above could also be done by defining the strings, not as contours, but as cross-sections over the OGL model.

 

Note:

Strings are stored separately from any other data in the strings File menu.
