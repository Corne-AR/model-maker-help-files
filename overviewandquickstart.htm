MODEL-MAKER

 

The name Model Maker refers to the whole software program that include all the modules in the software suite.

There are currently 19 modules. When Model Maker is sold for irrigation design purposes, which normally consists of modules 1,2,6 and 17, it is referred to as IRRI-MAKER.

 

Irri-Maker can also be compiled to incorporate any of the other Model Maker modules.

 

Model Maker Basics

 

The program runs in windows and utilizes the graphics environment extensively.

 

The main screen consists of many different components. Please check out the main screen topic for more information.

 

Model Maker consists of 3 major sections

 

1) DTM. (Digital Terrain Modeling or Survey)

The functions associated with the DTM are:

               MAIN MENU                        : DATA (The database that holds all the survey co-ordinates).

               MAIN FUNCTIONS                : PNT & TRI (The functions which manipulates survey points on the graphics screen and create the triangular network which produces the  contours).

 

2) CAD. (Drawing functions)

               MAIN FUNCTIONS                : CAD (Find all the drawing functions on the function submenu).

 

3) IRRIGATION SYSTEMS DESIGN. (Irri-Maker)

               MAIN FUNCTIONS                : IRR (A dialog that holds all the irrigation functions).

 

 

DTM. (Digital Terrain Modeling or Survey)

 

A) Survey data

 

Depending on the system, Model Maker can hold a various amount of survey points.

Irri Maker can hold up to 9000 survey points. These points can be entered into the system by using the following methods.

 

1)                Manual field book data.                        [Working Example 2 (Enter survey data)]

2)                Digitizing data.                                [Digitize]

3)                DXF                                        [DXF - File Import]

4)                ASCII Data                                [Import - File import Define]

5)                Electronic data                                [Electronic - INPUT CO-ORDINATES]

6)                Adding points                                [Point Add]

 

1) Manual field book data

 

Entering survey data into the DTM data base requires that you click the DATA option on the Main Menu. On the Stations page, enter the co-ordinates of the first station and then click the Points page. Set the station configuration and enter the points that were surveyed from the station. If multiple stations are used it is important to calculate the correction angle of each station.

 

If you need to run a traverse between the stations in order to calculate the co-ordinates of each station. Use the Traverse function located on the Sur (survey) menu.

 

After all the survey points have been added, close the Data dialog box and plot the data on the screen. Use the Normal button to plot all the visible points.

 

2) Digitizing data

 

Information can be imported from existing plans by using a digitizing tablet. This routine requires that you first orientate the plan by digitizing 2 known co-ordinates after which Model Maker will calculate a conversion factor to reduce other digitized coordinates to the regular plotting system.

Model Maker employs an open ended communication format with any digitizer. The advantage to this is that Model Maker can access any digitizer but the user has to setup certain parameters that will allow the software to interpret the communication strings correctly.

Additionally, the Model Maker installation CD has a visual tutorial of how the digitizer works. On the CD, locate the tutorial section.

 

3) DXF

 

This is an ASCII data file produced by CAD programs to transfer drawings in electronic format.

Model Maker can both read and write DXF files. When importing a DXF file, the electronic information can either be read into the CAD system, producing the stored drawing or DTM points can be generated on all the defined CAD lines.

When you need to recreate a DTM from a DXF file consisting of contour lines, Model Maker will place survey points all along the dxf contours and if the DXF is in 3D format, the survey points will also have the correct elevations.

 

4) ASCII Data

 

Another format to transfer surveyed co-ordinates between computers is in an ASCII table format.

These files normally consist of rows of data where each row describes a DTM point by means of the points name, Y/easting, X/northing and elevation co-ordinates. This is however not a fixed format and Model Maker allows many different settings to accommodate all the various possibilities.

 

5) Electronic Data

 

Many of the newer survey instruments have the facility to capture the surveyed information in real time onto an electronic storage device. Model Maker will read this type of data directly from the file format and reduce the directions and distances to final co-ordinates. Use the Utility, Electronic data function to access these formats. See the help file for more information.

 

6) Adding points

 

Another way to add survey points into Model Maker is to use the Point function located second from the top on the main function bar. On the functions sub menu, use the ADD button to access the different methods of adding points directly on the screen. This includes adding points in free hand mode, on CAD lines and on circles.

 

B) Surface modelling

 

Most of the time survey data is entered into the system for the purpose of generating contours and to interpolate elevations at random positions. Model Maker uses a network of triangles to represent the surface between the surveyed points. The functions used to create, edit and delete triangles are found on the TRI menu on the main functions.

To create the total network of triangles select the Csearch function on the function sub menu.

 

After the triangles are generated you sometimes need to manually edit or delete triangles. See the help file topic, [MENU : Triangles] for a description of all the different options available.

 

Model Maker uses the triangles for the following purposes.

1)                To interpolate and generate the contours.

2)                To generate sections (profiles) through the model.

3)                To calculate volumes of a single or multiple surface.

4)                To interpolate new elevations at any defined coordinate when adding other dtm points, cad lines or irrigation nodes.

5)                To do slope and height analysis.

6)                To produce a 3D view of the survey site.

7)                To calculate the flat or surface area of all the triangles.

8)                To calculate relative height differences between various survey models.

 

Triangles do not need to be created if you are not going to use any of the above options.

 

CAD. (Computer Aided Drafting)

 

The CAD module is fully incorporated with the survey (DTM) and irrigation design sections of Model Maker. All the cad functions are located on the functions sub menu when the CAD button on the main functions menu is pressed.

The different elements that can be used in CAD are lines, text, circles, hatch, patterns, arrows, dimensions and bitmaps. Although CAD can exchange data to and from the survey and irrigation design sections, it can also be used as a standalone tool. This means you can use CAD to draw house plans, mechanical detail and other general drafting plans without getting involved with any of the other Model Maker functions. It is a powerful and versatile tool.

If you are already using another CAD program as your main drafting engine then Model Maker's CAD ideally complements that because of its integration with the survey and irrigation modules.  

 

To view the full layout of all CAD functions see the help file [MENU : CAD].

 

1) Lines

 

Lines are normally the element used most abundantly in a CAD drawing. Many different details like fences, roads, property borders, rivers, power lines, telephone lines and houses are usually represented by CAD lines. Model Maker allows you to draw lines with different colours and line thickness. Lines types can vary from solid lines to various types of dotted lines. The user may also define line types

 

2) Text

 

Adding text to a drawing is a very important part of the final detail. Model Maker recognizes that and uses a unique dialog menu that gives you total control over the font, size, colour, position and orientation of the text. Text elements can be copied, aligned, resized and manipulated with the greatest of ease. Pages of text, for example the report of the irrigation design results, can be imported and neatly displayed along with all the other CAD elements. Different fonts, including any installed windows font can be used.

 

3) Circles

 

All the functions necessary to add, edit, move, copy and to delete circles, are available. Similar to lines, circles can also be drawn with all the different line types and colours. Working with full circles or arcs you will find that Model Maker has all the functions available.

 

4) Hatch

 

The hatch option allows you to define any area and to fill it with either a grid of lines or with a solid colour. Use this to easily identify and draw attention to certain areas.

 

5) Patterns

 

Patterns are used to group a set of CAD elements together in a single entity. This entity can then be saved to a file and at any future stage, recalled from disk and inserted in other drawings. The pattern concept is normally used to draw certain terrain features like trees, houses, cars and other small shapes. The patterns can then be used repeatedly on different scales, sizes and orientations.

There are also functions that will automatically place a predefined list of patterns on survey points having a certain code. See the help file [Pattern - CAD : Pattern - Auto].

 

6) Arrows

 

Arrows are used to point to a defined position. The arrows can have different colours and the arrow head is user definable.

 

7) Dimensions

 

Dimensions are used to indicate the distance between 2 points or various other elements. Model Maker will calculate the distance but also allow the user to edit the final value.

 

8) Bitmaps

 

With the advent of the new ink jet plotters the manipulation and plotting of bitmaps images has become an integrated part of many CAD drawings. Up to 9 images in either BMP or JPG format can be loaded into Model Maker and plotted with all the other vector elements. The bitmaps can easily be positioned and resized allowing you to align any aerial photograph with the actual surveyed information.

 

IRRIGATION SYSTEM DESIGN (Irri-Maker)

 

The irrigation module is a comprehensive design tool to help develop irrigation projects. Operating within Model Maker, Irri-Maker integrates with the powerful CAD and survey functions to provide the most complete software package available.

 

Irri-Maker makes the design of irrigation systems a pleasure by dividing the whole process into logical steps.

The two main components are the nodes and pipes.

The system (blocks or main line) are build by placing nodes on the ground which are then connected by means of pipes.

Blocks are defined in which the sprinkler or micro/drip layout is placed and managed. Each block consists of lateral lines (connecting the emitters), the manifold line (connecting all the laterals) and valves which connects the block to the main line.  The main line transports the water from the source to the valves of all the blocks. The blocks are easily grouped to operate in different shifts.

 

Comprehensive hydraulic functions are used to size the pipes and to calculate all the pressures and flows. A complete list of parts and clusters can be defined and automatically placed on the nodes, providing a detailed list of materials with prices.

Impressive drawings of the design are produced along with CAD details and contours by means of the plotting functions provided by Model Maker.

 

To access the main irrigation dialog box click on the Irr button on the main functions panel.

 

Where the functions of the DTM and CAD are displayed on the bottom functions sub menu, all the irrigation functions are grouped together on the main irrigation dialog box. Pages with descriptive tab names are used to lead to the individual functions.

 

See the help file [Irrigation].

 

1) Project

 

On this page you can set the project units and use the LOAD and SAVE options to save the irrigation project to disk. The irrigation data is saved separately from the Survey and CAD, which are saved using the FILES option.

Because Irri-Maker is used worldwide a variety of pressure, flow and diameter units can be defined.

 

2) Blocks

 

The irrigation design is simplified by grouping different areas into separate blocks. The main line will then distribute the water from the pump node to the valves at each block. Useful functions are available to fill the defined blocks with sprinklers and pipes.

 

A) Main line (Block zero)

The main line is defined and manipulate by activating block 0. The main line must have at least one pump node (optionally 2 pumps can be used) in addition to any other nodes needed to connect the pump with the blocks. Only the main line layout may be in a loop configuration.

 

 

B) Blocks

Blocks are added to the system by indicating the block perimeter points. Up to 40 perimeter points may be used to define the block area. Once the area has been defined, use the automatic function (Fill laterals) to define the lateral pipes and sprinkler nodes. Nodes and pipes may also be added to the block using any of the other manual methods. Before using the Fill Laterals method select the correct sprinkler option by checking the sprinkler or micro/drip options on the page.

 

Each block must have one valve. This valve node is used as the source node to the block and will be used to link the block with the main line.

 

3) Nodes

 

The irrigation project consists mostly of nodes and pipes. Nodes can be sprinklers, pumps, valves or fittings.

The database option is used to enter the standard node types. The plot parameters on the nodes page are used to specify the display format of the nodes.

 

Each node can have a maximum of 4 pipes connected to it. If a DTM model is loaded in memory at the time when nodes are added to the design, the elevations of the nodes will automatically be interpolated from the triangles. Nodes are automatically allocated a zero elevation if no triangles are available.

A re-elevate option is also available to update the node elevations if a survey model is only available at a later time.

 

4) Pipes

 

Pipes are added between existing nodes. The database option is used to enter pre-defined pipe types. Pipes are defined in 3 categories namely laterals, manifold and main line pipes. The laterals may either be a regular pipe when modeling sprinklers or a CES (Constant emitter spacing) pipe which is used to simulate a micro/drip system. Manifold pipes are used to connect the lateral pipes to the valve of each block. Main line pipes are used to connect the valves to the pump. The main reason for the different pipe categories is so that Irri-Maker can apply the automatic pipe sizing functions to each specific group under user control.

 

5) Global

 

Nodes and pipes may be entered in any sequence. However, before calculations can be done, Irri- Maker will renumber the whole system by using the valve as the source node for the blocks and the pump as the source node of the main line.

The hydraulic calculations can only be performed if the main line (block 0) has a pump node and each block a valve node. The design is also checked for nodes not connected to pipes and other sections of the design not connected the source nodes.

 

The Design & Calculate option is used to apply water to the system. For the main line, the water will be entered into the system (all the nodes and pipes of the main line) through the pump node.

 

The hydraulics of each block will be calculated using the valve as the source node and distributing the water to all the down stream nodes.

Automatic sizing of the distribution pipes is also available.

The group copy, move and delete options are used to globally manipulate a section of the data.

The Report option is used to produce a Bill of Materials of the irrigation design.

 

6) Shifts

 

Irri-Maker makes extensive use of designing groups of nodes and pipes together in blocks. These blocks normally consists of sprinkler nodes, lateral pipes joining them up and are connected to the source valve by means of manifold pipes and fitting nodes.

Often the main line source pump cannot supply enough water to the whole irrigation system to simultaneously maintain the flow to all the blocks. The blocks are then designated to operate in allocated shifts. Irri-Maker allows the user to graphically assign the blocks to different shift numbers.

 

Apart from grouping sprinkler in blocks, regular valves and sprinklers can also be defined to operate directly in the main line. The purpose of this is not to eliminate or substitute the blocks but to manage certain pipe layouts not associated with blocks like wind rows between the blocks.

 

7) Pivot

 

Functions are available to assist in the positioning and preliminary design of a pivot. The span lengths, precipitation required and system efficiency may be specified. Irri-Maker will then draw the circles representing the spans in plan and allow the user to dynamically position the circles. The system area, required flow, elevation at the pivot point and elevations on the outer perimeter are reported. This function is quite useful when trying to optimize the layout of the irrigation area.

 

8) CAD

 

Most of the time the plotting of the pipes, nodes and other irrigation detail are produced by the irrigation module itself with limited control by the user. On certain occasions for example, one needs to have more control over the text sizes and positions. The irrigation design data can then be copied to CAD elements and all the rich cad editing functions can then be used to enhance the drawing.

 

9) Defaults

 

Certain plotting, editing, calculation and design defaults are user definable. You can for example draw the lateral, manifold and main line pipes in different line widths, set the blocks to draw in predefined shift colours, use the Hazen-Williams or Darcy-Weisbach friction formulas, etc.

 

10) Parts

 

Assigning specific parts to all the various nodes can be quite a challenging task. Irri-Maker uses 2 different methods to assist in the itemizing of the irrigation project.

 

A) Automatic Reporting

 

With this function a theoretical report is automatically produced. In this report all the nodes are analyzed and reported according to the number of pipes connected to each node. Nodes are then categorized as:

         End piece                        : Only one pipe connected.

         Reducer                                : 2 Pipes of the same or different sizes are connected.

         T Piece                                : 3 Pipes are connected to the node.

         Cross                                : 4 Pipes are connected to the node.

 

The nodes are also grouped together according to the node type. The final printout will thus list all the nodes of the same type, having the same number and diameter of pipes connected to it.

 

B) Detailed bill of materials

 

This method is more involved and requires some assistance from the user. The advantage is that you will get a complete list of part/cluster numbers and prices. A database of all the different parts i.e.: T pieces, elbows, reducers etc. must be compiled. Additionally you must teach Irri-Maker which parts will fit on what type of node and pipe configurations. This requires some effort to setup but the benefit of the system analyzing itself can be very beneficial.

 

SUMMARY

 

The above descriptions of the various functions of Irri-Maker is just a small peek into the total capabilities of the system. Depending on the type of results you want from the program you can either go deeper into all the functions or just use the few commands necessary to achieve your objective. Irri-Maker has been developed with a wide range of possible users in mind and has  proven itself during the years.
