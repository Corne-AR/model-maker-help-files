Link manufacturers databases

 

This dialog box allows you to:

 

A. Define a manufacturer name (OEM).

B. Link the name to an existing database that defines the span info.

C. Optionally create a new database that can by populated with span info from scratch.

 

Database definition.

 

1.A DBF database file format is used.
2.The database will have records describing each span for that OEM.
3.The minimum number of fields in the dbf is 7 and they need to be named: Model,Name,Diameter,Length,Outlets,Nominal and Type.

 

How to obtain or create such a database.

 

As indicated above, there need to be at least 7 fields in the dbf file.

 

A) The OEM span database files used by Senninger's Sennpaq program has more that 200 fields (which are used by Sennpaq) but it also has the seven fields required by MM. Those Sennpaq DBF file can thus be used without any changes.

   Note: The Sennpaq dbf file are in English units. The column (3) need to say NO when they are used.

 

B) If you do not use Sennpaq and want to start your own DBF file. Use the (5) (Create new database) and MM will create a DBF file with the 7 fields that you can use to enter (type) your own span details.

   Once the basic file is created, enter a (1) Manufacturer name in a new line and use (4) (Link existing) to link your Manufacturer name to the database.

   Remember to set column (3) to YES if you define your own database in metric units.

To create a link, do the the following:

 

1) Select a blank line and enter the OEM name in the manufacturer column.

 

4) Click on Link existing to select a DBF file that goes with that oem.

 

The default DBF file is the ones used by Senninger Irrigation's SennPaq program. If you are using SennPaq and already have span databases, just point to that dbf file to use when linking.

 

5) You may also use the Create New to create a simple dbf file that you can populate yourself.

 

2) The linked file name will be displayed in the DBF column.

 

3) Click on the this column to select between YES and NO. Select YES if the DBF is defined in metric units.

   Select NO if the dbf is in Feet. MM will then, if your design is in meters, translate the feet values in the dbf to metric.

 

6) Click on this tab to edit the selected DBF file. You would normally not edit files linked to SennPaq but if you created your own DBF file with (5), use this tab to define the span lengths.

 

Note. At present MM only uses the length values. The other attributes are only for informational purposes but may formally be used in future versions. Although MM does not use the other fields now (and can be left blank), you could populate them (at least the Model and Name fields) to better identify the spans in future.

 

As indicated, the length is specifically required.

 

The pipe diameter is optional. If the pipe diameter has a value other than 0, then that value will also be used to define the pipe diameters in the pivot.

Important. MM will only use the database pipe diameters if all the spans of the pivot have defined diameters. If any span has a zero value then you need to manually define the diameters in the pipe diameters function.

 

The Type is optional but if a span with a O (overhang) type is used as last span, then MM will assign that span length as the overhang.

 

 

 

 

 

Entering data in the DBF in either Metric or English units.

 

1) Model description.

 

2) Span Name. (ID)

 

3) Pipe Diameter is in mm (Metric) or inches (English)

 

4) Span Length is in meter or feet.

 

5) Number of outlets on the span.

 

6) Nominal Outlet Spacing is in meter or inches.

 

7) Span type.  Any character to define the type.  Normally an O (for overhang)  and I (for intermediate).

   If the type is defined as O then MM will automatically assign the length to the overhang value when used as last span in the list.

 

8) The database filename being edited.
