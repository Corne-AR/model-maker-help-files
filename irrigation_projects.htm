MENU : Irr - Projects

 

Up until version 6.11, Model Maker was only able to work with one irrigation project at a time. In version 6.12 a new multi project option has been added to the irrigation functions.

 

The functions on the PROJECTS page can be used to create and manage multiple irrigation projects.

 

The purpose of the multiple project option is to allow the irrigation designer to load alternatives of the same design or look at different

sections of a bigger agricultural area.

The advantage of this is that Model Maker will plot all of the projects simultaneously on the screen and plotter.

 

Each project is independent from the others and works in it's own hydraulic system.

 

Background information.

 

·You may ignore the multi projects feature if you are not working with more than one irrigation project.
·By default the projects list will always have at least one project.
·If you load older irrigation designs, the design info will always go into project one.
·

 

Saving and loading projects.

 

 

 

 

 

 

 

When using the regular File Save/Load options, all the projects are automatically saved and loaded in the MIR or MAL file format.

 

Opening a MIR file and set it to ADD the file, the projects in the file is added to the current design projects.

 

 

 

 

To save and load separate irrigation projects, use the Irrigation File functions.

 

Useful hints.
·Clicking on the green projects button will open the Projects Tab

 

·If the projects tab is open, the Zoom option at the bottom will zoom to the area of the current projects.

 

·If the tab is not on Projects (Nodes, Pipes etc) then the Zoom option zooms to the current block.

 

·There is no hydraulic connection between the different irrigation projects.

How to access projects 2 to 250.

·The project list can hold 250 different projects. To activate a project, just click on the list number.
·If another project number is selected, all the other functions on the irrigation dialog box then refer to the selected project.
·The active project number is shown in the green info box.
·The project list shows the number of blocks and the total number of nodes in the project.

 

 

Display multiple projects        

·Only active project - all the other projects will not show on the plot.
·All the projects - shown in their regular colors.
·Others in gray - The active project in the regular colors, all the others in a dotted gray color.

 

 

Project Functions

·Use the Copy project to make an exact copy of a project into another project.If the selected new project number is empty, an exact copy of the current project will be made. If the selected destination project already has irrigation data, the current project will be added to the destination project. Adding one project to another works like this: The contents of block 0, the mainline, is added to block 0 of the destination project. The other blocks will be added as new blocks. If the original project has 5 blocks and the destination project has 10 blocks, the destination project will end up with it's original mainline items PLUS what was in the mainline of the source project. It will also have 15 blocks. It's own 10 blocks plus the 5 blocks from the original project.

       

·Use the delete project to completely remove the active project.
·Use the Move YX option to move all the elements of a project to a new position.

 

FILE - NEW        Here you can delete all the irrigation projects or a single project

 

FILE - OPEN        A MIR file may be opened and the contents can be added to the projects or the current projects can be replaced by the design from the file.

 

FILE - SAVE        All the projects can be saved or only the active project can be saved.
