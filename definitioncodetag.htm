DEFINITION : Code & Tag

 Top  Previous  Next

 

The code & tag refer to the 2 description values of  a DTM point. The maximum length is 6 characters. Normally, use the code that describes or names the point, eg., TREE, MH, ROAD, FENCE.

 

The name of a survey station is also placed in the code field of the point.

 

The tag is not intensively manipulated by Model Maker but may be used for many other purposes.

 

You may identify all the points of a given survey. For example, in a monthly survey of a quarry that uses all the codes to identify top and bottom of bank, you can change the tags of all the points to the date of the survey. Then, do the same the next month. You will be able to identify the survey date of a given area by looking at the tags of the points. The tags can also be treated as a secondary code.

 

The only fixed code values implemented by Model Maker are:

1)                '01', this code by default identifies a field spot which has no special status.

2)                Any code starting with a '.',  eg., '.Rock', means that this point should not be used in the triangle model. Use the '.' in front of the code when you know that the point's elevation is

         not related to the ground and should not be used to create contours (such as points on a water tower).

3)                The code '*.*' means that the point is marked as deleted and will not be plotted or triangles created to it. Points are often marked and not physically removed from the data base

         in order to keep the data base sequence intact so the model fits correctly.
