MENU : sUr - Convert

 

This option will take all the points of the active project and convert the co-ordinates from one system to another. The points must all be in a co-ordinate format. This means they must belong to a 'CO-ORD' station and not a tachy station.

 

 

 

Your current system is set on the Top Menu -> Settings -> Configuration Page.

 

 

 

 1 

The current system you are working in. All Datum, Coordinate, Central Meridian or Zone  changes are now made in the Configuration window, allowing it to be saved along with each project.

 

 

 2 

The system that to data must be translated to.

Datum, Coordinate system en Cm/Zone becomes the new system settings if Convert output to editor only is not checked.

 

 3 

The data types that must be converted. Check all the appropriate options.

 

 

 4 

Press the convert button to perform the translation.

A report reflecting the status of the translation will be displayed after completion.

 

 5 

If this box is checked, Model Maker will not alter the data base of the survey points in any way, only the result of the converted co-ordinates will be outputted to the editor. From the editor you may then save the coordinates in ASCII format or print out the result.

 

Notice that when the output to editor is checked, only the DTM points will be converted and the option to convert any of the other formats, CAD, Strings etc. are deactivated.

 

 

 

Clarke / WGS84

This module converts plane (Gauss Conform, or "Lo") co-ordinates between the old (Cape) and new (WGS84) South African datum. The conversion is accomplished using a correction grid with a mesh size of 0.01degrees.

The following steps are carried out.

 

 

The module is entered with the Cape datum Lo co-ordinates (y,x), together with the longitude of the central meridian (an odd degree value). These data are converted to latitude and longitude (f, l), and corrections df and dl are interpolated linearly from the correction grid and applied to the Cape datum values. The corrected (f, l), now on the WGS84 datum, are converted to WGS84 Lo co-ordinates (y,x). For this step the appropriate central meridian longitude is needed (generally, this will be the same as that for the Cape datum).

 

 

This conversion can proceed in either direction (Cape to WGS84; or WGS84 to Cape), and the module can be used to transform from one Lo panel to another, on the same datum (e.g. from Lo 25 to Lo 27 on the Cape datum). The correction grid is supplied in the form of a binary data file one file per degree square. The user should ensure that the data fall within the area covered by the available correction grids.

If a point falls outside the grid, a warning message will be generated.

 

 

 

GEO / Clarke

You may also convert geographical coordinates to and from the Clarke system. Geographical co-ordinates on the Clarke system, can not be converted directly to the WGS84 system.

 

Note: Whenever data is converted the respective data bases are automatically updated. The converted elements, DTM points, CAD strings etc. will, after the conversion, plot on the screen using the newly assigned co-ordinates. Using the regular file save functions will then save the data with the new co-ordinates.

 

 

 

 

The facility to import geographical data and convert it directly to YX co-ordinates as it is added to Model Maker's internal data base structure, can be found in the Utility,Electronicmenu.

 

 

Note:

The geographical angles - Latitude and Longitude - must be written in the form : degminsec.decimalseconds

 

Example  GEO Angle 27deg25'30".12 is entered as 272530.12

 

       This is different to the normal angle format that Model Maker uses where the decimal dot is between the degrees and minutes.

 

EXAMPLE:

To load GEO data from an ASCII file and convert it to the Clarke format, do the following.

 

         Use the File, Import, Define format option and load the GEO data.

         Load the Latitude as the X co-ordinate and the Longitude as the Y co-ordinate.

         

         EXAMPLE: how to load Lat and Long. data.

         

         Note that Model Maker does not really know it is now working with GEO co-ordinates. The co-ordinates are treated as Y and X values.

 

         Use the Survey, Convert option and convert the data from GEO to LO.

 

Hint:

Before you start converting co-ordinates on a large scale, first run a test on a sample of data and compare the results to that of other programs or manual calculations. In general, be vigilant when converting data and always compare the results to another framework of reference.
