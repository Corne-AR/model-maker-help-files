LAYOUT

 

These functions will automatically fill the block with nodes and pipes using the specified parameters.

 

Note

The Info page is not active until a block has been filled with laterals. If laterals have been defined, the info page will indicate the block type, i.e. Sprinklers or Mirco.

Also, the layout parameters can only be inspected and not changed once laterals are defined. All the nodes and pipes of a block must first be deleted if a new layout is to be defined.

Emitter Type

Model Maker can handle both blocks with regular sprinklers or blocks with drip or micro emitters.

 

Sprinkler:

In this mode, the block will be filled at the selected intervals with the selected sprinkler node type. Each sprinkler will be represented by an actual node and regular lateral pipes will convey the water between the nodes. Water will only be emitted from the system at the defined sprinkler nodes.  Because Model Maker can accommodate a maximum of 20000 nodes per block, this type of layout is mostly used with large rotating sprinkler heads that are spaced some distance apart.

 

Stagger

 

Select if the nodes must be placed at a regular square pattern, i.e., no stagger, or if the nodes must be staggered along the rows. This option is only available when filling the block with regular sprinklers. If the micro/drip option is used, the stagger is replaced by the short stop option.

Micro/Drip:

 

 

In this mode, each emitter will not be represented by individual nodes. A single emitter type and design pressure will be defined for the whole block. The lateral pipes will intelligently simulate the position of the defined emitters all along the length of the pipe by means of the CES (Constant Emitter Spacing) value. With this method, the only actual nodes in the block will be the fittings that are used to define connections between pipes. Normally no sprinkler nodes will be used.

 

When placing the lateral pipe, Model Maker initially assumes that the ground profile is a straight line between the 2 end nodes. The following check is then performed to determine how undulated the surface is and if more nodes are necessary to simulate the ground elevations more accurately.

 

                         

The lateral is originally drawn between the 2 end points A and B. The elevation of the ground midway (C) between the 2 terminals is then calculated and checked against the elevation of the lateral at that point. If the height difference between the pipe and ground at point C is more than 1 meter, an additional node (fitting) will be added to the lateral.

 

                         

Model Maker will further inspect the lateral at positions D, midway between A and C and at position E, midway between C and B. If the difference at these positions are more than 1 meter (3ft), additional nodes will be added. A maximum of 5 nodes may thus define a lateral pipe when Model Maker adds it to the block in the automatic layout mode.

 

When the hydraulics of the lateral is calculated, it will be done based on the position and elevations of the nodes defining the lateral. Model Maker will assume a straight pipe line between the nodes. If the terrain is so undulated that it cannot adequately be defined by the above method, you will have to introduce more nodes manually by using the node and pipe functions.

 

On the defaults page you will find the function 'Profile Check' that allows you to test the difference between the elevations of the pipes and the ground surface.

 

Short Stop

When using micro laterals lines, this value determines the distance that the lateral lines must end before the block boundary.

Direction

Select the orientation angle at which the emitters must be laid out by typing a desired direction angle.

Although the direction can be specified before the actual layout is done, it is often easier to use the rotation shortcuts to rotate the whole layout when it is placed.

Fill Laterals

By default, Model Maker will assume you want to enter sprinklers and lateral pipes. You may override the default selection.

Note, when using Micro/drip layouts, Model Maker will ask you to define 2 node types.

One will be the sprinkler type that will be used globally as emitter on defined lateral pipes.

Secondly a fitting node type must be selected that will be used to define the end nodes of the lateral pipes.

A small square of the defined grid layout will be attached as a ghost image to the cursor. Move the cursor around, inspect the proposed layout and set the origin on a selected position.

 

 

The following are useful functions that can be performed while the ghost layout is moved around.

1) Change the cursor snap mode.

Note the purpose of the free and border setting.

 

Free : In this mode the user has to align and orientate the ghost and ultimately the final layout.

Border : Use this option to quickly align the layout relative to one of the block border segments. The laterals will run parallel to the indicated border. This option is mostly used when drip/micro irrigation is defined. When indicating the block border segment to be used as base, Irri Maker makes the following decision.

 

 

 

In block A the base segment was indicated by placing the cursor to the outside of the block area. This results in the first lateral being placed away from the segment at row spacing distance.

 

In block B the base element is indicated by keeping the cursor to the inside of the block area. This results in the first lateral being defined on the boundary and the second lateral then at row spacing away from the first.

 

Note

The layout difference between a block with regular sprinklers and a micro system.

Sprinklers - Nodes with the selected sprinkler type will be placed at the desired intervals.

Regular lateral pipes will connect the nodes.

 

       Micro/Drip - Nodes will only be placed at the lateral end points. (Short stop distance from block boundary). These lateral pipes will inherently know what the Constant Emitter Spacing is and will for hydraulic purposes simulate emitters (as defined for the whole block) all along the pipe.

 

If the layout is approved, you will be given the opportunity to define the manifold pipes by means of the Cut Pipe function. The input valve may also be defined. Even if the manifold pipes and valve are not defined at this stage, you may at any later stage do so by means of the manual node and pipe functions. In any case, the network must be continuously linked to the source node before hydraulic calculations can be performed.

Spacing

 

Enter the distance between the emitters and also the distance between the rows. If the block is defined as a micro/drip layout, the emitter spacing also relates to the distance between the small nozzles.

         

Please note that the values entered for the emitter and row spacings are used when the initial layout is placed. When the layout is changed afterwards by using commands like move node, move pipe and others, the initial spacing values are not updated and will still reflect the original values as the original values were only used to make the layout and do not continuously control the layout.

 

Likewise, when doing a drip or micro block design, the emitter spacing value is used to define the CES value for the lateral pipes. If the CES value of all or some of the lateral pipes are changed by using the edit single or edit type command the emitter spacing will still reflect the original inputted value.

 

Summary : The spacing values are just used to make the original layout and are not updated nor have any effect on the layout when the CES, nodes or pipes are changed afterwards.

 

 

 

 

 

 

 

2 CAD Lines

 

 

The 2 CAD Line stagger option (1) is used to define the lateral and tree row direction based on two CAD lines. The emitter spacing is derived from the intersections of offsets parallel to each CAD line. The emitter spacing is therefore not entered. Only the lateral spacing (2) is defined.

 

To start, define a CAD block as usual.

 

Draw two CAD lines to define the lateral position and direction and also the diagonal planting direction.

 

Note: The CAD lines should be drawn inside the irrigation block. The line positions are important because the lateral position and emitter positions will be based on these CAD lines. These lines do not need to touch each other. IrriMaker will automatically calculate the first emitter where the two lines intersect.

 

After drawing in the 2 CAD lines, go back to the Irrigation blocks page. Click the button to start the placement of the laterals.

 

Select your emitter and pipe combination then select the first CAD line, then immediately after that, indicate the second CAD line

 

Enter the required tree spacing.
