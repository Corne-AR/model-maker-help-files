MODEL MAKER POWER LINES

 

Model Maker assists in the designing of rural power lines by using the strings module.

 

Modules necessary to use this function are -         Modules 1,2,6,11,15

 

The strings module manipulates data like long sections and other line applications in both plan and vertical mode, making it ideal for power line design.

 

Approach to line design

 

The elements needed to design power lines are:

 

1) The ground data to produce the long section as a starting basis for the power line.

2) The design string holding the plan positions and design elevations of the poles. (nodes)

3) The dialog box holding the definition of the pole types.

4) The SPT file with the conductor characteristics

 

1) Ground data

The DTM data that will create an original ground profile must be in the active project at the time that the power line is designed. This data can be loaded into the data base through any of the data input means and can consist of surveyed data or imported ASCII co-ordinates.

 

This model may be a strip survey or a single line survey exactly on, or within a reasonable distance from, the center line.

If the surveyed model consists of a single line of points along the alignment of the profile, you do not need to create triangles on the model. When the profile is displayed in section format, Model Maker will extract a temporary section from the DTM points by inspecting the alignment and locate the DTM points that are within the selected strip width from the center line. Intersections with the center line will be calculated and chainage distances determined.

 

2) Line design

After the survey data is modeled as a DTM in Model Maker, the powerline option will use the string module to design the line.

Strings have many different uses. To identify the string as a rural power line you need to adhere to the following.

The layer name of the string must be in the format:                                ESKOM_*****   or  PWR_*******

         The ***** stands for the different types of line. ie.

         FOX MINK HARE FIR PINE & OAK.

         A fox line will thus have an ESKOM_FOX layer name or PWR_FOX layer name.

         The layer name is defined in the default pen menu or may be changed in the GXT INFO menu.

 

3) Poles

The poles are defined using the Edit Pole Database option.

 

 

The POWERL folder are the standard folder where the default pole, shortlist and SPT files are. If there are any subfolders inside POWERL, they will be displayed as options so the MM can read the pole, shortlist and SPT out of them.

There are no limit on the amount of sub folders. Each subfolder should contain complete pole information.

 

When adding a pole to the database, the first entry is the pole name.

This may be 10 characters long and is used to identify the pole.

Plan and vertical pattern names may be defined. These present the pole in plan mode and in vertical mode.

Define the pole patterns in the normal manner in the CAD.

After defining the pole, enter the maximum wind span, weight span and attachment height values for each type of line.

 

4) SPT data

The conductor types and their controlling SPT files are defined using the Edit Conductor Database option.

 

Step by step

After the DTM data is loaded, use the mining  string. Use the Manipulate Lines tab in the mining dialog box. Use pen&linetype button to set the layer name of the new string to correct format.

 

Enter the string. First define the start position, turn (deflection) points and end position of the power line. The string must be a General string.

It is advisable that DTM points already exist at the positions where the initial string nodes are placed. Then the Tacpoint cursor snap mode can be used and the string will get its startup elevations from the surveyed points.

 

A note on string node names

The node name identifies the type of pole that it represents.

The node name and pole name as defined in the list of poles in the default Pole menu, are the same.

The ! character is used to identify a fixed (strain) pole.   ie.  !11wood.

Any one of the defined poles may be used as either an angle, strain or suspension pole. Putting the ! in front of the name makes it a fixed pole.

The EL (Equivalent Length) is calculated between the strain poles. Model Maker assumes that the first and last pole of the line are strain poles.

 

After the line is defined in plan with nodes for the start, end and any other horizontal deflection points, use the Vert option to view the string as a vertical long-section..

The ground line may be viewed with the design line(string). If triangles exist in the DTM model, Model Maker will automatically extract a ground long section from the model on the alignment as defined by the string.

If no triangles exist Model Maker uses the surveyed points as they are and temporarily produces a ground long section by calculating chainages for every DTM point along the string within the user specified strip width.

 

The design string will be plotted on the height as defined by the elevations of the nodes. The node will have a zero elevation if none was defined at the design stage.

 

Use the Special Power functions to plant the poles

The elevation of every node of the design line will be changed based on the following.

1) The elevation on the ground line at the specific chainage is calculated. The bottom attachment height for the pole and line type is taken from the database and added to the ground elevation.

The string is thus placed along the bottom attachment heights of the poles.

2) The node elevation will not be updated if no ground line exists at the specific chainage.

 

Adding more poles

Other poles may now be added between the fixed poles. Use the add node function to add individual poles or use the multiple add function in the Power menu. Use the pole info option to check the weights and names of the poles.

 

Use the tower loading option to print a standard line report. Model Maker will check all the spans and weights of each pole against the maximum values for the particular line type and alert you if any maximum value is exceeded.

 

The low points option will graphically display the lowest point on each span. Use the dump graphics option to produce a picture of the line on a printer. Use the Copy to CAD function to produce a detailed drawing of the long section in Model Maker CAD.

 

SPT files

The parabolic shape of the power line is determined by the standard profiling templates as specified in the SPT files. These files reside in the subdirectory. (The directory where the program is installed)

Model Maker will automatically create the POWER subdirectory the first time it is run and the Power Line module is active.

 

Create Up to 40 different SPT files. One each for every different type of line

Crossing services

Define any other services like roads, telephone lines, etc , crossing the design line with additional strings. The name of the crossing string identifies the type of service. To obtain the correct elevation between the main string and the crossing service string, the service string must be defined with the correct elevations at the node points. Normally the tacpoint option will be used for the string to get the elevation from the surveyed point.

 

Model Maker can calculate the chainages of these crossings and display it on the vertical section. Use the Name function on the power menu to display the service name.

 

If service data is displayed on the section, Model Maker will add another option on the Power menu called Service. This option will produce a report of all the chainages and relevant elevations at the service crossings.

 

Save the string data with the files option in the GXT menu or FILE - SAVE .

Set the minimum clearance value. A warning will show if the clearance is lower.

The patterns can be shown in full which could slow down the plotting or as quick lines only.

 

General defaults:

 

The starting pole number refers to any option where the pole numbers must be displayed. Notice that pole number is different to pole (node) names which is user definable when the nodes are defined.

 

The default folder where MM will look for the pattern names.

 

Model Maker checks for crossing strings and treat them as services. Here you can specify single DTM points to also act as crossing services.

 

When plotting the profile in vertical mode, select if both the hot and cold profiles must be shown.
