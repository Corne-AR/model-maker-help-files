Copy the grid contents to the cad as cad text in a list.

 

                         

 

 

 

A) There are 2 ways to copy the grid to cad.

    Quick copy copies each grid row of text to cad as regular text. The row height and column spacing are automatically controlled by the default cad text parameters.

    Pattern Copy. This is a more comprehensive method and uses a pre-defined Cad Pattern to specify the layout of the lists.

Please see below the complete explanation of how the pattern must be defined.

 

B) The name of the pattern to be used as a framework. Use the Select option to load the file.

 

C) Fixed Columns : The pattern is static horizontally and does not change column size. Only grows vertically.

   Variable Columns : The pattern is defined to change column sizes and also the height.

 

D) For variable columns this is the definition of the column headers and sizes.

 

E) When the text in the grid rows is copied to the cad, specify if you want lines or spaces between the list rows. Also set at how many rows must the line or space be repeated.

 

F) All the grid rows can be placed in a single cad list or the list can be limited to a maximum number of rows. If a limit is specified then a new list to the right of the previous will be generated each time.

Specify the distance in millimeters between each subsequent list.

 

The elements on the lists can all be placed on one cad block or each list may be on its own block.

If all are on one block then all the lists can be moved as one entity. If each list is on a separate block then you can easily rearrange them individually.

 

G) The pattern may have text entities on layers ?Header1,2,3  These text entities will be replaced by the text in (G)

 

 

If you do not need vertical lines or a bottom frame line then D and E is not needed.

 

Lines on layer ?VERTICAL can be of any length in the pattern, the list will grow them accordingly.

Line ?LASTLINE should be the width of the list but can be at any vertical position, the list will move it to the bottom.

 

 

 

FIXED COLUMN LIST

 

Here is how a pattern must be defined to correctly manage the list layout.

 

The pattern can be defined with any number of text or lines. However, entities on certain layers will be used to format the list correctly.

 

In the example, the 3 header text and 5 column headers are just regular text and not required. They can be on any layer or absent in total.

The first 3 horizontal yellow lines are also optional. They are present in this example just to show how the user can set up his own header info for a list.

 

A) This horizontal line is required and must be on a layer called ?LINE0  The line serves as the starting point for the list entries and will normally run from the left to the right side of the list.

 

B) This line must be on layer ?LINE1 and is below ?LINE0. The vertical distance between the lines is used to determine the spacing between the list rows.

 

C) This is a line of text and will serve as blueprint for the column text. The sample text must be on layers called ?A  ?B

?C etc.  When the grid text is copied to the cad list, the sample text will be replaced by the first row of the grid and the following rows will create new entries in the list using the same text attributes.

When defining the pattern the user therefore has full control of the lorg, pen, font and size of each column text. By default, the sample text also defines the spacing between the columns.

 

D) These are vertical lines and will grow longer or shorter depending on the height of the list. For a vertical line to adjust its length to that of the list, place the line on a layer called ?VERTICAL

These lines must also be defined from top to bottom.  The second point (bottom) of the line will grow according to the list size.

 

E) This is the bottom line of the frame and must be on layer ?LASTLINE and all ?VERTICAL lines will be trimmed to it.

 

Summary:

elements for A,B,C must be present in the pattern.

The 2 lines on ?LINE0 and ?LINE1 are required and so is the text on layers ?A  ?B  etc.

Without those 2 lines the list would not know what the row spacing must be.

Without the first row of sample text, the list would not know where the grid columns must be and also not what the text attributes are.

 

 

 

 

 

VARIABLE COLUMNS

 

This format requires a pattern with some additional information.

 

Column Definition:

This is a user defined string describing the column header text and column width in millimeters.

eg   A=CODE+15

Relates to column A,  the header of this column is CODE and is 15mm wide

The other columns are defined in the same way.

Note there is a , (comma) between the definitions in the string.

 

For the columns to change width and header text, the layer of vertical lines must be defined:

 

A) left line of the frame on layer   ?VERTICAL_LEFT

 

B) lines between the columns on layer ?VERTICAL_A  ?VERTICAL_B  etc

 

C) right line of the frame on layer   ?VERTICAL_RIGHT

 

D) the header text to be replaced by the header definition in the column definition string must be on layer ?H_A  ?H_B etc.

 

E) text replaced by headers 1 to 3 must be on ?HEADER1 ?HEADER2 and ?HEADER3

 

The actual list text must be on layer ?A  ?B  etc.

 

Summary:

Define a pattern with frame lines on layers ?VERTICAL_LEFT  ?VERTICAL_RIGHT and ?LASTLINE

Define columns with header text on ?H_A ?H_B etc and divider lines on ?VERTICAL_A etc.

Define one row of text (sample column text) on layers ?A  ?B etc.

Define header text if needed.

 

The column widths and headers are defined by the Column definition string. The total width of the list is calculated and adjusted by the number of columns used and their widths.

The column text and headers are spaced accordingly.
