MENU : Irr - Nodes - Database

 

 

This is a comprehensive database and editing system that allows you to enter and maintain all the node types used by the irrigation project.

 

Navigating the dialog

It is important to understand the layout of the dialog box to make full use of the functions.

 

The data base dialog box consists of several pages defining the different node types.

The first page is called SERIES and holds the descriptions of all the different series names of all the suppliers.

Note that the series page works in conjunction with the node type selected in the bottom panel.

Valve

Enter the description name and pattern name for each valve. Again, double click on the Part Number column to access the parts data base. The selected part number will also define the cost.

 

Valves have the following purposes.

1) It is used as the source node of defined blocks. In this case the valve will hold the flow demand and pressure of the block as determined when the hydraulic calculations were performed. The valve flow and pressure are thus not directly manipulated by the user. Use the normal shift functions to operate the sequencing of the block valves.

 

2) Valves may be added as regular valve nodes to the main line and the flow and pressure of each can then be entered directly by the user. Use a valve in thus configuration when you want to simulate a pivot or any other big take off position. Valves in this format are not scheduled in the same way as that of the block valves and must manually be switched on and off.

Fitting

Fittings are used as a neutral node type to allow pipes to be connected. Like the valve nodes you must enter the description and optional patterns name. If you want a cost associated with the fitting, double click the part number column and select a part number.

Secondary Loss - K factor.

 

 

Regulator

Use this type of node to define a pressure regulator.

 

The purpose of a pressure regulator is to reduce the pressure at the node to the specified maximum. You can also select that the reduced pressure should only be applied to certain pipe types. Like in all the other node types, enter the description, patterns name and double click to set the part number and price.

 

In the Regulate Pressure column enter the maximum pressure. When the hydraulics are calculated the pressure at all the nodes are determined. If the output pressure (the pressure at the start of the pipes carrying the water away from the pressure regulator) is higher than the maximum, only the maximum specified value will be allocated to the out flowing pipes.

 

Note:

You can select that only certain pipes should be affected.

 

 

In the Pipe Type column enter the code of the pipe type to regulate.

EXAMPLE: manifold and lateral layout.

In this example a pressure regulator is defined at point A. The water feed into point A is from point X. In this layout points B, point C and points D are all downstream of A and can be effected when the pressure at A is regulated.

Setting A to regulate all the pipes have the effect that the reduced pressure will be carried over to both B points and to C.

Setting A to regulate only the lateral pipes will only effect nodes B (and others on the same laterals) as those are the 2 lateral pipes connected to A.

Setting A to only regulate manifold pipes will only have an effect on node C. Because nodes D and others downstream of C are effected by the pressure at C, their pressures will also decrease.

Pump

After defining the pump series name, proceed to the page and enter the specific attributes of each pump.

 

Enter for each pump the following:

 

Description :

Enter a definition of the pump. This description will be used to describe the pump when editing single nodes or when a bill of materials is produced.

 

Pattern Name:

Type the name of the pattern  that should be used to represent the pump when the pattern option is selected. You may leave this input blank if you do not require the pumps to be plotted with different pattern shapes.

 

Operating pressure and flow :

Four pressure and flow values should be entered. These values are used in the following 2 instances.

1) When doing hydraulic calculations and the Overload Pump warning is selected a warning will be posted if the required flow or pressure exceeds the maximum pump capacity.

 

2) When sizing the main line pipe and the economic options are used, Irri Maker must have a pump pressure range in order to determine the best solution.

 

Part Number :

The correct method to enter the part number is to double click on the part number column.  The parts dialog box will be opened allowing you to select a part or cluster from the data base.

 

Cost :

The cost of the pump is not directly entered here. The cost is calculated from the part/cluster number that is selected from the parts data base.

 

Pumps can only be added to the main line. The main line can either have one or more pumps.

Emitter

After defining the sprinkler series name, proceed to the page and enter the specific attributes of each sprinkler type.

Sprinklers have the largest database selection and are defined by model name, nozzle, flow and pressure.

 

Model Names

 

Use the Manage series button to define and edit model names.

After a new model name is defined you may double click the part number column to access the parts data base. From the parts data base, select the specific part number that must be used when defining the node. The cost will automatically be taken from the select part number and added in the cost column.

Note:

By deleting an existing model name, all the nozzles associated with it will also be removed from the database.

 

Highlight the required model name to view or edit the associated nozzles.

 

Nozzle Names

 

After selecting a model name, nozzle data may be added to it. Again, use the

Manage Nozzles

button to manipulate the nozzle names.

 

Nozzle data consists of the description and the optional pattern name. The pattern name defines a cad picture that can be used to represent the sprinkler as a node on the graphics drawing.

 

Node colour

To define the colour of the circle that will draw the sprinkler, double click on the selected nozzle type. The pen colour box will be displayed. Select the desired colours.

 

Sprinkler pressures

 

After selecting the model name and nozzle type, enter the applicable range values under which the sprinkler normally operates. Note that the selected units are defined on the main Series window.

When a sprinkler is added to a design, Irri Maker allows you to select the operating pressure from the defined pressure range.

When the hydraulics is calculated, Irri Maker will determine what the available pressure at each node is. This data base with the defined pressure range for the specific sprinkler will then be checked and the flow corresponding to the available pressure, will be determined.

Simulating a booster pump (old method)

Please see the new BOOSTER PUMP node type to specify an actual booster pump. (Model Maker version 6.0)

 

In the past Irri Maker did not make provision for a booster pump node.

 

A booster pump can however be simulated by using a regulator (other) node and set it's regulating pressure value to a negative value.

 

When the hydraulics are calculated and the inlet pressure of the booster node is less van the (positive) value of the booster regulator pressure, the outlet pressure will be increased to the set value.

 

Please see the new BOOSTER PUMP node type.

Booster Pumps

 

Previous versions of Irri Maker (before version6) did not make provision for booster pumps. See explanation above.

 

The new Booster Pump node type allows you to place booster pumps in the mainline design.

 

Like other node type, boosters nodes have fields for the Part No., Description and pattern name.

 

The important field is the booster pressure and if it is a Relative or Absolute booster.

 

Absolute boosters: If the pressure in the system at the entrance of the booster pump is less than the booster pressure value, the system pressure will be set to that of the booster pump when exiting the booster.

 

Note: The older simulate method (described above) used this approach.

 

Relative boosters: The booster pressure will be ADDED to that of the inlet system pressure. The pressure out of the booster is the sum of the inlet + booster pressure.
