EXAMPLE : Working Example 3 (Horizontal alignment)

 Top  Previous  Next

 

MENU : Hor

 

Object

 

This example shows you how to create the horizontal alignment on which to extract sections and then to extract the long- and cross-sections.

 

You will need to know the position of all horizontal points of an intersection (PI) on the horizontal alignment and you must have data and a triangle model loaded.

Model Maker extracts sections from DTM points even if no triangles are formed. This is usually the situation when a strip survey is taken and consists of a single row of points exactly on the alignment.

 

Procedure

You will be taken step-by-step through the following sequences.-

 

STEP 1 : Define alignment

Load your data and triangles.

 

Select the Horizontal option HOR from the left menu.

From the active button menu select LOAD or NEW

if you want to load a previously saved (.PID) alignment.

 

 

STEP 2 : Extract cross sections

After completing the horizontal alignment definition, select the SEC

option from the left menu and then the X-SECTION from the bottom menu.

Model Maker will know that an alignment is in memory. Verify the use of this alignment file.

 

 

 

 

 

This box will show the start and end peg distances. You may also specify the interval on which sections must be extracted. To extract cross-sections at a regular interval at the same time and at some odd peg distances in between, press the User define button.

This will then activate a box to specify odd chainage positions where a cross-section must be extracted.

 

 

Enter any defined chainage.

If the direction of the section (usually for culverts) is not perpendicular to the center of the road, enter the direction in the direction column. This value is the actual direction (eg 180deg = north) and not the relative angle towards the road.

 

CONSTANT INTERVAL : You may specify the sections to be extracted at this regular interval in addition to the user definable intervals. Check the CALCULATE CONSTANT box to activate the option.

 

If the regular interval format is checked, the constant interval cross sections will be extracted at chainages that's a multiple of the constant value.

If the regular interval format is NOT checked, the constant interval cross sections will be extracted at chainages that's a multiple of the start chainage of the alignment PLUS the constant interval value.

 

Example: Start chainage = -47, constant interval 20

In the regular format the constant chainages will be

-47,-40,-20,0,20,40,60 etc...

If the regular interval is not checked the constant chainages will be

-47,-27,-7,13,33,53 etc..

 

Use the FILES open & save buttons to transfer your input values to files for reuse later.

The SORT list will sort your chainages. Insert data by adding it to the end and then sorting it to the correct position.

The CLEAR list will erase all the user chainages.

 

 

 

 

Include ID points on section as service offsets

Along each section, if identified dtm points are found then the offset and Z values of service point will be added to the x-section file.

 

ID PNT : Define chainages as defined by identified Dtm points on the alignment.

 

2 Point : Define chainage & direction by indicating 2 points either side of alignment.

 

ID CAD : Regular cad lines, poly cad lines and GXT strings can be used to define chainages and directions.

The number of identified elements are shown in brackets.

Only the entity type that has identified elements will be available to check.

 

After selecting the format of the output, specify the filename and directory.

 

You will then be prompted for the total width of the sections (left and right) to be extracted, and the maximum number of offsets to be stored. To extract the sections over the total width of your model, enter a value of 0. An offset will be calculated at each position where a triangle leg cuts the cross-section line.

 

Model Maker will now extract the cross-sections and display them on the graphics screen.

 

STEP 3 : Extract long-section

Select the sections option  SEC  from the left menu and then the L-SECTION option from the bottom menu. Model Maker will now know that there is an alignment in memory and you will be prompted to verify the use of this alignment. Select  CANCEL  if you would like to load another alignment file.

 

 

 

This box will show the start and end peg distances. Specify if the alignment is for a road or pipeline. The road alignment will extract the long section taking the radius at the curves into account. Sections for a pipeline will ignore the curve and go directly from PI to PI.

The section can be extracted at an offset from the centre. Enter the value as a negative if the offset is to the left.

 

To store the extracted long-section data to disk select  SAVE FILE.

 

Saved the data in the normal MODEL MAKER SYSTEMS  format or in the Caddie format.

 

You will then be prompted for a reference description and then for the interpolation type. The options are:

         *                Elevation from model

         *                Elevation from DTM points

 

Model Maker  will now extract the long-section and display it on the graphics screen. Note that Model Maker only extracted the data for the section and saved it in an ASCII file. This may now be plotted with a long- or cross-section program like PIPE MAKER or  ROAD MAKER,  or with  Model Maker's  module 7.
