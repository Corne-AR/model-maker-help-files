MENU : Data - General - Utilities: Constants & code changes        

                                 

 

The functions on the general data editing dialog box are used to do the following:

               Add constant values to the Y,X and Z co-ordinates of the data.

               Rotate the data around a fixed base point by adding a constant rotation angle.

               Change the codes of selected survey points.

 

 

Add a distance factor to coordinates. PLEASE NOTE!!! This will not work if your data is in MMS format, eg. a tache file.  You must save it as a MMB file to get the results.

 

Enter the base coordinates, leave the rotation at 0, enter the distance factor (other than 1),

check the ID points (if only they have to be changed) and select Apply constant

Add constants to co-ordinates

Any value may be added to either of the Y, X or Z co-ordinates of the DTM (survey), Cad or String data.

 

 

The sequence of steps to follow.

               Click in the Y, X or Z input boxes and enter the value to be added.

           Note:              If you only want to add a constant to one of the attributes, leave the other fields at zero.

 

 

 

To multiply any of the co-ordinates add a * sign to the constant value.

 

In this example the X co-ordinates will be multiplied by -1 which in effect means the sign of the co-ordinates are changed.

 

       In the To what type of data box, check the data types that must receive constant.
       Click the Apply constant button.

 

Note:

       Adding constants should only be performed on survey data consisting of KOORD stations.

       If the project consists of tachy stations, change the co-ordinates of the station data. Surveyed points belonging to a tachy                 station are  always calculated in relation to the station co-ordinates.

       Constants are only added to the current active DTM project.

       If you need to add constants to a project consisting of tachy stations, save the data in an ASCII file, and then read the                 ASCII back into Model Maker. The tachy project will be converted into data consisting of one KOORD station. You can then apply the Global editing functions to the data.

· Data converted into a KOORD station cannot be changed back into its original tachy field book format. Therefore, always                    save the tachy data in the tachy (.MMS) format even when it is also saved in ASCII (.YXZ) format. This allows you to recover                the original field book data if you find errors in the ASCII data.
