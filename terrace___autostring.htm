MENU : teR - Autostring

 

This quite unique function provides for an elegant solution to a very repetitive exercise by stringing a few existing  functions together.

Model Maker is equipped to produce platforms, update the elevations to get a balanced cut&fill excavation volume, create toe lines and generate a final platform model. Although different functions must be utilized to create the desired result, it is easy to do if only a few platforms need to be used.

The AUTOSTRING function was designed to do all these tasks automatically.

If you have purchase the TERRACE (Module 4) and STRINGS (Module 11) modules, the Autostring function will be available.

 

Back ground

This function requires that the model with the original ground surface be loaded in Project #1.

Project#2 accumulates all the designed platform models.

Project#3 develops each individual platform before it is added to Project#2. Because Project#3 is used for this purpose, it will be cleared of any existing data when the exercise begins. Ensure that Project#3 is cleared of any data or that the data in that project is not important and may be deleted by the system.

 

The outline of the platforms is placed in position by using the Strings functions. The best approach in developing a typical string is to first draw the platform using regular CAD lines and cad functions. Then add a general string on top of the drawn cad platform.

Usually most of the platforms will be same shape. Copy the first string to all the other locations using the Page up & down and other cursor functions to position it correctly.

Once all the strings are in position, activate the Autostring function.

 

The dialog box consists of three pages and will always display the DEFAULT page on activation unless there are no points or triangles in project #1. In this case, the third warning page will be displayed. If the warning page is displayed, exit the dialog box and get the correct surface model in project #1.

The default page contains all the settings used when the platforms are generated.

 

A string by itself cannot generate new triangles. New DTM points will be added to project#3 at all the nodes of the string. These DTM points will then be used to generate the triangles and the platform.

The new points added to the DTM will have a standard code and tag. These codes are very functional and used to identify the points and are essential attributes when setting out the points.

Enter the code to identify the points. Check the box if the points must each be coded individually.

Remember to enter a base code that ends in a numeric if the codes are to be incremented, eg., Plat1 or Top100.

         

As explained above, each platform is first developed in project#3. Check this box if each developed platform must be added to project#2 where a combined model of all the platforms will be maintained. This process involves the automatic usage of the Project Copy function.

 

This option will, after the toe lines have been generated around the platform, hatch the space between the edge of the platform and the toe line with regular CAD lines.

 

EXAMPLE: embankment hatching.

 

This option will place a CAD text string with the platform number right in the centre of the platform. This is useful as the platform number is also displayed in the volume calculation report.  Any platform in question can then easily be identified by locating the number on the drawing. Also specify the starting number for the first platform.

 

This is an important option. As explained above, the starting point for each platform is its defining string. Strings are 3 dimensional entities and can be defined with a well-defined elevation. When Model Maker defines new DTM points at the nodes of the string, the elevations of these points are taken to be that of the string itself. If the BALANCE box is not checked, the platform will be calculated - (volumes & toe lines) - using the string elevation values.

By selecting the balance option, an average elevation based on the platform position in relation to the surface of the model in project#1 will be calculated and a new relative elevation determined. All the elevations on the platform will be adjusted by a constant value and a first run volume amount will be determined. A few iterations using the grid volume method will be conducted and a new elevation will be determined to yield a balanced - within 5% - cut & fill excavation amount.

Notice that the initial elevation of the DTM points are taken from the defining string. If all the elevations of the string are the same (a flat string in terms of the vertical component) a flat platform (horizontal) will result. If the string had nodes with different elevations, then the resulting DTM points will also have the same height differences. If the platform is balanced and, for example, a constant of 1 meter must be added to all the elevations to balance it, the meter will be added to each point individually. The platform will have a different elevation to that when it was defined, but the height differences between the points will be preserved.

 

House keeping will delete any triangles that extend beyond the limits of the defining string. This may happen if the platform is designed in a L shape. Triangles are normally generated between the 2 legs  These triangles are mostly not applicable and the house keeping process will eliminate them. Only be in extreme situations would this box be left unchecked.

 

 

The final elevation of the terrace will be written in the centre of the area as regular cad text.

 

The cut and fill points will be added to project 4. This is normally done if you want to use the hatch option to hatch the cut and fill areas after the autoterrace option is completed.

 

 

 

 

Parameters

This button will lead you to the terrace dialog box. Here you can specify the grid size used to calculate the platform volumes. In the regular Terrace cut&fill option, you can output the results as line, text or circles. This option is not available in the automated process. Run the regular Terrace option between projects 2 & 1 again after the automated process is completed if you want to generate the output results.

 

The search radius used to search for new triangles to the toepoints is defined in this menu.

Note that Model Maker uses the normal continue search triangulation method when new triangles are added to the toepoints after the platform is balanced. This format of searching is based on the search radius set in the parameters menu. Check that the specified radius is large enough to be able to find triangles to the toepoints. The radius should be equal or greater than the longest side of the design platform.

Define the parameters that control the embankment lines (hatching).

 

 

Change the toe line parameters in this menu. The toe line parameters control the number of points and the embankment slopes. You can specify the cut and fill batter slopes ( ie Cut 1:1.5 and Fill 1:2)

 

Strings

 

 

The three buttons in the frame start the automation process.

 

 

A platform will be designed based on a single indicated string.

 

 

All the strings will be inspected and platforms generated. If you want to abort the process while Model Maker is busy with the strings, hit the Escape key a few times. This is a very involved process and the program may take a while to exit the routines.

 

Platforms will be built on all the strings with the defined name.

 

Hints:

When the process is completed, a report of all the calculated volumes is presented in the Editor. The center co-ordinate of the platform is also displayed. If the original string was flat (all the same elevations) the new designed elevation will be printed. If not all the nodes of the original string were on the same level, the word VARIES will be printed.

 

If a combined contour plan is to be made of the new designed platforms and the original ground model, use the Project Copy function and copy the data of project#2 down onto project#1

 

Note:

This is an automated process based on defined parameters and should, under normal conditions, produce the desired results. It is still important to closely inspect the new platforms and be satisfied that the model covers the intended area.

Activate project#2 and use the Side option to see the extend of the triangles.

If in doubt, do a trial run on a few platforms and first inspect the output before tackling a major exercise.
