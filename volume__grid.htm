The Grid Volume method is intended to calculate volumes of dumps and works as follows.

 

 

1) The user points to an island of triangles. If only one dump exists, then the island is in fact the whole model.

In this example there are 5 different dump models.

 

2) After a dump has been identified the program works on a selected grid and;

 

 a)  determine the elevation per grid point on the model

 b) interpolate a base elevation on the grid point as follows:

 

                       

 

For each grid point, radials are used to determine the shortest distance between the coordinates of the grid point and the perimeter of the model.

 

In this example the radial (orange circles) gives the shortest distance between the grid point and the opposite model perimeter.

 

Based on the elevations where the radial intersects with the perimeter, an elevation is interpolated at the grid point.

 

The difference between the elevation on the model and the interpolated point from the perimeter lines is then used to calculate the volume.

 

3) The differences between the model elevation and the interpolated elevation are then accumulated to determine the dump volume.

 

 

Advantages of this volume method:

 

1) The user does not have to generate a base model with triangles. The radial interpolation method simulates the base triangles automatically.

2) Unlike the volume by contours or volume by sections this method can handle cut and fill situations.

 

Cautions:

 

Since this method interpolates between the closest perimeter points of the model, it will only be accurate of the triangles between multiple dumps are trimmed so that each dump has it's own toeline. This means the user should delete any triangles that connect one dump to another.

 

Also be aware that interpolations are used to simulate the base model underneath the dump and the method should only be used if the original ground surface was fairly even.

If the you are uncertain about using this method, rather use the Terrace cut and fill methods which could be more accurate.
