MENU : Irr - Pipes

 

Model Maker defines an irrigation network with blocks, nodes and pipes. Nodes are hookup points for the pipes. Pipes convey the water between the nodes.

 

A separate database holds the information regarding all the pre defined pipe types. Before pipes can be added to the irrigation project, at least one default pipe type must be defined in the database. Also, nodes must be available to hook the pipes onto.

 

You have full control over the graphical manipulation of the pipes.

Extend Lats

 

This function is normally used after an automatic layout of the laterals has been performed in the block definition page and is intended to connect all the lateral pipes to the valve by means of the manifold.

The user would add the necessary nodes so that the manifold pipe is defined in the correct position. Model Maker will automatically calculate the intersections between the laterals and manifold, place new fittings at those points and extend the manifold pipes.

 

 

EXAMPLE: the lateral pipes are shown as point (a). The manifold pipe (b) is defined between the 3 orange nodes and the intersection points are marked as (c). The laterals will be extended to meet the manifold at (c) by placing an additional link lateral pipe between the last node of the lateral and the intersection fitting.

In this example the laterals will be cut between points A and B. To achieve this, indicate the 2 outside marker points at positions A and B. Irri Maker will draw a temporary line between A and B and determine all the intersection points with the crossing laterals. The crossed laterals segments will be divided in two by means of the fitting node.

Divide 2

 

This easy function allows you to divide any pipe in two segments.

         

 

The pipe on the left consist of one segment between 2 nodes. The layout to the right was achieved by using the Divide 2 function for a total of three times, each time dividing a segment and adding a new node.

Note:

This method cannot be used to divide a pipe and in the same function place the new node on top of another pipe and hope the second pipe will also be divided in two.

         

 

When using divide 2 between nodes 3 and 4, the new segments a and b will be created. If the new node (5) is placed onto of the pipe 1-2, that pipe will not automatically be cut into segments c and d. To obtain segments c and d, do one of the following.

 

1)         Delete pipe 1-2 and with add single, add the segment 1-5 and 5-2,

            OR;

2)        Use the Move pipe function and move endpoint 2 of pipe 1-2 to node 5 and then add a new pipe between 5 and 2.

 

Follow 1 CAD

 

This function will track a pre-defined CAD line or arc or continuously linked groups of lines and arcs, and define nodes at regular intervals. These nodes will automatically be linked up with the preselected pipe type. Any of the CAD functions can be used to define the original layout. Use this function to define an existing irrigation system or curved laterals.

         

EXAMPLE: 2 defined lines of pipes. On the top line, the nodes were added at regular intervals. This is normally the case when defining micro/drip systems on curved lines. On the bottom line the nodes were added to the end points. This would normally be the case when adding sprinklers or valves along a CAD line.

 

Follow Id CAD

 

Much in the same way as the Follow 1CAD function is used, Id CAD will track all the identified CAD lines and place nodes and pipes on it. The difference however is that you do  not need to indicate each CAD line that must be followed. Additionally you must indicate which line will be used as the base starting point.

 

EXAMPLE: 5 curved CAD polylines added parallel to each other. A line is also drawn between CC. The 5 lines would have beenidentified by means of their colour, layer or area. The line between CC should not be identified.

 

When the Follow Id CAD function is selected, define the node positions (regular intervals or node end points), the node type and pipe type. When asked to "Indicate Baseline to be used as Reference for starting the laterals", indicate line CC. With this information, Irri Maker will know to start each new lateral with the first node at side A of the CAD lines. Any non full interval will then be developed on side B.

Note:

This option will not track arc/circle CAD elements. If you really need to describe the position of the lateral by means of an arc, use one of the following methods to convert the arc into CAD lines.

 

       Use the CAD polyline function and manually trace over the arc. Use the near arc snap mode to place the line onto the arc        it self.
       Use the circle, divide option to automatically place co-ords along the circle.
       HINT : Although Model Maker cannot follow a single arc by itself, it can be overcome by adding CAD lines to the start and end of  the arc. This means that a combination of  identified lines and arcs will allow Model Maker to track the proposed layout.

 

 

This function allows you to develop the laterals along the contours without first having to draw cadlines as base elements.

This example shows a set of contours (brown), the outside of an irrigation block (green), the main regulating cadline (yellow) and the solid set base line (blue).

 

The (yellow) cadline will normally indicate the proposed position of the manifold and it will be used to determine the distance between laterals. Because of the variable slope within the irrigation block (depending on the survey points and contours), laterals cannot be following the contours AND have a constant spacing between them. The base (yellow) cadline is used to set the defined spacing between the laterals at the position where they cross this line. As the lateral then continues on to the block boundaries, the actual slope at regular intervals will then determine whether adjacent laterals grow closer or further apart.

 

MM will also ask what the absolute minimum spacing must be. The purpose of this value is to prevent laterals from growing too close to each other. Although each lateral will start at the position where it crosses the base cadline (yellow), at the nominal spacing away from the previous one, it may grow further away from it's neighbor or closer. If the absolute minimum is slightly less than the nominal spacing, the lateral may get closer to another but when it gets closer than the absolute minimum, it's development is stopped.

 

Here is an image of the laterals after following the contours.

Notice that at the base line intersections (red dots) the spacing is as defined. In the 2 areas (blue circle) the spacing got closer than the absolute minimum and development of that line stopped. When the slope of the terrain allowed it, the lateral continued again.

 

During the process you may see new cadlines being drawn ontop of the contours. These are temporary lines drawn by MM and assists in the development of the layout.

 

A few things to remember about the baseline.

·The baseline (yellow in this example) must be a regular polyline and have more than 1 segment. Even if it's only a straight line, define it with 3 nodes.
·Define the baseline to cross the contours at right angles. In the following example the yellow line is defined correctly. The red line is cutting across the contours and will not produce good results.

Display pipes

 

Use this check box to show or hide the plotting of the nodes. If the Display pipes is selected, all the pipes will be shown according to the following setup.

 

Only Active block

 

In general all the pipes of all the blocks are displayed on the plot. This can sometimes be confusing when you want to identify the position of the active block. By checking this check box, only the pipes of the current selected block will be displayed after doing a replot.

Remember to un-check this box before doing a printer plot unless you desire only to display the pipes of the active block on the paper plan.

 

Shift Colour

 

The pipes are normally drawn in the colours assigned to it when the pipes were added to the project. This is the colour specified in the node data base. Check the Shift Colour box if you want the pipes to be displayed in the colour of their allocated shifts.

 

Only pipes in shift #

 

This feature allows you to display only the pipes that are part of the last calculated shift number. You will also notice that this option is only selectively available and is only visible directly after the hydraulic calculations of the mainline has been done.

To make use of this feature, do the following:

         1) Design and calculate all the blocks.

         2) Define the mainline to feed water to all the blocks.

         3) Assign all the blocks to operate in desired shifts.

         4) Activate the mainline and use the hydraulics function to calculate the flows in a selected shift.

         5) Return to the pipes page, the "Only pipes in shift" will be available. Check the option and replot the design. Only the pipes of the active blocks in the calculated shift and the active mainline pipes are displayed.

 

Notice that this feature will automatically disappear when you make any changes to the design. To make it available again, you must use the calc option to recalculate the flows for the shift.

User defined pattern to display pipe detail

 

 

The Pat option allows you to use a specific pattern to display all the relevant information on the pipes in the mainline.

Use the regular cad functions and create a pattern based on the parameters as described below.

 

This is an example of a pattern comprising of cad text and lines. The elements have specific meaning and all the different settings are explained based on this example.

There are 10 pipe attributes that can be displayed on the pipe by means of a pattern. In addition, there are also 5 different unit values that can be displayed.

 

The 10 attributes are the following. Notice that the spelling must be exactly the same and in capital letters.

 

Five codes are used to display pipe attributes

 

CODEVAL: Pipe code

NAMEVAL: Pipe name

 

 

 

 

 

 

 

CLASSVAL : Pipe Group name

ODVAL        : Actual pipe outside diameter

IDVAL : Actual pipe inside diameter

 

Five codes are used to display pipe hydraulic values for each pipe segment.

 

FRICVAL        : Friction loss in the pipe

PRESSVAL        : Maximum calculated pressure in the pipe

FLOWVAL        : Flow volume in the pipe

LENVAL        : Length of the pipe

VELOCVAL        : Velocity in the pipe

 

The five unit display codes are:

 

PRESSUNIT        : Pressure

DIAMUNIT        : Diameter

FLOWUNIT        : Flow

LENUNIT        : Length

VELOCUNIT        : Velocity

 

Here is an example of the information displayed by the pattern when showing the pipe attributes.

Notice how the specific codes (cad text) have been replaced with the correct values.

 

There are 2 other specific settings that are also used for specific purposes.

 

Pen Color

a) Normally the elements in the pattern (text and lines) are drawn in the pen color used when the pattern was created.

If you want the elements to be drawn in the same color as the pipe, then they must be defined in the pattern with pen number 50.

Notice in the example, the first line of text and the arrows at the bottom. In the pattern the have a white color (pen 50 in the example). when the pattern is displayed on the pipe they are using the yellow pipe color.

 

Line Direction

b) In the example there are 2 arrows indicating the direction of flow. Since these arrows (lines) may have to indicate the opposite direction when the flow is not in the direction of the text, the layer DIRECTION is used to identify them. Any regular line element drawn on the layer DIRECTION will be switched by 180 degrees when the flow goes the opposite direction.

This example shows the arrows going the other direction.

 

In summary:

a) Use the pattern option when you want complete descriptions shown on the main line pipes.

b) Draw the pattern in cad using the specific codes to identify the attributes.

c) Use pen 50 if the color of the line or text element must be the same as the pipe.

d) Use layer DIRECTION if the lines must show the direction of flow.

e) Only regular text and regular lines should be used in the construction of the pattern.
