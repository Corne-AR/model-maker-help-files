MENU : Irr - Parts

 

Irri Maker makes adding part numbers to your irrigation project an easy process by providing you with a set of highly interactive tools.

 

Before the Parts page is explained, take note of the following:

               That part numbers do not have any influence on the layout and hydraulics of the system.

         Normally the layout is done and the hydraulics calculated before the parts are placed on the nodes.

               Part numbers only need to be added to the nodes if it is important to generate a complete bill of materials.

               A simple list of the number of different nodes and pipes can be generated without needing to manage the part numbers.

 

Background

 

The functions on this page are used to define the parts that makes up the assembly of each node.

The irrigation layout is developed by means of adding nodes and pipes to the design. At this stage the nodes are selected from the nodedatabase based on their hydraulic - pumps, sprinkler, valves etc. - attributes and only after the layout is completed and the hydraulics calculated, does one start to put the correct part numbers onto these nodes.

It is a two stage process.

               Do the layout of the irrigation design and calculated the hydraulics.

               Use the options on this - the parts page - to add the necessary parts numbers to the design.

 

 

Specify a scale factor so the selected nodes are more visible.

 

 

Adding the part numbers has a few facets

               Define the part and cluster numbers in the parts data base.

               Manually add the numbers to each node by :

         Editing each node and input the parts. (This is the long way)

         Use this parts editor and add the part to all the nodes of the same type.

               Automatically adding part numbers by teaching and applying the template option.

                         

 

PARTS to Nodes, grid editor

When the parts page is opened, all the nodes of the active block is displayed in the grid editor.

Nodes that have the following items in common, are grouped together.

               The same part numbers.

               The same node type, series and model name.

               The same number of connecting pipes with the same nominal diameters.

 

Part numbers

 

Each node used in the irrigation design layout can have up to 4 part numbers associated with it. Adding these part numbers is done by editing each node or by using the following options.

 

EXAMPLE: grid editor, showing the column headers and parts added to some nodes.

 

#

 

The number of nodes in the active block with the specific parts, node type and pipe combinations.

 

Part No. 1,2,3,4

 

The part numbers defined for these nodes. Normally, part no. 1 will hold the initial part number as defined in the node data base or which have been selected at the stage when the node was defined. On a new project, part numbers for the second, third and fourth part no. column will be blank. Addition part numbers, if needed, will be placed in these positions.              

 

Diam 1,2,3,4

 

The diameters of the pipes connected to the node. If for example only 2 pipes are connected to the node, only diam1 and diam2 will have values. Diam3 and diam4 will have zero values.

         

In the example, the node on line 1 has one pipe connected, the nodes on line 2 have three pipes and the nodes on line 3 have 4 pipes connected.

 

Adding part numbers manually

 

EXAMPLE:  some of the parts were automatically added to the nodes through the part definition in the node data base. The others could have been added manually or by using the template function.

 

EXAMPLE : to add an additional part number to the nodes in line 3:

               Point to the cell below Part No2 in the first row and double click. The parts dialog is displayed.

               From either the parts or cluster database, select the desired element and click on either the PARTS or CLUSTER button.

         The part / cluster number will be placed in the selected cell and added to all the nodes with the specific node type and pipe                combination. In this case there are 32 nodes with the same combination.
