MENU : dB - data base - Gen

 

 

The data base function in Model Maker allows you to access an existing or create a new data base file (dbf) that can be plotted along with other CAD or survey projects.

The DBF data base file structure is of a universal type and can be opened by other programs not familiar with Model Maker's own file format.

The data base can operate independently by displaying the data base information on the screen and on printed outputs or it can be used along with CAD and survey data. Information can also be transferred between the DTM points and the data base information.

 

Note:

A data base file can hold any type of information. For Model Maker to use it practically, some fields in the data base must at least contain a pair of co-ordinates which identifies in plan, the position of the data base entry.

 

 

 

 

Files

 

 

Open                                

Opens an existing dbf file, analyses the defined fields and reads the information into the editor window.

 

 

Close                                

Closes the dbf file. Any changes made to the data is updated in the file. The data base information will no longer be displayed on  the graphics screen. To temporary de-activate the plotting of the dbase points without losing the file, uncheck the display dbase option.

 

 

Import                                

This allows you to read ASCII coordinates directly from a co-ordinate file into the opened dbase structure. A dbf file must be        opened        and the co-ordinate links updated before this function can be used.

 

 

Exit                                

Exit the dialog box. The close button does the same work.

 

Create

Opens the dbf create and modify utility. The field structure of existing dbf files can be analyzed or a new file structure can be created. Use the file option to load or save modified structures.

 

Editor window

The first line holds the data base field names. In the example there are 5 fields with names "name, y ,x ,z, mm". If the field structure of this example is opened one finds that the "name" field is of type character, the "x,y,z" fields of type numeric and the "mm" field is a memo field.

A memo field can hold up to a page of data and cannot be displayed in the same line as the item. For that reason a separate window is created where the memo field data is displayed.

 

Memo window

To add or change the memo data, type the information directly in the memo window. The revised data of the active item will automatically be updated in the dbf file.

 

Navigate functions

The functions on this bar can be used to scroll up and down the data base, add and delete items. Read the hints on the buttons for more information.

 

 

 

 

Co-ordinate link to dbase fields

 

The fields in a dbf file can be of different formats and have different header names. The link options allow you to indicate which fields must be used to establish a connection between Model Maker's points format and the dbf file.

A dtm point in Model Maker consists of a code, tag, Y or easting, X or northing and a Z co-ordinate. To get the dbf file to plot correctly on the plan, fields with similar co-ordinate types must exist in the dbf file structure and then mapped to the associated Model Maker points fields.

Use the drop down buttons to display the available fields that can be mapped and select the appropriate field type.

 

 

Dbase / DTM Project

 

 

Dbase to project                

The items in the data base will be copied to the selected project as regular survey points. Note that a dtm point can hold a maximum of 6 characters in each of the code and tag fields. Items from a data base field longer than 6 characters will be truncated.

 

Project to dbase                

The DTM points of the selected project will be copied to the data base. This allows you to map the tag and code items to data base fields with lengths in excess of 6 characters or even a memo field.

 

Dbase / Graphic

 

Add                                

After a .dbf file is opened you may use this function to add points to the data base by means of the graphics screen. Use any of  the snap modes. A new entry in the data base will be created with the co-ordinates of the graphic position. Manually type additional information related to the new point in any of the other .dbf fields. Typically a memo field can be used, like in this example, to add additional information to the points.

 

Edit                                

Locate and then indicate the data base point on the graphics screen. The editor window will scroll to the correct item. You can then update the data.        

 

Move                                

Locate and then indicate the data base point on the graphics screen. Indicate a new position for the point. The co-ordinates in the data base will automatically be updated.

 

Plot Parameters

 

Display Dbase                

If checked the points of the active dbase file, if mapped correctly, will be plotted on the graphics screen. Uncheck the box if the data base items must not be displayed.

 

Code,tag,Z                

If checked, the data in these fields will be displayed on the graphics screen.

 

Plotting colour                

Set the colour used to plot the data base information.

 

Text Sizes

Set the text sizes to be used when plotting the tag, code and Z fields. Sizes are in millimeters at any scale.

 

Plot

The plot button allows quick access to the graphics window. Use the plot manipulate functions to update and move the graphics when needed.

 

Replot on close

If checked the graphics will automatically be refreshed when the dialog box closes.
