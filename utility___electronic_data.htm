MENU : Utility - Electronic data

 

 

This utility is used to import and convert survey data from various electronic data recorders, into Model Maker.

 

The process works as follows.

1) Some kind of hand held computer or device on your survey instrument is used to record your survey data.

2) A communication program is used to download the data from your capturing device onto your computer. These programs are normally made available by the supplier of your

    survey instrument or you can try and use the Download function on this menu.

3) The applicable function on this dialog box, relative to your data format, is selected. Model Maker will then read the survey file, use the applicable formulas and reduce the

    observations to final co-ordinates.

4) The reduced data is stored in the DTM data base. There it can be edited and the normal plotting functions used to display the data on the screen.

5) Finally the File Save options are used to save the data in a Model Maker file format.

 

Two file types are normally used to complete the reduction of the survey points.

1)                The observation file which holds the data of the station setup and surveyed points.

2)                The co-ordinate file which holds the coordinate values of the position of the setups and the trig beacons.

 

The actual process of locating the observation and co-ordinate files on your data recorder and transferring them onto your computer varies from one survey instrument type to another survey instrument type. The following discussions assumes that you are able to download the 2 required files and that they are on your computer and available to be processed.

Data Format

 

 

 

Gre

 

Load and reduce data recorded in the standard GRE format.

The default file extension is GRE but can be changed by pressing

the right mouse button while pointing to the GRE button.

 

Two GRE formats are used.

Two line station setup        : The Station information is on line 1and the orientation information on line 2.

Three line station setup        : The Station information is on line 1, the prism information on line 2 and the                                                                                                                                                                                                                                                             orientation information on line 3.

 

Inspect your specific data file and determine which of the options must be use. Inspect your results afterwards.

 

 

 

Sdr

 

Load and reduce data recorded in the standard SDR format. The default file extension is SDR but can be changed by pressing the right mouse button while pointing to the SDR button.

Model Maker can accommodate 4 different SDR formats. (Examples)

         Final co-ordinates                : The SDR file contains final reduced co-ordinates.

         Unreduced readings                : SDR survey data that must be converted to horizontal distance and height difference before the co-ordinates are calculated.

         Reduced readings                : SDR already in the format of horizontal distance and height difference.

         SDR 33 model                                : SDR data in the model 33 format.

If you are using these options for the first time, use the function most applicable to your survey data but also inspect the results carefully afterwards.

Geographical data (Latitude / Longitude)

This option will read an ASCII file with latitude and longitude coordinates and convert it to the Clark (LO) system. The normal ASCII import utility is used to allow the user to indicate the correct position of the co-ordinates within the file structure. Then the standard survey conversion routine is used to convert the latitude and longitude data to Y and X co-ordinates before it is added to the Model Maker internal data base and displayed on the screen.

Three things to keep in mind.

       The geographical coordinates must be formatted as degreeminutesecond.decimal seconds

 

                         EXAMPLE of GEO coordinates converted to              Clarke 31

                         Lat.  DMS                Long.  DMS.                                Y                                X              

                         294850.9588                303843.2817                                34283.033                3299306.930

 

       It is the responsibility of the user to specify the correct longitude of the central meridian (an odd degree value).

   The APPLY sea level and scale correction option is not relevant to this conversion.

 

It is important that you check the work and ensure that the conversion has been carried out correctly.

 

A good check would be to do the following.

               1) After the geographical data has been loaded and converted to YX co-ordinates and displayed on the screen, use the normal survey conversion routine and do the        following.

               2) Ensure that the is checked and convert the YX (Clarke) co-ordinates back to Geo format.

               3) The internal YX format will not be effected but the co-ordinates, along with the geographical equivalent will be displayed in the editor.

               4) Compare these reversed co-ordinates to your originals to ensure the accuracy.

 

EXAMPLE of a geographical file:

         Base Meridian for Y = Lo 28

         Name                                Lat.  D M S              Long.  D M S

         01                                2 29 44.7366      27 59 59.3090

         01                                2 29 44.7408      27 59 59.3090

         01                                2 29 44.7451      27 59 59.3090

 

 

 

To manually type data to convert, click the editor tab.

 

 

Sea level and scale correction

 

 

When the sea level and scale correction check box is checked, the following formula will be used to adjust the survey reading.

 

Note:

Even if the check box is not selected, the standard Curvature and refraction formulas will still be used to calculate the horizontal distance (HD) and height difference (HE).

 

1) Sea level correction (SL)                H/R*S

         H = Elevation of survey point.

         R = Radius of the earth as defined in defaults dialog box.

         S = Distance measured from station to surveyed point.

2) Scale correction (SC)                                Y^2/(2R^2)*S

         Y = Y co-ordinate value of the survey station.

         R = Radius of the earth as defined in defaults dialog box.

         S = Distance measured from station to surveyed point.

3) Temperature correction (TC)                1+281.95-((0.75*0.58440276^(H/4500))*1015))/(T+273.15)*105.5-0.5)/1e6

         H = The elevation of the station.

         T = The temperature as defined in the defaults dialog box.

 

4) Curvature and refraction

         Horizontal distance (HD)                =S*sin(VA)-(1-0.142/2)/6.372e6*(S^2)*sin(VA)*cos(VA)

         Height difference (HE)                =S*cos(VA)+(1-0.142)/(2*6.372e6)*(S^2)*sin(VA)*sin(VA)

         VA = Vertical Angle

         S = Distance measured from station to surveyed point.

 

The order in which the factors are applied:

         1) The measured slope distance between the station and surveyed point (S) is multiplied by the temperature correction (TC).

         2) The curvature and refraction formulas are then applied to calculate HD and HE

         3) The horizontal distance is then adjusted by adding the sea level and scale factors.

 

         S                = S * TC

         Curvature & refraction applied. HD and HE calculated.

         HD                = HD + SL +SC

 

Check the results

It is important that you should check the results after Model Maker has loaded and reduced the survey data, especially if it is the first time that you are importing data of a certain kind. All the formats are different and it cannot be guaranteed that any format you select is applicable to your field survey data files.

               Compare the data to calculations that you have done manually.

               Use a test run and load data that have been reduced in the past and compare the results.

               Reduce the data once with the sea level and scale corrections active and once without the corrections.

         Compare the results and satisfy yourself that the corrections have been applied in the desired format.

               When the data is plotted on the screen, measure and compare the distances between known points.

               Compare your plans to other plans of the same survey area produced by other systems.
