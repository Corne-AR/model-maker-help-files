Use the DWG/DXF/DGN to read a dxf file in a normal way without manipulation.

 

 

 

If you need to read a dxf and have to change coordinate systems or add the elements to Dtm points, use the Pre-2004 format.

 

The data may be read into either the Survey, CAD or Strings environments.

 

 

Use the Pre-2004 option if you need to change any of the options in it's dialog box.

 

 

 

 

Survey

Any co-ordinated point in the .dbf file will be added to the current active DTM project. The following options will regulate the importing process.

 

Two options can specify if any of the .dxf points should be skipped and not added as DTM points.

         Num                : Check this box for each n'th point to be read. In the example, 1  means that every                                            point will be read. A 4 would read every fourth point.

         Meter               : Use this option to thin out the points by the distance between them. A new point will                                            added only if it is the specified distance from the point added previously.

         Load Zero?    : Check this box for Model Maker to ignore any point that has no elevation or if the                                            elevation is zero.

 

The above options will be used mostly on .dxf files containing splined contours. If they are not filtered, an unpractical number of survey points will be generated.

 

 

3D face

Convert 3D face elements in the .dxf to points in the DTM. The advantage of this option is that Model Maker will automatically create triangles between the 3 points forming the 3D face. A complete survey model of points and triangles may be transferred in this format. Use the DXF out function to export a 3D face .dxf.

 

 

CAD

Read the complete .dxf to Model Maker CAD.

 

 

Strings

Read the DXF and convert all the defined polylines to General Strings.

This is often useful when the .dxf contains zero elevated contour polylines that must be converted to DTM points after the strings have been re-elevated correctly.

 

 

 

Select the correct co-ordinate system to translate the .dxf to Model Maker format. .dxf files generated by survey programs like Model Maker are usually  read with the As Is option.

 

If the .dxf file is read in this format, and there's a mirror or transposed picture on the screen, reread the file with a different co-ordinate selection.

 

As Is :  The coordinates will be saved or read without any sign change.

 

Swap : The Y and X coordinate will be swapped in the process.

 

Invert: The sign of the coordinates will change by multiplying the coordinates with -1

 

Swap and invert. Both options will be applied. The coordinates will be swapped around and both coordinates be multiplied by -1.

 

 

 

Some programs compile their .dxf files in millimeters. Model Maker converts the values.

 

 

 This option is only applicable on .dxf files containing arcs or half circles. Arcs in Model Maker are usually drawn from directly south and continue clock-wise. Other formats start the arc looking east and go counter clock-wise. Use the hemis option in CAD to change the format of arcs.

 

Model Maker reads patterns included in the .dxf file. These patterns are normally loaded from the file and added to the CAD drawing as pattern entities. The explode on load option instructs Model Maker to automatically add the contents of the pattern as regular CAD entities (lines,circles, text etc.) to the drawing. This also prevents your own generated patterns from being overwritten by patterns in the .dxf with the same pattern name.
